Broker — the accountability primitive EXPLAIN

TL;DR: The broker binds one market observer to one position observer in an N×M grid. It owns a minimal state machine: outcome counts, a grace-rate gate, and an active direction. The broker submits papers to the treasury, discovers outcomes by reading paper state, encodes anxiety atoms, and records results. 8 fields, 4 methods, 6 tests. No paper trades, no accumulators, no PropagationFacts, no EMA, no runner history — all stripped in the backlog sweep.

What it is#

"The accountability primitive. Binds one market observer + one position observer. The broker owns the game — gate 4, anxiety atoms, exit/hold decisions. The treasury owns papers and capital." — broker.rs

The broker sits at the intersection of a market observer (which predicts direction) and a position observer (which proposes risk parameters). It receives chains from the post, submits papers to the treasury, checks outcomes, and encodes anxiety atoms for gate 4 (to be built).

Crucially:

  • The broker submits but the treasury owns papers and capital.
  • The broker records outcomes via record_outcome() — simple counts, no PropagationFacts.
  • The broker gates itself with a grace-rate threshold (> 0.5).

The N×M Grid#

Brokers live in a flat array indexed by slot_idx. With N market observers and M position observers:

slot_idx = market_idx × M + position_idx
market_idx = slot_idx / M       (integer division)
position_idx = slot_idx % M

Each broker's slot_idx is its identity — it determines which market observer and which position observer it binds. For example: slot_idx=5, M=3 → market_idx=1, position_idx=2. Verified in test_market_position_idx.

Methods (4 total)#

Broker::new(observer_names, slot_idx, position_count)#

Constructor. Asserts position_count > 0 (divide-by-zero guard). All counters start at zero. active_direction starts as None (cold start).

market_idx() / position_idx()#

Grid derivation from slot_idx and position_count. Pure integer math:

  • market_idx = slot_idx / position_count
  • position_idx = slot_idx % position_count

gate_open()#

Two-phase gate:

pub fn gate_open(&self) -> bool {
    let cold_start = self.grace_count < 50 || self.violence_count < 50;
    cold_start || self.expected_value > 0.5
}
  1. Cold start — if either outcome count is below 50, the gate is always open. Not enough data to judge.
  2. Warm — after ≥50 of both Grace and Violence, the gate opens only when expected_value > 0.5. A broker must have a majority grace rate to keep trading. Note: 0.5 is NOT enough — the threshold is strictly greater than 0.5.

record_outcome(outcome)#

Updates the track record:

pub fn record_outcome(&mut self, outcome: Outcome) {
    match outcome {
        Outcome::Grace => self.grace_count += 1,
        Outcome::Violence => self.violence_count += 1,
    }
    self.trade_count += 1;
    self.expected_value = self.grace_count as f64 / self.trade_count as f64;
}

Simple grace rate. No EMA, no dollar P&L, no swap fees, no PropagationFacts return value. The method mutates the broker directly.

The Broker Program (Thread Body)#

The companion file broker_program.rs (207 lines) contains the broker's runtime loop.

Function signature#

pub fn broker_program(

    cache: CacheHandle<ThoughtAST, Vector>,
    vm: VectorManager,

    console: ConsoleHandle,

Returns the trained Broker when the chain source disconnects — "the accounting comes home".

Per-candle loop (5 steps)#

  1. Direction — Extract Direction from chain.market_prediction. Set broker.active_direction.
  2. Submit paper — Direction determines asset pair (Up: source→target, Down: target→source). Call treasury.submit_paper(). If approved, push receipt to active_receipts.
  3. Discover outcomes — Iterate active_receipts, call treasury.get_paper_state() for each. Active papers are kept. Grace/Violence outcomes call broker.record_outcome(), then the receipt is removed.
  4. Encode anxiety atoms — For each active receipt, compute 4 atoms and encode them via the cache. Plus an active-positions count atom. The encoded vector is available for gate 4 (to be built).
  5. Logging — BrokerSnapshot every 100 candles. PhaseSnapshot every candle (slot 0 only). Telemetry every candle. Console diagnostic every 1000 candles.

Anxiety atoms (4 per receipt)#

Computed by anxiety_atoms(receipt, current_candle, current_price):

Atom Encoding Description
candles-remaining Log Candles left until deadline. Floored at 1.0
time-pressure Linear age / total_life — fraction of position lifespan consumed
unrealized-residue Linear (current_value - amount) / amount — percentage unrealized P&L
paper-age Log How old the paper is in candles. Floored at 1.0

Plus one global atom: active-positions (Log encoding of receipt count).

Verified Properties#

  1. No shared mutation. The broker owns its 8 fields and mutates only itself. No observer references.
  2. Gate has two distinct modes with a clear boundary at 50 resolved trades.
  3. expected_value is simple grace rate — grace_count / trade_count. Not EMA, not dollar P&L.
  4. Gate threshold is > 0.5, not > 0.0. A 50/50 broker is suppressed.
  5. 6 tests cover construction, grid math, cold-start gate, low-grace gate, high-grace gate, and outcome recording.
