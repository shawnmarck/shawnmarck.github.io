What is MarketObserver?

A struct that predicts price direction (Up/Down) from candle-encoded thought vectors. It does not label itself — reality labels it via broker propagation. Each observer has a "lens" (vocabulary selection) and is one of N per-asset-pair market observers in the architecture.

How does predict() (observe) work?

observe() is the prediction path (there is no method literally named predict — prediction lives in observe):

  1. Clone raw thought — preserved for consumers who need both raw and cleaned vectors.
  2. Update noise subspace — the noise model learns the current thought as part of the background distribution.
  3. Strip noise — anomalous_component() projects the thought onto the noise subspace and returns the residual. This residual IS the signal (Proposal 024).
  4. Predict on the anomaly — the Reckoner.predict() runs on the noise-stripped vector, not the raw one.
  5. Compute edge — calls reckoner.accuracy_at(conviction) to get the accuracy edge at the current conviction level (0.0 if curve not valid).
  6. Store last_prediction — maps the reckoner's label index (0→Up, 1→Down) to Direction.
  7. Return ObserveResult — contains raw_thought, anomaly, prediction (scores + conviction), and edge.

Key design: prediction and learning are aligned — both operate on the anomaly, not the raw thought.

How does it learn?

Learning happens in resolve(), called by broker propagation when a trade settles:

  1. Map direction to label — Direction::Up → index 0, Direction::Down → index 1.
  2. Check correctness — compares last_prediction against the actual direction.
  3. Reckoner learns — reckoner.observe(thought, label, weight) feeds the labeled experience to the discrete classifier.
  4. Feed the conviction curve — a second predict + resolve cycle updates the internal conviction→accuracy curve with the actual correctness signal.
  5. Engram gate check — accumulates recalib_wins and recalib_total, then calls check_engram_gate() which evaluates whether the observer has proven itself. The engram gate uses good_state_subspace to learn what good discriminant states look like. Gate state (wins/total/recalib_count) is updated in place.

The observer is described as "its own teacher. The market is the judge." It self-grades previous predictions when new candles arrive.

Constructor (new)

  • Reckoner: discrete, two labels ("Up", "Down"), named "direction-{lens}".
  • noise_subspace: 8 principal components.
  • good_state_subspace: 4 principal components.
  • resolved starts at 0. last_prediction defaults to Down (arbitrary).
  • Takes lens, dims, recalib_interval, and window_sampler.

Helper Methods

  • strip_noise(thought) — standalone noise-stripping, returns the anomalous component. "The residual IS the signal."
  • experience() — delegates to reckoner.experience(). Returns how much the observer has learned.

Tests (6)

All use 4096-dim vectors from VectorManager, DowTrend lens, recalib interval of 100, WindowSampler::new(42, 10, 500):

  1. test_market_observer_new — verifies initial state (lens, resolved=0, last_prediction=Down, recalib counters=0).
  2. test_observe_returns_result — observe() returns result; anomaly has correct dimensions.
  3. test_strip_noise_returns_vector — strip_noise() returns a DIMS-length vector.
  4. test_resolve_increments_counters — resolve() increments resolved to 1, recalib_total to 1.
  5. test_experience_starts_at_zero — new observer has 0 experience.
  6. test_observe_sets_last_prediction — observe sets last_prediction to Up or Down.
  7. test_multiple_resolves — 10 resolves → resolved=10, recalib_total=10.

No test validates prediction accuracy, noise stripping quality, or engram gate behavior. Tests are structural, not behavioral.
