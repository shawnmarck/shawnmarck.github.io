T0-008 — Reckoner: The Learning Primitive HOW

Verified against source. Every claim below is traceable to reckoner.rs (1107 lines), core/primitives.wat (lines 28–83), and std/memory.wat (28 lines). Nothing is fabricated.

1. What It IS

The Reckoner is one primitive, two readout modes:

  • Discrete mode — N-ary classification with symbol labels. "Which of these labels does this thought match?" Returns a Prediction with per-label cosine scores, a winning direction, and a conviction strength.
  • Continuous mode — Bucketed nearest-neighbor regression over (thought, scalar) pairs. "Given this thought, what scalar should I predict?" Returns an interpolated float.

The module docstring states:

"Discrete mode: N-ary discriminant learner with symbol labels. Continuous mode: nearest-neighbor regression over (thought, scalar) pairs. Same accumulation mechanism. Same decay. Same recalibration."

Both modes share a common Reckoner struct. The mode is determined at construction via ReckConfig and is immutable for the lifetime of the instance.

The WAT Specification

The formal spec calls the Reckoner a coalgebra — an opaque state machine with co-generators:

"Opaque state. The co-generators produce all observable behavior. Removing any one makes the external characterization incomplete." — primitives.wat

It defines four operations + three introspection functions:

WAT operationRust equivalentRole
(observe reckoner thought label weight)Reckoner::observe()State transition
(predict reckoner thought)Reckoner::predict()Observation (query)
(decay reckoner rate)Reckoner::decay()Aging

(recalib-count reckoner)Reckoner::recalib_count()Introspection: how many recalibrations

2. Construction

The Reckoner struct


pub struct Reckoner {
    name: String,           // human-readable name
    dims: usize,            // vector dimensionality
    recalib_interval: usize,// how often to rebuild discriminants
    updates: usize,         // total observe() calls
    recalib_count: usize,   // times recalibrate() has run
    last_cos_raw: f64,      // cosine between first two prototypes (diagnostic)
    last_disc_strength: f64,// discriminant magnitude / sqrt(dims)
    mode: ReckMode,         // Discrete or Continuous
}

Constructor


Reckoner::new(name, dims, recalib_interval, config) -> Reckoner
  • dims: Must match the dimensionality of all Vector objects passed to observe/predict/query.
  • recalib_interval: Every N observe() calls in discrete mode, recalibrate() runs automatically.
  • config: ReckConfig::Discrete(label_names) or ReckConfig::Continuous { default_value, buckets }.

Discrete mode initialization

On construction with ReckConfig::Discrete(vec![...]):

  • Creates one Accumulator per label (all zeroed).
  • Creates discriminants as vec![None; N] — no knowledge yet.
  • Creates mean_proto as None.
  • Initializes the curve-fitting state: resolved = empty vec, resolved_cap = 5000, min_resolved = 500, n_bins = 20, curve_a = 0.0, curve_b = 0.0, curve_valid = false.

Continuous mode initialization

On construction with ReckConfig::Continuous { default_value, buckets }:

  • Creates K ScalarBucket instances, each with an Accumulator(dims) and center: 0.0.
  • range_min = 0.0, range_max = 0.0, total_observations = 0, obs_since_rebalance = 0, initialized = false.
  • The spec notes: "Range is discovered from observations. Decay + rebalance = breathing range. No magic min/max. The data teaches the range. K is the only parameter."

Chainable configuration


.with_curve_params(resolved_cap, min_resolved, n_bins) -> Self  // discrete only, no-op for continuous
.with_max_observations(max) -> Self  // legacy no-op: "Bucketed accumulators don't cap — they compress them."

3. Discrete Mode

3.1 observe(vec, label, weight) — State Transition


pub fn observe(&mut self, vec: &Vector, label: Label, weight: f64)

Panics if called on a continuous-mode reckoner (with message directing to observe_scalar()).

Computation path:

  1. Bounds-check: if label.index() >= accumulators.len(), silently returns (no panic).
  2. Increment self.updates.
  3. If updates % recalib_interval == 0, call recalibrate().
  4. Call accumulators[idx].add_weighted(vec, weight) — adds the vector weighted into the label's accumulator.

The Accumulator::add_weighted implementation: for each dimension s, s += (v as f64) weight, then count += 1. This is frequency-preserving: "Unlike bundling, accumulation preserves frequency information: patterns seen many times contribute proportionally more."*

3.2 predict(vec) — Observation


pub fn predict(&self, vec: &Vector) -> Prediction

Returns Prediction::default() for continuous mode or when no discriminants exist yet.

Computation path:

  1. If all discriminants are None, return default prediction (no knowledge).
  2. For each label i:
  • If mean_proto is Some(mean): compute stripped = vec - mean componentwise, then cosine(stripped, discriminant[i]).
  • If mean_proto is None: compute cosine(discriminant[i], vec) directly.
  • If discriminant is None for this label, cosine = 0.0.
  1. Sort scores by cosine descending (highest raw cosine wins — not absolute value for sorting).
  2. The direction = the label with the highest raw cosine.
  3. The raw_cos = the actual signed cosine of the winning direction.

The Prediction struct:


pub struct Prediction {
    pub scores: Vec,      // cosine per label, ordered by absolute cosine descending
    pub direction: Option

Important note on sorting vs. direction selection: Scores are sorted by raw cosine descending in the code (b.cosine.partial_cmp(&a.cosine)), but conviction is set to best.cosine.abs(). This means the winning label is the one with the highest raw cosine, and conviction measures magnitude.

3.3 recalibrate() — Internal (called automatically)


fn recalibrate(&mut self)

No-op for continuous mode. Only meaningful when label_count >= 2 and all accumulators have at least one observation.

Computation path:

  1. Compute mean_proto: the componentwise mean of all N prototypes.
  2. Diagnostic (for 2-label case): store cosine(protos[0], protos[1]) as last_cos_raw.
  3. For each label i:
  • Compute other_mean = componentwise mean of all other prototypes.
  • Compute discriminant: disc = protos[i] - other_mean componentwise.
  1. Increment recalib_count.

The discriminant formula per the WAT spec:

"discriminant(label) = prototype(label) - prototype(other)."

For binary classification, this simplifies to prototype(A) - prototype(B), which is the classic Fisher-like linear discriminant direction.

3.4 decay(factor) — Aging


pub fn decay(&mut self, factor: f64)

Works for both modes. Calls accumulator.decay(factor) on every accumulator.

Accumulator::decay: sums *= factor. The count is not decremented — it tracks history, not current weight.

"Useful for time-weighted baselines where recent patterns should have more influence."

3.5 resolve(conviction, correct) — Curve Fitting Input


pub fn resolve(&mut self, conviction: f64, correct: bool)

Discrete mode only (no-op for continuous).

Pushes (conviction, correct) onto the resolved vec. If resolved.len() > resolved_cap, the oldest entry is removed (remove(0)). This is a sliding window of bounded size.

3.6 curve() — Self-Evaluation


pub fn curve(&mut self) -> Option<(f64, f64)>

Discrete mode only. Fits the conviction-accuracy relationship.

Prerequisites (any failure → returns None):

  • n_labels >= 2
  • resolved.len() >= min_resolved (default: 500)
  • bin_size >= 10 (i.e., resolved.len() / n_bins >= 10)

Fitting procedure:

The model is: accuracy = base + a × exp(b × conviction), where base = 1/N (random chance).

  1. Compute base = 1.0 / n_labels, base_epsilon = base + 0.005.
  2. Sort resolved entries by conviction ascending.
  3. Divide into n_bins (default: 20) equal bins. Last bin gets remainder.
  4. For each bin: compute mean_c (mean conviction) and acc (fraction correct).
  5. Filter: only keep bins where acc > base_epsilon (accuracy exceeds chance).
  6. For kept bins: transform y = ln(acc - base). This linearizes: ln(acc - base) = ln(a) + b × conviction.
  7. Require at least 3 valid points.
  8. Linear regression on (conviction, ln(acc - base)) points:
  • b_val = (n·sxy - sx·sy) / (n·sxx - sx²)
  • ln_a = (sy - b_val · sx) / n
  • a_val = exp(ln_a)
  1. Cache curve_a = a_val, curve_b = b_val, curve_valid = true.
  2. Return Some((a_val, b_val)).

The WAT spec confirms:

"accuracy = (1/N) + a × exp(b × conviction), N = number of labels. At zero conviction, accuracy = 1/N (random chance among N labels). The curve IS the proof. Monotonically increasing = real signal."

3.7 accuracy_at(conviction) — Evaluate Curve


pub fn accuracy_at(&self, conviction: f64) -> Option

Returns None if curve not valid or fewer than 2 labels.

Formula: min(base + curve_a × exp(curve_b × conviction), 0.99) — capped at 99%.

4. Continuous Mode

4.1 observe_scalar(vec, value, weight) — State Transition


pub fn observe_scalar(&mut self, vec: &Vector, value: f64, weight: f64)

Panics if called on a discrete-mode reckoner.

Computation path:

  1. Increment updates and total_observations.
  2. Implicit decay: every observation decays all buckets by 0.999. The code comments: "every observation is a tick of time".
  3. First observation: if !initialized:
  • Set range_min = value - 0.001, range_max = value + 0.001.
  • Distribute K bucket centers uniformly across [range_min, range_max].
  • Set initialized = true.
  1. Range expansion: if value < range_min or value > range_max, expand the range and call rebalance_continuous().
  2. Bucket assignment: idx = floor((value - range_min) / width).min(K - 1). Add the weighted vector to buckets[idx].accumulator. Weight is floored at 1e-10.
  3. Periodic rebalance: every 100 observations, call rebalance_continuous() and reset obs_since_rebalance.

4.2 rebalance_continuous() — Internal


fn rebalance_continuous(buckets, k, range_min, range_max, dims)

This is the "breathing" mechanism. It contracts the range to where active weight lives.

Computation path:

  1. If fewer than 2 alive buckets: just reset bucket centers on current range (no contraction).
  2. Compute new_min = min(alive_centers), new_max = max(alive_centers).
  3. If range is degenerate (< 1e-10), return.
  4. Add 10% margin: margin = (new_max - new_min) × 0.1. Clamp new_min to max(0.0, new_min - margin). Extend new_max += margin.
  5. Replace buckets: allocate K fresh empty accumulators. Set new centers uniformly across [new_min, new_max].
  6. Pour old data: for each alive old bucket, find its nearest new bucket by center position, and merge() the accumulator in.

This is how the continuous reckoner adapts its range to the data without hardcoded bounds.

4.3 query(vec) — Observation


pub fn query(&self, vec: &Vector) -> f64

Returns default_value if no observations yet, or 0.0 if called on discrete mode.

Computation path:

  1. Convert the thought to f64, compute its norm.
  2. Score each bucket: for every bucket with count > 0:
  • Compute cosine between the thought and bucket.accumulator.raw_sums().
  • Keep only buckets where cos > 0.0.
  • Store (cos, bucket.center) pairs.
  1. If no scored buckets, return default_value.
  2. Soft-weight top-3: sort by cosine descending, take top 3.
  • Weighted interpolation: result = Σ(cos_i × center_i) / Σ(cos_i).
  1. If weight sum is negligible (< 1e-10), return default_value.

The docstring explains: "Cosine against K bucket prototypes, soft-weight top-3, interpolate. O(K × D) — constant in observation count."

The interpolation means: "interpolate, don't argmax" — the reckoner returns a blend of the three closest bucket centers, weighted by how well the thought matches each bucket's prototype direction.

6. Key Properties

6.1 Accumulation, not storage

The Reckoner never stores individual observations. It uses Accumulators that maintain running weighted sums of vectors. The key insight:

"Unlike bundling, this does NOT threshold after each addition, preserving frequency information." — accumulator.rs

6.2 Cosine as the sole measurement

All similarity computation uses cosine similarity. The WAT spec states:

"(cosine thought direction) → Float ; [-1.0, +1.0]"
"The only measurement. One number."

The discrete discriminant is compared via cosine. The continuous bucket prototypes are compared via cosine. The conviction is an absolute cosine. This is consistent throughout.

6.3 Frequency preservation vs. bundling

  • bundle([a, a, a, a, a]) = a (idempotent — bundling is a set operation).
  • accumulator.add(a) × 5 preserves that a was seen 5 times (the sum vector has 5× magnitude).

6.4 Decay is multiplicative on sums, not on count

decay(factor) calls sums *= factor. The observation count is never decremented. From the test:

"Count stays — it tracks total observations ever, not current weight. The accumulator sums decay to near-zero but the count is history."

6.5 Continuous range is data-driven

The continuous reckoner has no hardcoded value range. The first observation seeds a tiny range (±0.001). Every out-of-range observation expands the range and triggers rebalancing. Every 100 observations, the range contracts to where alive weight still lives. The comment:

"Range is discovered from observations. Decay + rebalance = breathing range. No magic min/max. The data teaches the range. K is the only parameter."

6.6 Recalibration is periodic, not event-driven

In discrete mode, recalibrate() is called every recalib_interval observations (checked modulo in observe()). It is a batch operation: extract all prototypes, compute discriminants, write them back. There is no incremental update.

6.7 Curve fitting validates the reckoner's own confidence

The curve mechanism answers: "When this reckoner says it's confident, is it actually right?" By recording (conviction, correct) pairs and fitting accuracy = 1/N + a·exp(b·conviction), the system gets a self-calibration measure. A positive b means higher conviction correlates with higher accuracy — the reckoner knows what it knows.

6.8 Computational complexity

  • Discrete predict: O(L × D) where L = labels, D = dimensions.
  • Continuous query: O(K × D) where K = buckets, D = dimensions. The top-3 selection is O(K log K) for the sort, but K is typically small (e.g., 10).
  • Both are constant in the number of observations — the accumulator compresses everything into a single D-dimensional sum vector.

6.9 The with_max_observations is a no-op

The method exists as a legacy API. The code comments:

"Legacy — was max observations for brute-force mode. Now a no-op. Bucketed accumulators don't cap observations — they compress them."
