T-1-006 — Lab Order (ORDER.md)

TL;DR: ORDER.md is not an order-type specification for trading. It is the construction order — a dependency-sorted reading/inscribing sequence for the .wat specification files that implement the trading system. Each file can only depend on files listed above it. The order IS the path from leaves (no dependencies) to root (depends on everything).

Source

wat/ORDER.md — 109 lines, ~5KB.

What It Actually Is

Despite the filename, this document does not define trading order types (market, limit, stop, etc.). It defines:

  1. A reading order for the project documentation
  2. A dependency-sorted file list for the .wat specification files
  3. The inscription rule — each file is written after its dependencies and verified against the guide before proceeding

The project uses "order" in the sense of sequence/arrangement, not in the financial order sense.

Reading Order (Section 1)


1. GUIDE.md                    — the specification (read first, always)
2. CIRCUIT.md                  — the visualization (read after the guide)
3. wat files (leaves to root)  — the implementation
4. src/ (leaves to root)       — the compiled code (future)

The guide is the source of truth. The circuit visualizes it. The wat implements it. The Rust compiles it. The market proves it.

The Inscription Rule

Each file is inscribed after its dependencies. Each file is judged by the ignorant against the guide before the next is inscribed.

This is the project's verification discipline — a form of incremental, dependency-ordered review. No file is written until its inputs exist and have been verified.

External Dependencies (from holon-rs)

The wat files depend on these holon-rs primitives (not defined in the wat layer):

  • VectorManager — vector storage and operations
  • Reckoner — learning mechanism (discrete and continuous readout)
  • OnlineSubspace — noise reduction for reckoners

Notes

  • The file lists exit-observer.wat; per CLAUDE.md this was renamed to "position observer" in Proposal 050, but ORDER.md has not been updated.
  • src/ is listed as "(future)" in the reading order — the Rust implementation exists but the ordering document was written before it was complete.
  • The dependency graph is a DAG (directed acyclic graph) — the topological sort from leaves to root ensures no circular dependencies.
  • Total: ~40+ wat files across all layers.
