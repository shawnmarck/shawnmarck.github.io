T0-015 — PaperEntry (Rust) — Verification

Source: holon-lab-trading/src/trades/paper_entry.rs (370 lines)

Method: Context-free ignorant — read only the file and its immediate type dependencies.

Derived from: wat/paper-entry.wat

What is a PaperEntry

A hypothetical ("paper") trade inside a broker. It does not execute. It tracks a single predicted direction (Up or Down) against live prices to measure whether the market observer's prediction was correct. Two triggers govern its lifecycle:

  • Trail (Grace): A trailing stop that follows the best price in the predicted direction. Signals confirmation that the prediction was right.
  • Stop (Violence): A fixed stop loss for capital protection. Signals the prediction was wrong.

Construction (new)

Takes: paper_id, three thoughts (composed, market, position), prediction, entry_price, distances, entry_candle.

Initial state:

  • extreme = entry price.
  • trail_level = entry price (for both directions).
  • stop_level:
  • Up: entry - entry * distances.stop (below entry).
  • Down: entry + entry * distances.stop (above entry).
  • signaled = false, resolved = false, age = 0.
  • price_history = vec![entry_price].

tick(&mut self, current_price: f64)

No-op if already resolved. Otherwise increments age, appends price to price_history, then branches on direction.

Direction::Up

  1. Update extreme: If current_price > extreme, set extreme = current_price.
  2. Update trail: Compute new_trail = extreme - extreme * distances.trail. If new_trail > trail_level, advance it (trail only ratchets in the favorable direction).
  3. Check stop (Violence): If current_price <= stop_level → resolved = true, return immediately.
  4. Check Grace signal: If extreme >= entry + entry * distances.trail → signaled = true.
  5. Check trail fire: If signaled && current_price <= trail_level → resolved = true.

Direction::Down (mirror)

  1. Update extreme: If current_price < extreme, set extreme = current_price.
  2. Update trail: new_trail = extreme + extreme * distances.trail. If new_trail < trail_level, advance it.
  3. Check stop (Violence): If current_price >= stop_level → resolved = true, return immediately.
  4. Check Grace signal: If extreme <= entry - entry * distances.trail → signaled = true.
  5. Check trail fire: If signaled && current_price >= trail_level → resolved = true.

Key ordering: Stop (Violence) is checked before Grace signal. If both would fire on the same tick, Violence wins (the return after stop check prevents Grace from being set). This means a price that simultaneously hits the stop and the trail-cross threshold is classified as Violence, not Grace.

Resolution Classifications

MethodConditionMeaning
is_grace()signaled (regardless of resolved)Market confirmed the prediction. Trail crossed at some point.
is_violence()resolved && !signaledStop fired before trail ever crossed. Prediction was wrong.
is_runner()signaled && !resolvedTrail crossed, still running. Not yet resolved.

These three are not mutually exhaustive across all states but are mutually exclusive in any given state:

  • Unsignaled and unresolved: none of the three return true.
  • Signaled and resolved: is_grace() = true but is_violence() = false and is_runner() = false.

excursion() -> f64

How far price moved in the predicted direction, as a fraction of entry.

  • Up: (extreme - entry) / entry
  • Down: (entry - extreme) / entry

Always ≥ 0 by construction (extreme only moves favorably). Zero at creation.
