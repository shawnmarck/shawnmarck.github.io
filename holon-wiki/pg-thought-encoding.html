Thought Encoding — the pipeline from thought to vector EXPLAIN

TL;DR

Thought encoding transforms structured observations (ASTs emitted by vocabulary-modules) into high-dimensional vectors. The pipeline has three layers:

  1. Vocabulary emission — modules emit ThoughtAST trees (Atom, Linear, Log, Circular, Bind, Bundle, Sequential) with round_to quantization at emission time to make float keys cache-safe.
  2. ThoughtEncoder — a stateless recursive AST walker that evaluates each node into a Vector using ScalarEncoder for scalars, VectorManager for atoms, and Primitives (bind, bundle, permute) for composition. Zero internal cache.
  3. EncoderService — a single-threaded pipe loop that owns the persistent LRU composition cache, serving N concurrent callers via per-caller pipe sets. Callers compute on miss and fire-and-forget install back. "Values up, not queues down."

An optimization layer — IncrementalBundle — avoids re-encoding unchanged facts between candles via running element-wise sums with sign-threshold, proven bit-identical to full re-bundle. Downstream, EngramGate gates reckoner quality by snapshotting discriminant vectors into a good-state subspace after periods of accurate prediction.

Vocabulary modules produce ASTs — WHAT to think. The ThoughtEncoder evaluates them — HOW to think efficiently. Walks the AST bottom-up, checking memory at every node. Minimum computation.
— thought-encoder.wat, lines 1–5

The AST — ThoughtAST enum

Seven variants, each mapping to a specific evaluation strategy:

VariantFieldsStrategy
Atom(name)StringDirect dictionary lookup from VectorManager. Never misses.
Linear { value, scale }f64, f64Encode scalar via ScalarEncoder with ScalarMode::Linear { scale }.
Log { value }f64Encode scalar via ScalarEncoder::encode_log(value).
Circular { value, period }f64, f64Encode scalar via ScalarEncoder with ScalarMode::Circular { period }.
Bind(left, right)two ThoughtASTRecursively encode both, then Primitives::bind(&l, &r).
Bundle(children)VecEncode all children, then Primitives::bundle(&vecs). Superposition (set semantics).
Sequential(items)VecEncode each child, permute by positional index via Primitives::permute(&v, i), then bundle. Order-sensitive: [A, B] ≠ [B, A].
"Same structure, same vector."
— GUIDE.md, line 1475

The AST structure IS the cache key — structural equality (with f64::to_bits() for reliable hashing) determines identity.

round_to — cache key quantization


let factor = 10f64.powi(digits as i32);
(v * factor).round() / factor

Called at vocabulary emission time, not at encode time. The ThoughtAST carries the rounded value. This is load-bearing: without quantization, floating-point drift would cause false "changed" detections in incremental bundle diffing, degrading to full recompute.

"The cache key IS the exact AST. The rounding happens at emission, not evaluation."
— thought-encoder.wat, lines 30–32
"round_to at vocab emission is load-bearing for the AST diff. Quantized floats compare reliably. Remove round_to and this degrades to full recompute (correct, but no savings)."
— T0-011, lines 208–211

"ThoughtEncoder is stateless — no cache. Production encoding goes through encoding::encode::encode() which checks the LRU. This struct exists for tests and IncrementalBundle."
— T0-011, module doc

encode(&self, ast: &ThoughtAST) -> Vector

Recursive AST walk. Each call re-evaluates the entire subtree. For each variant:

  • Atom → self.vm.get_vector(name) — direct lookup.
  • Linear/Log/Circular → delegate to ScalarEncoder.
  • Bind → encode left, encode right, Primitives::bind.
  • Bundle → encode all children, Primitives::bundle. Empty → Vector::zeros(dims).
  • Sequential → encode each child, Primitives::permute(&v, index), then bundle. Empty → Vector::zeros(dims).

The wat specification returns a tuple (Vector, Vec<(ThoughtAST, Vector)>) — the vector plus accumulated cache misses. The Rust implementation strips this to just the vector; misses are handled externally.

EncoderService — the distributed cache

The EncoderService is a single-threaded pipe loop that owns the persistent LRU composition cache, serving multiple concurrent callers without any mutex.

"ThoughtEncoder cache as a single-threaded pipe loop. The encoder holds an LRU cache. Each caller gets its own pipe set (EncoderHandle). The loop iterates all pipes once per iteration. No mutex. One thread. N callers. One cache."
— encoder-service.wat, lines 1–7

EncoderHandle — per-caller pipe set

PipeCapacityDirectionPurpose
lookupbounded 1caller → serviceSend a ThoughtAST for cache lookup
answerbounded 1service → callerReceive Some(Vector) (hit) or None (miss)
installunboundedcaller → serviceFire-and-forget insertion of computed (AST, Vector) pair

Service thread loop — three phases per iteration

  1. Drain installs — empty all install pipes, insert (AST, Vector) pairs into LRU cache.
  2. Service lookups — for each pending lookup, check cache, send Some/None back. Increment hit/miss atomics.
  3. Select & block — idle until any channel has data. Zero CPU when idle. If all handles disconnected → shutdown.

Protocol


Caller: write AST to lookup pipe → block on answer → receive Some/None
Caller: if None, compute locally → write (AST, Vector) to install pipe (fire and forget)
Encoder: one pass per iteration — drain installs, service lookups, select, repeat.
"Does NOT hold sender copies — the handles ARE the senders. When all handles drop, the channels close, the cascade flows, the encoder thread exits."
— encoder-service.wat, lines 39–41

Three Arc counters provide lock-free reads for hit-count, miss-count, and cache-len.

IncrementalBundle — optimization, not cognition

Maintains running element-wise sums across candles to avoid re-encoding unchanged facts. Proven bit-identical to full re-bundle.

"Can be reconstructed from one full encode."
— thought-encoder.wat, line 108
FieldTypePurpose
sumsVecRunning element-wise sums. threshold(sums) == bundle(all facts).
last_factsHashMapPrevious candle's facts for diffing.
dimsusizeVector dimensions.
initializedboolWhether at least one full encode has been performed.

The algebra

"bundle = threshold(Σ vectors). If fact k changes from old to new, sums_new = sums − old + new. threshold(sums_new) == bundle(all current facts). Proven bit-identical. Integer addition is commutative and associative."
— thought-encoder.wat, lines 110–113

encode — incremental diff

  • First candle: full encode — encode all facts, populate sums and last_facts.
  • Subsequent candles: diff against last_facts. Removed facts subtracted. Unchanged facts skipped. Changed/added facts encoded and added. Then threshold().

threshold — sign function

s > 0 → 1, s < 0 → -1, s == 0 → 0. Converts integer sums back to the bundled binary vector space {-1, 0, +1}.

EngramGate — gating by subspace alignment

EngramGate is downstream of encoding. It gates reckoner quality by accumulating discriminant vectors from periods of accurate prediction into a "good-state subspace."

After each resolved prediction, the caller increments win/total counters and calls check-engram-gate. When a recalibration boundary is crossed and accuracy exceeds a threshold, the reckoner's discriminant is snapshot into the good-state subspace. The subspace accumulates knowledge of what good discriminants look like over time.

"Tracks wins/total since last recalibration. When recalib-count advances and accuracy exceeds the threshold, snapshots the discriminant into the good-state subspace."
— GUIDE.md, lines 1685–1687

Supporting functions

extract(thought_vec, forms, encode_fn) → Vec<(ThoughtAST, f64)>

Encodes each candidate form, computes cosine similarity against a target vector. Returns all (ast, score) pairs with no threshold, no filtering — the consumer decides. Marked rune:reap(scaffolding) — awaiting Phase 6 discriminant decode.

collect_facts(ast) → Vec

Recursively collects leaf-level factual statements from an AST. Bundles are transparent (recurse into children). Bare Atoms are excluded ("name without value — not a fact"). Bind, Linear, Log, Circular are returned as facts.
