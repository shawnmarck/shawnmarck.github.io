Chain Types EXPLAIN

The typed values that flow through the organism.

TL;DR — Two pure-data structs in src/programs/chain.rs (40 lines) carry all information between pipeline stages. The type name is the proof of which stage produced it. No methods, no behavior, no shared references — just values on the wire.

Design Principle#

"Each stage produces a new value. The type IS the proof of which stage produced it. No methods. No behavior. Values on the wire." — chain.rs module doc, via T0-017

Chain types enforce a linear, typed pipeline through the organism: the main thread produces an enriched candle, the market observer wraps it in a MarketChain, the position observer flattens that into a MarketPositionChain, and the broker consumes it. At no point does any thread hold a shared mutable reference to another's state — every piece of information is either owned outright or carried behind an Arc that is never written through.

MarketChain — Clone#

Producer: Market Observer
Consumer: Position Observers (via fan-out)
Transport: Bounded Queue (capacity 1) into market observer; Topic (1→N) out to position observers.

Field Type Meaning
candle Candle The enriched OHLCV candle that triggered this step
window Arc<Vec<Candle>> Window of recent candles (currently length 1)
encode_count usize Global monotonic encode counter
market_raw Vector Raw encoding of market features via VectorManager
market_anomaly Vector Anomaly component (deviation from noise subspace)
market_ast ThoughtAST The abstract syntax tree that was encoded
prediction Prediction Direction prediction (Up / Down)
edge f64 Edge / confidence score

Implements Clone because a Topic clones the message to every subscriber. This is the only chain type that needs fan-out.

MarketPositionChain — not Clone#

Producer: Position Observer
Consumer: Broker
Transport: Bounded Queue (point-to-point, one broker per chain).

Flattens all MarketChain fields — duplicated, not borrowed — then adds position-specific fields:

Inherited Field (renamed) Original Meaning
candle MarketChain::candle Same enriched candle
window MarketChain::window Same window
encode_count MarketChain::encode_count Same counter
market_raw MarketChain::market_raw Same market raw encoding
market_anomaly MarketChain::market_anomaly Same market anomaly
market_ast MarketChain::market_ast Same market AST
market_prediction MarketChain::prediction Renamed from prediction
market_edge MarketChain::edge Renamed from edge
New Field Type Meaning
position_raw Vector Position observer's own raw encoding
position_anomaly Vector Position observer's anomaly vector
position_ast ThoughtAST Position observer's AST
position_distances Distances Trailing stop and safety stop distances from Reckoner

Does not implement Clone. The deliberate choice to duplicate (rather than borrow) market fields makes this struct self-contained but large — no shared references, no lifetime entanglement, pure ownership transfer through the queue.

Data Flow Through the Grid#

flowchart TD
    main["Main thread\nsend ObsInput"] --> mkt_obs["Market Observer i\n(thread)\nencode → predict"]
    mkt_obs -->|"Topic fan-out\nsend MarketChain"| pos0["Position Observer 0"]
    mkt_obs --> pos1["Position Observer 1"]
    mkt_obs --> posM["Position Observer M"]
    pos0 -->|"send MarketPositionChain"| broker["Broker [i, ei]\n(thread)"]
    pos1 --> broker
    posM --> broker
    broker -.->|"ObsLearn → market observer i's mailbox"| mkt_obs
    broker -.->|"PositionLearn → position observer ei's mailbox"| posM
    broker -.->|"TradeUpdate → position observer ei's trade mailbox"| posM


See signal-propagation for the full wiring table including ObsLearn, PositionLearn, and TradeUpdate return-path signals.

Key Design Choices#






  Choice
  Rationale






  Pure data, no methods
  Types carry information; behavior lives in the program functions that produce and consume them.




  Flattened, not composed
  MarketPositionChain duplicates market fields rather than holding a MarketChain field. This avoids lifetime complexity and shared-reference entanglement.




  MarketChain clones; MarketPositionChain does not
  Clone is required only for fan-out to multiple position observers. Brokers receive point-to-point, so ownership transfer suffices.




  Arc<Vec<Candle>> for window
  The one shared allocation — read-only, never mutated after creation.




  Type-is-proof
  The compiler ensures a MarketPositionChain can only be produced by a position observer. Passing the wrong type to the wrong stage is a compile error, not a runtime bug.
