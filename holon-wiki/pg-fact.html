Fact — a scalar claim about the world, encoded as a vector.

TL;DR: A fact is a named scalar value emitted by a vocabulary module, represented as a ThoughtAST leaf node, and encoded into a fixed-width bipolar vector via the scalar encoder. Facts are the atomic inputs to holonic cognition — no booleans, no categories, only signed scalars. The discriminant learns where the boundaries are.

What a fact is

From the GUIDE:

"No zones. No categories. Only scalars. 'Overbought' is a human label on a continuous value — a magic number wearing a name. WHO decided 70 was the boundary? The vocabulary emits 'RSI is at 0.73.' The discriminant learns where the boundaries are."
— T2-004 (vocabulary), GUIDE lines 1398–1402
"Every relationship is a signed scalar. Not 'close is above SMA20' — the relative distance, with sign. [...] The sign IS the direction. The magnitude IS the distance. No 'above' atom. No 'below' atom. No boolean. Just the number."
— T2-004 (vocabulary), GUIDE lines 1346–1356

A fact has three components:

ComponentExampleRole
Namersi, dist-from-sma200, phaseIdentifies what is being measured
Scalar value0.73, -0.023, peakThe quantitative claim
Encoding schemeLinear { scale: 1.0 }, Log, Circular { period: 24 }Determines how value maps to vector space

Vocabulary modules are "pure functions. Something in, facts out. No state." (T2-004, GUIDE line 1245).

ThoughtAST representation

Facts are represented as leaf nodes in the ThoughtAST enum. The four factual variants are:

AST VariantFieldsMeaningExample
Linear { value: f64, scale: f64 }value, scaleNaturally bounded scalarLinear { value: 0.73, scale: 1.0 }
Log { value: f64 }valueUnbounded positive scalar (log-compressed)Log { value: 5.2 }
Circular { value: f64, period: f64 }value, periodPeriodic/wrapping scalarCircular { value: 14.0, period: 24.0 }
Bind(Atom, …)left, rightA named scalar fact (name bound to value)Bind(Atom("rsi"), Linear { … })

Bare Atom(name) nodes are not facts — they are names without values. The collect_facts function explicitly skips them: "name without value — not a fact" (T0-011, line 174).

All four factual variants — Linear, Log, Circular, and Bind — are returned as-is by collect_facts, while Bundle and Sequential containers are transparently flattened (T0-011, lines 341–352).

Encoding schemes

The encoding scheme determines how a scalar value maps into vector space. Three schemes exist (T2-004, GUIDE lines 1376–1395):

"The encoding scheme IS the bounding strategy."
SchemeUse caseKey propertyExamples
Linear { scale }Naturally bounded scalarsEqual absolute differences → equal similarity dropsRSI [0,1], Bollinger position [-1,1], distances scale: 0.1
LogUnbounded positive scalarsEqual ratios → equal similarity dropsVolume ratio, ATR ratio, band widths
Circular { period }Periodic scalarsEndpoint wraps to startHour (24), day-of-week (7), minute (60)
"Linear: Equal absolute differences have equal similarity drops — 100 → 200 has same similarity drop as 1000 → 1100."
— T0-004 (scalar encoding), scalar.rs:9–11
"Logarithmic: Equal ratios have equal similarity drops — 100 → 1000 has same similarity drop as 1000 → 10000."
— T0-004 (scalar encoding), scalar.rs:13–15
"For periodic quantities (hour of day, day of week, angle), use circular encoding where the endpoint wraps to the start."
— T0-004 (scalar encoding), scalar.rs:19–20

The bounding philosophy

"Some facts are bounded. Some aren't. That's honest. The log doesn't bound — it compresses. The circular doesn't bound — it wraps. Only linear needs bounds, and linear's bounds come from the math."
— T2-004 (vocabulary), GUIDE lines 1393–1395

How a fact becomes a vector

When ThoughtEncoder::encode() walks a factual AST leaf (T0-011, lines 147–198):

  1. Linear { value, scale } → calls ScalarEncoder::encode(value, ScalarMode::Linear { scale })
  2. Log { value } → calls ScalarEncoder::encode_log(value) (log₁₀ then rotational encoding)
  3. Circular { value, period } → calls ScalarEncoder::encode(value, ScalarMode::Circular { period })
  4. Bind(left, right) → encodes both children, then Primitives::bind(&l_vec, &r_vec)

The output is always a fixed-width bipolar vector in {−1, 0, +1}^dim (typically dim=4096). Closer scalar values produce more similar vectors — verified by tests confirming sim(0,10) > sim(0,50) (T0-004, scalar.rs:331).

Quantization awareness

"bipolar thresholding causes quantization; use accumulator averaging to recover gradient behavior."
— T0-004 (scalar encoding), scalar.rs:206–207

Log encoding in particular "works best when used with accumulators that average multiple samples, recovering gradient behavior" (T0-004, scalar.rs:364–365).

Fact lifecycle in the system


Vocabulary module (pure function)
    ↓ emits Vec
collect_facts() — flattens bundles, strips bare atoms
    ↓ produces leaf-level facts only
ThoughtEncoder::encode() — walks AST to vectors
    ↓ cached via external LRU in encoding::encode::encode()
IncrementalBundle — maintains running sums across ticks
    ↓ threshold(sums) == bundle(all facts)
Downstream: lens observers, discriminants, accumulators

The IncrementalBundle optimizes repeated encoding by maintaining element-wise sums and diffing facts between ticks. It depends on round_to at vocabulary emission time — "load-bearing for the AST diff. Quantized floats compare reliably" (T0-011, lines 208–211).

Key properties of facts

  1. Deterministic: Same seed → same vector. ScalarEncoder::with_seed(4096, 42) is reproducible (T0-004, scalar.rs:403).
  2. Stateless emission: Vocabulary modules are pure functions — no cache, no side effects (T2-004, GUIDE line 1244).
  3. Similarity gradient: Nearby values → high cosine similarity; distant values → low similarity. This is the core property that enables discriminant learning.
  4. Conditional emission: Some facts fire only when a condition is active (e.g., rsi-divergence-bull only when divergence > 0). The number of facts per tick can vary.
  5. Named, not categorized: The vocabulary emits the raw scalar. It does not label "overbought" or "trending." Those interpretations belong to the discriminant layer.
