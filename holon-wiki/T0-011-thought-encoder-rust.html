Eq implementation (line 84)

Blank impl Eq for ThoughtAST {}. Relies on the derived PartialEq. Safe because PartialEq for f64 uses bit equality, and the derived impl compares all fields structurally.

Hash implementation (lines 86–113)

Uses std::mem::discriminant first, then hashes fields by variant. For f64 fields, calls to_bits() to get integer representation before hashing — avoids NaN-related issues and ensures consistent hashing.

Method: ThoughtEncoder::encode(&self, ast: &ThoughtAST) -> Vector (lines 147–198)

Recursive AST walk. No caching. Each call re-evaluates the entire subtree.

How encode() walks the AST, branch by branch:

AST variantAction
Atom(name)Calls self.vm.get_vector(name) — direct lookup from VectorManager.
Linear { value, scale }Calls self.scalar_encoder.encode(value, ScalarMode::Linear { scale }).
Log { value }Calls self.scalar_encoder.encode_log(value).
Circular { value, period }Calls self.scalar_encoder.encode(value, ScalarMode::Circular { period }).
Bind(left, right)Recursively encodes left → l_vec, encodes right → r_vec, then Primitives::bind(&l_vec, &r_vec).
Bundle(children)Maps encode() over all children, collects into Vec, then Primitives::bundle(&refs). If empty, returns Vector::zeros(dims).
Sequential(items)Maps encode() + Primitives::permute(&v, index as i32) over enumerated items, collects, then Primitives::bundle(&refs). If empty, returns Vector::zeros(dims).

Key detail for Sequential: each item is first encoded to a vector, then permuted by its positional index (i as i32) via Primitives::permute, THEN all permuted vectors are bundled. This makes [A, B] ≠ [B, A] because A gets permute(0) in position 0 and permute(1) in position 1, etc.

Method: IncrementalBundle::encode(&mut self, new_facts: &[ThoughtAST], encoder: &ThoughtEncoder) -> Vector (lines 240–283)

First candle (!self.initialized): delegates to full_encode().

Subsequent candles — incremental diff:

  1. Build a HashSet<&ThoughtAST> from new_facts for O(1) lookup.
  2. Removed facts: for each (old_ast, old_vec) in last_facts not present in the new set, subtract old_vec from sums.
  3. New/changed facts: for each fact in new_facts:
  • If it existed in last_facts with the same key (Eq+Hash match): skip — sums already has this contribution.
  • Otherwise: encode it via encoder.encode(fact), add to sums, insert into new last_facts.
  1. Replace self.last_facts with new_last_facts.
  2. Call self.threshold().

Critical invariant from comments (lines 208–211): "round_to at vocab emission is load-bearing for the AST diff. Quantized floats compare reliably. Remove round_to and this degrades to full recompute (correct, but no savings)." The HashMap key equality depends on f64::to_bits() — which is bitwise. Without round_to, floating-point drift would cause false "changed" detections.

Method: IncrementalBundle::full_encode(&mut self, facts: &[ThoughtAST], encoder: &ThoughtEncoder) -> Vector (lines 286–304)

Zeroes sums, clears last_facts. Encodes every fact, accumulates into sums, stores in last_facts. Sets initialized = true. Returns threshold().

Method: IncrementalBundle::threshold(&self) -> Vector (lines 307–313)

Sign-threshold the sums buffer: sign(s) → 1 if s > 0, -1 if s < 0, 0 if s == 0. This is how bundling works: the bundle of a set of vectors is the element-wise sum, then sign-threshold to get back to {-1, 0, +1} values. Proven "bit-identical" to a full bundle from scratch (comment line 207).

Free function: extract(thought_vec: &Vector, forms: &[ThoughtAST], encode_fn: F) -> Vec<(ThoughtAST, f64)> (lines 323–332)

Generic over F: Fn(&ThoughtAST) -> Vector. For each form in forms, encodes it via encode_fn, computes Similarity::cosine between the form vector and thought_vec. Returns all (ast, cosine_score) pairs — no threshold, no filtering. The consumer decides.

Comment: "On hot paths, pass encoding::encode::encode (which checks the LRU). At startup or in tests, pass ThoughtEncoder::encode directly."

How caching works (the full picture from source)

ThoughtEncoder has zero caching. The struct holds only a ScalarEncoder and a VectorManager — both stateless evaluators.

Production caching is external. The module doc (line 5) says: "Production encoding goes through encoding::encode::encode() which checks the LRU." The extract function's doc (line 321) confirms: "On hot paths, pass encoding::encode::encode (which checks the LRU)."

IncrementalBundle has its own caching mechanism — the last_facts: HashMap field. But this is an optimization cache (avoiding re-encoding unchanged facts between candles), not the production LRU. It depends on round_to at vocab emission to ensure AST key equality is bitwise-reliable.
