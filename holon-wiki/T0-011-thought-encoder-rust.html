DELETED: Struct ThoughtEncoder#

ThoughtEncoder struct has been removed from source. It previously held ScalarEncoder and VectorManager and walked the AST tree to produce vectors. Encoding now lives in encoding::encode::encode() which uses:

  • EncodeState — per-entity L1 LruCache<u64, Vector> (default 16 384 entries)
  • CacheHandle — L2 shared cache program behind pipes (cross-entity sharing)
  • Precomputed u64 hashes for O(1) cache lookups

The encoding algorithm is the same recursive AST walk, but now cached at two tiers.

DELETED: Struct IncrementalBundle#

IncrementalBundle struct has been removed from source. It previously maintained running Vec<i32> sums across candles for incremental bundle updates (avoiding re-encoding unchanged facts). This optimization was removed in favor of the two-tier cache (L1/L2) in encoding::encode, which absorbs the same redundancy — repeated facts hit the L1 cache after first encode.

How caching works (the full picture from source)#

No caching in thought_encoder.rs. This file is purely data definitions (ThoughtASTKind, ThoughtAST, helpers).

All caching lives in encoding::encode (302 lines). The encode() function uses a two-tier cache:

  1. L1 — per-entity LruCache<u64, Vector> (default capacity 16 384). Created once per thread via EncodeState::new(). Absorbs repeated lookups across candles for the same entity.
  2. L2 — shared CacheHandle (cross-entity cache program behind pipes). Handles cross-entity sharing — different entities that encode the same ThoughtAST hit the L2 instead of recomputing.

Cache keys are precomputed u64 hashes. ThoughtAST::new() hashes the kind at construction time. ThoughtAST::hash() returns this value in O(1). No tree walking during cache lookup — nanoseconds per key.

The AST-as-value invariant is load-bearing: vocabulary modules embed quantized scalar values in ThoughtAST nodes (round_to at emission, Hash via to_bits). Changed values produce different hashes. Stale cache entries are unreachable, not incorrect.
