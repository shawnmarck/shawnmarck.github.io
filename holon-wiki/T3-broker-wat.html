T3 — Broker (wat spec) Verification WHAT

Source: holon-lab-trading/wat/broker.wat (224 lines) Approach: Context-free ignorant — read only the file, quote what it declares.

Preamble (lines 1–8)#

The accountability primitive. Binds one market observer + one exit observer. N×M brokers total. Holds papers. Propagates resolved outcomes to every observer in the set. Measures Grace or Violence. The broker's identity IS the set of observer names it closes over.

Dependencies declared:

  • primitives, enums, distances, scalar-accumulator, engram-gate, paper-entry, simulation

A paper has two sides (buy and sell). Each side resolves independently. Each resolved side produces one Resolution with its own direction.

Interface: Methods (6 functions)#

1. make-broker (lines 71–95)#

Constructor. Creates the reckoner as :discrete with labels ("Grace" "Violence"). noise-subspace gets k=8, good-state-subspace gets k=4. All accumulators init to zero. Papers init as empty deque.

2. propose (lines 97–104)#

Signature:

Noise update → strip noise → predict Grace/Violence via the reckoner.

3. edge (lines 106–113)#

Signature:

(edge [broker : Broker]) : f64

Reads from the reckoner's internal curve. Returns conviction→accuracy. 0.0 = no edge. Used for proportional treasury funding.

4. register-paper (lines 115–121)#

Signature:

(register-paper [broker : Broker]
                [composed : Vector]
                [entry-price : f64]
                [distances : Distances])

Creates a PaperEntry and pushes onto the papers deque. Called every candle, every broker.

5. tick-papers (lines 123–166)#

Signature:

(tick-papers [broker : Broker] [current-price : f64]) : Vec<Resolution>

Ticks all papers. Each side (buy/sell) resolves independently. Produces a Resolution the tick a side fires. Paper removed only after both sides resolved. Outcome is :grace if excursion > trail-distance, else :violence.

6. propagate (lines 168–220)#

Signature:

(propagate [broker : Broker]
           [thought : Vector]
           [outcome : Outcome]
           [weight : f64]
           [direction : Direction]
           [optimal : Distances]
           [recalib-interval : usize]
           [scalar-encoder : ScalarEncoder])
  : PropagationFacts

The core learning function. Five internal steps:

  1. Reckoner learns — observe the outcome with weight
  2. Feed the internal curve — resolve conviction vs correctness
  3. Track record — increment cumulative grace/violence and trade-count
  4. Scalar accumulators learn — observe trail and stop distances from optimal
  5. Engram gate — check gate state, update recalib counters

Returns PropagationFacts for the post to apply to its observers. Derives market-idx and exit-idx from slot-idx / exit-count and slot-idx mod exit-count.

7. paper-count (lines 222–224)#

Signature:

(paper-count [broker : Broker]) : usize

Returns len of the papers deque.

Summary#

The wat spec declares:

  • 3 structs: resolution, propagation-facts, broker
  • 6 fields in broker for accountability (reckoner, noise-subspace, track record ×3, papers)
  • 4 fields for scalar learning (2 accumulators in a vec)
  • 4 fields for engram gating (good-state-subspace + 3 recalib counters)
  • 7 interface functions: make-broker, propose, edge, register-paper, tick-papers, propagate, paper-count
  • Identity model: slot-idx is the primary identity; observer-names is the diagnostic identity
  • Learning is values-up: propagate returns PropagationFacts for the post to distribute; the broker does not directly message observers
  • Reckoner mode: :discrete (Grace/Violence) — the spec explicitly declares this despite CLAUDE.md saying "continuous reckoners only" (Proposal 051). The wat says Discrete.
