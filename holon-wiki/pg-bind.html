Bind — the self-inverse combinator EXPLAIN

TL;DR

bind is the core relational primitive of the holon-rs vector algebra. It composes two concept vectors into one via element-wise multiplication over bipolar values {-1, 0, +1}. Its defining property is self-inverseness: bind(bind(A, B), A) = B exactly (not approximately). The WAT spec names it one of four generators of the Vector Algebra. In Rust, it is implemented as Primitives::bind, with unbind exposed as a semantic alias. Every structured encoding in the system — from Walkable role-filler pairs to ThoughtAST Bind nodes — ultimately reduces to calls to bind.

Bind — the self-inverse combinator

Algebraic definition

From the WAT spec (wat/core/primitives.wat):

"(bind role filler) → Vector"
"Compose two concepts into a relationship."
"Self-inverse: (bind (bind A B) A) = B"

The WAT defines the Vector Algebra as:

"A commutative monoid under bundle, with bind as the group action and cosine as the measurement homomorphism."

Bind is the group action — the operation that creates structure (relationships) out of flat atoms.

Implementation

The Rust implementation lives in holon-rs/src/kernel/primitives.rs:


/// Bind two vectors (element-wise multiplication for bipolar).
///
/// Creates an association between two concepts. The result is
/// dissimilar to both inputs, but `unbind(bind(A, B), A) ≈ B`.
///
/// For bipolar vectors, this is element-wise multiplication.
pub fn bind(a: &Vector, b: &Vector) -> Vector {
    let mut out = Vector::zeros(a.dimensions());
    Self::bind_into(a, b, &mut out);
    out
}

The operation is literally o = x y per dimension — element-wise multiplication of bipolar values. A parallel f64 pipeline also exists (bind_f64), identical in semantics.

unbind is exposed as an explicit alias:


/// Unbind a key from a bound vector.
/// For bipolar vectors, unbind is identical to bind (self-inverse property):
/// `unbind(bind(A, B), A) = B`
pub fn unbind(bound: &Vector, key: &Vector) -> Vector {
    Self::bind(bound, key)
}

Why self-inverse works

For bipolar vectors where each element is {-1, +1}: (a × b) × a = a² × b = 1 × b = b. This is exact, not approximate — confirmed by the Rust test:


#[test]
fn test_bind_self_inverse() {
    let ab = Primitives::bind(&a, &b);
    let b_recovered = Primitives::bind(&ab, &a);
    assert_eq!(b, b_recovered);  // exact equality
}
Source: T0-002-primitives, WAT spec line "Self-inverse: (bind (bind A B) A) = B", Rust test_bind_self_inverse.

Where bind appears in the encoding pipeline

In the Walkable encoder

The Walkable trait's encode_walkable recursive traversal uses bind for role-filler encoding on every Map node:


for each (key, value):
    role_vec   = encode_atom(key_path)        // key → vector
    filler_vec = encode_walkable_value(value)  // value → vector
    bound = bind(role_vec, filler_vec)         // role-filler binding
bundle(all bounds)                             // superimpose all bindings

The same pattern applies to List/Spread encoding, where positional indices serve as roles:


for each (i, item):
    pos_vec  = encode_atom("field.[i]")
    item_vec = encode_walkable_value(item)
    bound = bind(pos_vec, item_vec)
bundle(all bounds)

Set encoding also uses bind: bind(set_indicator, bundled_items) to tag a bundle as a set.

Source: T0-006-walkable, "The key insight is role-filler binding: we bind each value (filler) with its key (role) before bundling."

In the ThoughtAST encoder

ThoughtAST::Bind(left, right) maps directly to Primitives::bind:


Bind(left, right) → l_vec = encode(left)
                     r_vec = encode(right)
                     Primitives::bind(&l_vec, &r_vec)

This is the AST node for named scalar facts like "rsi is 0.73" — Bind(Atom("rsi"), Log(0.73)).

Source: T0-011-thought-encoder-rust, ThoughtEncoder::encode dispatch table: "Recursively encodes left → l_vec, encodes right → r_vec, then Primitives::bind(&l_vec, &r_vec)."

Relationship to the other generators

Bind is one of four generators of the WAT Vector Algebra:

GeneratorRoleRelationship to bind
atomGround truth — names concepts → vectorsProduces the vectors that bind composes
bindGroup action — composes relationshipsThis primitive
bundleCommutative monoid — superimposes relationshipsBundles the products of bind (role-filler pairs)
cosineMeasurement homomorphism — measures similarityThe only way to inspect what bind produced
Source: T0-002-primitives, "Two algebraic structures. Nothing here is derivable from the others."

Divergences and caveats

  • bind vs. unbind: The WAT spec describes the self-inverse property in terms of bind applied twice. Rust additionally exposes unbind as a named alias for clarity, but it is literally the same function.
  • f64 pipeline: bind_f64 exists for extraction/analysis work. It is identical math (element-wise multiply) but operates on floating-point vectors without thresholding.
  • No identity element: Unlike bundle (whose WAT-specified identity is the zero vector), bind has no identity element. Binding with a zero vector produces all zeros.
