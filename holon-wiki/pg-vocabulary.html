Vocabulary — the named atoms that give the system its words. EXPLAIN

TL;DR

Vocabulary — the named atoms that give the system its words.

What Vocabulary Is

Each vocabulary module is a pure function: data in, facts out, no state. From the GUIDE:

"Pure functions. Something in, facts out. No state. Each domain thinks about different things. Market vocab thinks about candles. Exit vocab thinks about candles and conditions. Risk vocab (future) thinks about portfolio state. The input is whatever the domain needs to form its judgment."
— GUIDE, lines 1244–1247
"Three domains. Each domain has scoped subfiles."
— GUIDE, line 1252

Every vocabulary module produces a Vec — a list of typed facts that are walked into vectors by the thought-encoder.

The Three Encoding Schemes

"The encoding scheme IS the bounding strategy."
— GUIDE, line 1376

The encoding scheme determines how a continuous numeric value becomes a bipolar vector. Each scheme maps to a ThoughtAST variant and a scalar-encoding algorithm.

SchemeThoughtAST VariantUse CaseBounding Strategy
Linear { scale }Linear { value, scale }Naturally bounded scalarsBounds are in the math: values 0→scale map to orthogonal vectors. Values beyond scale wrap.
LogLog { value }Unbounded positive scalarsLog₁₀ compresses naturally; equal ratios produce equal similarity drops. Non-positive → zero vector.
Circular { period }Circular { value, period }Periodic scalarsExplicit modulo wrap; endpoint equals start. Hour 23 ≈ hour 0 ≫ hour 12.
"Some facts are bounded. Some aren't. That's honest. The log doesn't bound — it compresses. The circular doesn't bound — it wraps. Only linear needs bounds, and linear's bounds come from the math."
— GUIDE, lines 1393–1395

For additional encoding detail (rotational vs positional, determinism, quantization), see scalar-encoding.

ThoughtAST — the language vocabulary speaks

Vocabulary modules emit ThoughtAST nodes. The seven variants:

VariantFieldsRole
Atom(String)nameA named symbol with no scalar value. Atoms are names — not facts.
Linear { value, scale }f64, f64A bounded scalar fact.
Log { value }f64An unbounded-positive scalar fact.
Circular { value, period }f64, f64A wrapping scalar fact.
Bind(left, right)Box, BoxTwo vectors bound together. Used for named scalars (e.g., RSI bound to its value).
Bundle(children)VecUnordered set — all children merged into one vector.
Sequential(items)VecPosition-sensitive sequence — each item permuted by its index, then bundled. [A, B] ≠ [B, A].

Key facts about the AST (from thought-encoder verification):

  • Atom nodes are not facts — collect_facts() skips them.
  • Bind, Linear, Log, Circular nodes are facts.
  • round_to() at vocab emission is load-bearing: the AST is used as a HashMap key in IncrementalBundle; quantized floats compare reliably via f64::to_bits().
  • The ThoughtEncoder evaluates the AST recursively with no caching; production caching is external (LRU in encoding::encode::encode()).

Design Principles

No zones, no categories — only scalars

"No zones. No categories. Only scalars. 'Overbought' is a human label on a continuous value — a magic number wearing a name. WHO decided 70 was the boundary? The vocabulary emits 'RSI is at 0.73.' The discriminant learns where the boundaries are."
— GUIDE, lines 1398–1405

Every relationship is a signed scalar

"Every relationship is a signed scalar. Not 'close is above SMA20' — the relative distance, with sign. [...] The sign IS the direction. The magnitude IS the distance. No 'above' atom. No 'below' atom. No boolean. Just the number."
— GUIDE, lines 1346–1356

Vocabulary owns encode AND decode

"The vocabulary owns the encode AND the decode — it put the value on the scalar, it can take it back. That's why scalar accumulators work."
— GUIDE, lines 1396–1397

Conditional emission

Not all facts fire every candle. Divergence atoms emit only when active (0–3 facts). Phase scalar summaries require ≥2 history records. This honesty — "sometimes there's nothing to say" — is by design.
