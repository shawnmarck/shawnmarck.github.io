Bundle — the lossy consensus operator EXPLAIN

TL;DR: Bundle superimposes multiple vectors into one by element-wise majority vote. It is the commutative monoid identity of the Vector Algebra: associative, commutative, and lossy. Every structured encoding in holon terminates with a bundle. It is how thoughts become single vectors.

Algebraic role#

The WAT spec defines the Vector Algebra as:

"A commutative monoid under bundle, with bind as the group action and cosine as the measurement homomorphism." — wat/core/primitives.wat via T0-002

Bundle is the monoid operation. The WAT declares its properties:

Property WAT claim
Associativity Yes (order-independent)
Commutativity Yes (input order irrelevant)
Identity Zero vector

These follow mechanically from majority vote over signed integers — the sum a + b + c is the same regardless of grouping or permutation, and the sign function applied afterward preserves those properties.

Implementation: element-wise majority vote#

Rust implements bundle in Primitives::bundle:

/// Bundle multiple vectors (element-wise majority vote).
/// Creates a superposition that is similar to ALL input vectors.
pub fn bundle(vectors: &[&Vector]) -> Vector {
    // sums each dimension into i32, then: >0 → +1, <0 → -1, ==0 → 0
}

The algorithm: sum each dimension across all input vectors, then apply a sign threshold:

Sum per dimension Output
> 0 +1
< 0 -1
== 0 0

This is not a plain sum — it is sign(Σ). The result stays in the bipolar set {-1, 0, +1}.

Source: T0-002, holon-rs/src/kernel/primitives.rs.

Weighted variant#

Primitives::weighted_bundle(vectors, weights) applies per-vector weight scaling before the same sign threshold. Same majority-vote semantics, but with unequal influence.

f64 variant#

bundle_f64 skips the sign threshold entirely — it returns the raw sum in f64 space. Used for the extraction and analysis pipeline:

"The i8 pipeline is for encoding and prediction. The f64 pipeline is for extraction and analysis." — T0-002

Lossiness — why it's called "lossy consensus"#

Bundle is not invertible. Given bundle(A, B) = C, there is no operation that recovers A and B from C alone. Information is destroyed by the sign threshold. This is by design: a bundled thought is a consensus — the dimensions where most inputs agree survive; where they conflict, the majority wins and the minority is erased.

This makes bundle fundamentally different from bind (which is exact and self-inverse). Bind creates structure; bundle destroys structure to produce summary.

Where bundle appears in the encoding pipeline#

Walkable map encoding#

Every Map-typed Walkable encodes as role-filler bindings, then bundles:

for each (key, value):
    bound = bind(role_vector(key), encode(value))
    push bound
bundle(all bounds)

The final bundle is the single vector that represents the entire map. Removing any binding degrades but does not destroy the vector — it still partially matches all remaining keys.

Source: T0-006, holon-rs/src/kernel/encoder.rs.

Walkable list and set encoding#

  • List: position-encode each element (bind(pos_vec, item_vec)), then bundle.
  • Set: bundle items directly, then bind with a "set_indicator" role vector to distinguish sets from random bundles.
  • Spread: position-encode each element, but each becomes an independent leaf binding (fan-out), not a single bundle.

Source: T0-006.

ThoughtAST bundle node#

The ThoughtAST::Bundle(children) variant maps encode() over all children, then calls Primitives::bundle. If empty, returns Vector::zeros(dims).

Source: T0-011, holon-lab-trading/src/encoding/thought_encoder.rs.

ThoughtAST sequential node#

Sequential(items) encodes each item, permutes it by its positional index (Primitives::permute(v, i)), then bundles all permuted vectors. The permutation breaks symmetry so [A, B] ≠ [B, A].

Source: T0-011.

IncrementalBundle — streaming optimization#

IncrementalBundle maintains a running sums: Vec<i32> buffer across candles (time steps). Instead of re-encoding and re-bundling every fact each tick, it diffs the new fact set against last_facts, subtracts removed contributions, adds new ones, and re-thresholds.

"Incremental bundling — maintains sums across candles. Optimization, not cognition. Can be reconstructed from one full encode." — T0-011

The threshold() method applies the same sign function as bundle(), producing a result bit-identical to a full bundle from scratch.

Load-bearing invariant: round_to at vocab emission quantizes floats so AST key equality (via f64::to_bits()) is reliable. Without it, floating-point drift causes false "changed" detections, degrading incremental savings to full recompute.

Source: T0-011, lines 212–313.

Identity divergence (WAT vs Rust)#

Specification Empty bundle behavior
WAT spec Returns zero vector (identity element)
Rust Primitives::bundle Panics on empty input

Rust does not implement the identity. ThoughtEncoder works around this by handling empty Bundle and Sequential nodes explicitly, returning Vector::zeros(dims) before ever calling Primitives::bundle.

Source: T0-002.

Confident bundle — with agreement margins#

bundle_with_confidence(vectors) returns both the bundled vector and per-dimension agreement margins — how strongly each dimension's sign is supported. Used when the system needs to distinguish strong consensus from weak majority.

Source: T0-002, extended operations table.

Relationship to other operations#

Operation Relationship to bundle
bind Bundle destroys information; bind preserves it (self-inverse).
cosine Cosine measures how much a bundled thought still resembles its constituents.
atom Atoms are the inputs that get bundled into composite thoughts.
negate Removes a component's influence from a superposition (approximate inverse of bundling).
weighted_bundle Bundle with unequal per-vector influence.
bundle_f64 Bundle without thresholding — raw accumulation for analysis.
bundle_with_confidence Bundle with per-dimension strength margins.
