EngramGate — gating observations by subspace alignment WHAT

Source: wat/engram-gate.wat (44 lines)

Guide reference: wat/GUIDE.md lines 1668–1688

Tier: 2 — Quality gating utility

What It IS

EngramGate is a shared utility for gating reckoner quality. After a recalibration period with good prediction accuracy, it snapshots the discriminant into a good-state subspace. The subspace accumulates knowledge of what good discriminants look like over time.

"Shared utility for gating reckoner quality. After recalibration with good accuracy, snapshot the discriminant into a good-state subspace. Used by both MarketObserver and Broker."
— engram-gate.wat, lines 1–4

It depends on: Outcome enum, Reckoner, and OnlineSubspace. The fields live on MarketObserver and Broker structs; the function lives in engram-gate.wat.

"the fields live on those structs, the function lives in engram-gate.wat."
— GUIDE.md, lines 1673–1674

Lines 17–44. Five parameters, returns updated EngramGateState.

Algorithm

The function is called after each resolved prediction. It operates in two branches:

Branch 1 — Recalibration boundary crossed (line 30):

When the reckoner's current recalib-count exceeds last-recalib-count in the gate state, a recalibration period has ended.

  1. Compute accuracy: wins / total (or 0.0 if total is 0) — lines 32–34.
  2. If accuracy ≥ threshold:
  • For each label in the reckoner, get the discriminant via (discriminant reckoner label).
  • If a discriminant exists (when-let), update the good-state subspace with it — lines 37–40.
  1. Reset counters to (engram-gate-state 0 0 current-recalib) — line 42.

Branch 2 — Same recalibration period (line 44):

No boundary crossed. Return gate-state unchanged. The wins/total will have been incremented by the caller before this call (the function tracks cumulative counts, but only evaluates at boundaries).

"Tracks wins/total since last recalibration. When recalib-count advances and accuracy exceeds the threshold, snapshots the discriminant into the good-state subspace."
— GUIDE.md, lines 1685–1687

Parameters Not Used in the Source

The recalib-interval parameter is declared in the function signature (line 20) but does not appear in the function body. The boundary detection uses recalib-count from the reckoner directly rather than computing from an interval. This may be intentional (the reckoner itself manages recalibration timing) or an unused parameter.

Data Flow


resolved prediction
  → caller increments gate-state.recalib-wins and recalib-total
  → caller calls check-engram-gate(...)
  → if boundary crossed AND accuracy ≥ threshold:
      → snapshot discriminant into good-state-subspace
      → reset counters
  → return updated gate-state

The good-state subspace accumulates discriminant vectors over time. A subspace that has seen many good discriminants provides a reference for evaluating future discriminant quality — hence "gating" by subspace alignment.
