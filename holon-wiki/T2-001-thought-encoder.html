ThoughtEncoder — the AST walker that produces vectors from thought expressions

Source: wat/thought-encoder.wat (198 lines)

Guide reference: wat/GUIDE.md lines 1416–1597

Tier: 2 — Encoding infrastructure

What It IS

The ThoughtEncoder evaluates abstract syntax trees produced by the vocabulary modules. The vocabulary answers WHAT to think (trees of named observations); the ThoughtEncoder answers HOW to think efficiently (walk those trees, reuse cached sub-results, produce a high-dimensional vector).

"The vocabulary produces ASTs — WHAT to think. The ThoughtEncoder evaluates them — HOW to think efficiently. Walks the AST bottom-up, checking its memory at every node. Minimum computation."
— thought-encoder.wat, lines 1–5

It depends on VectorManager (for atom allocation) and ScalarEncoder (from holon-rs, for encoding Linear/Log/Circular nodes). It lives on ctx — the immutable world created at startup.

"Lives on ctx — the immutable world created at startup. Passed to posts via ctx on every on-candle call."
— GUIDE.md, line 1434–1435

Two Kinds of Memory

The encoder maintains two separate memory stores with fundamentally different semantics:

Atoms — a dictionary

"Finite. Known at startup. Pre-computed. Never evicted."
— thought-encoder.wat, line 8

Atoms are a Map. The set is closed. Once registered, always available. Atom lookups always succeed.

Compositions — a cache

"Infinite. Optimistic. Use it if we have it. Compute if we don't."
— thought-encoder.wat, line 9

Compositions are a Map with capacity 4096. The set is open and evictable. Cache misses flow upward as return values — the enterprise collects them and inserts between candles.

"The encode function NEVER writes to the cache. Misses are returned as values — the enterprise collects them and inserts between candles. Values up, not queues down."
— thought-encoder.wat, lines 11–13

This is the one seam in the otherwise-immutable ctx: the cache mutates on miss, but only during the sequential phase between candles, never during parallel encoding.

The AST — thought-ast enum


(enum thought-ast
  (Atom name)                           ; dictionary lookup — always succeeds
  (Linear name value scale)             ; bind(atom, encode-linear)
  (Log name value)                      ; bind(atom, encode-log)
  (Circular name value period)          ; bind(atom, encode-circular)
  (Bind left right)                     ; composition of two sub-trees
  (Bundle children))                    ; superposition of sub-trees

Six variants, lines 21–27. Each variant maps to a specific evaluation strategy:

VariantStrategy
AtomDirect dictionary lookup. Never misses.
LinearLook up atom, then bind(atom-vec, encode-linear(value, scale)).
LogLook up atom, then bind(atom-vec, encode-log(value)).
CircularLook up atom, then bind(atom-vec, encode-circular(value, period)).
BindRecursively encode both sub-trees, then bind(left-vec, right-vec).
BundleRecursively encode all children, then apply bundle (superposition).

(struct thought-encoder

Four fields, lines 41–45. Note the comment on compositions: "LRU moved to EncoderService" — the wat file's local cache is a simple map, while the GUIDE declares LruCache. The encoder-service.wat holds the true LRU; the ThoughtEncoder's composition map is the per-candle working set.

Core Function: encode


(define (encode [encoder : ThoughtEncoder] [ast : ThoughtAST])
  : (Vector, Vec<(ThoughtAST, Vector)>)
  ...)

Lines 64–104. Returns a tuple: the computed vector plus a list of cache misses.

Algorithm:

  1. Cache check first (line 68): when-let on compositions map. Hit → return vector with empty misses.
  2. Evaluate the AST node via pattern match (lines 73–101):
  • Atom: direct map lookup from atoms dictionary.
  • Linear/Log/Circular: recursive encode of the Atom variant, then bind with scalar encoding.
  • Bind: recursive encode of both children, then bind their vectors. Misses are appended.
  • Bundle: map over all children, encode each, bundle the vectors, concatenate all misses.
  1. Record the miss (line 103–104): cons (ast, result) onto the misses list and return.

The function is recursive and caches at every node. The AST structure IS the cache key — structural equality determines cache identity.

"Same structure, same vector."
— GUIDE.md, line 1475

round-to — Cache Key Quantization


(define (round-to [v : f64] [digits : u32]) : f64
  (let ((factor (expt 10 digits)))
    (/ (round (* v factor)) factor)))

Lines 34–37. Used at vocabulary emission time (not at encode time) to quantize float values. The ThoughtAST carries the rounded value.

"The cache key IS the exact AST. The rounding happens at emission, not evaluation."
— lines 30–32

This is load-bearing: without quantization, floats would compare unreliably and incremental bundle optimization would degrade to full recompute.

IncrementalBundle — Optimization, Not Cognition


(struct incremental-bundle
  [sums : Vec]                     ; running element-wise sums
  [last-facts : Map] ; previous candle's facts
  [dims : usize]                        ; dimensions
  [initialized : bool])                 ; whether we've done at least one full encode

Lines 119–123. Maintains running sums across candles to avoid re-encoding unchanged facts.

The Algebra

"bundle = threshold(Σ vectors). If fact k changes from old to new, sums_new = sums - old + new. threshold(sums_new) == bundle(all current facts). Proven bit-identical. Integer addition is commutative and associative."
— lines 110–113

incremental-encode (lines 141–171)

Two paths:

  • First candle (initialized = false): calls full-encode — encodes all facts, populates sums and last-facts.
  • Subsequent candles: diffs against last-facts. Removed facts are subtracted from sums. Unchanged facts require zero work. Changed/added facts are encoded via the ThoughtEncoder and added to sums.

full-encode (lines 174–187)

Encodes every fact from scratch, accumulates sums, stores each fact's vector in last-facts, sets initialized = true, applies threshold.

threshold (lines 190–198)


(define (threshold [sums : Vec] [dims : usize]) : Vector
  ...)

Sign function: sums > 0 → 1, sums < 0 → -1, sums == 0 → 0. Produces the bundled binary vector.

"Can be reconstructed from one full encode."
— line 108

GUIDE vs WAT Divergence Note

The GUIDE declares compositions as LruCache (line 1440). The wat file declares it as Map with capacity 4096 (line 43), and annotates: "LRU moved to EncoderService". The EncoderService (encoder-service.wat) holds the actual LRU cache. The ThoughtEncoder's composition map serves as the per-candle working set. Per CLAUDE.md: "When layers diverge, the guide is right."
