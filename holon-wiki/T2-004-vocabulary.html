T2-004 — Vocabulary WHAT

The complete atom set and how vocabulary modules are organized.#

Overview#

From the GUIDE (lines 1244–1247):

"Pure functions. Something in, facts out. No state. Each domain thinks about different things. Market vocab thinks about candles. Exit vocab thinks about candles and conditions. Risk vocab (future) thinks about portfolio state. The input is whatever the domain needs to form its judgment."

Four domains, scoped subfiles. The GUIDE declares (line 1252):

"Three domains. Each domain has scoped subfiles."

Divergence note: GUIDE lists three, live Rust has four#

The GUIDE describes three domains (shared, market, exit). The live Rust codebase has four: shared, market, exit, and broker. The broker domain (Proposal 044, Phase 3) provides portfolio-level vocabulary for brokers — it has no wat spec file and exists only in Rust.

Divergence note: GUIDE exit domain vs Rust reality#

The GUIDE (lines 1310–1319) describes the exit domain with three wat modules (volatility.wat, structure.wat, timing.wat) and a ExitLens with variants :volatility, :structure, :timing, :generalist. These wat files exist and define atoms as documented below.

However, the live Rust position_lens_facts in src/domain/lens.rs does not call any of those three exit wat modules. Instead it calls:

  • encode_exit_regime_facts — regime atoms (kama-er, choppiness, etc.) — Rust-only, no corresponding wat file
  • encode_exit_time_facts — hour + day-of-week only — Rust-only, no corresponding wat file
  • encode_phase_current_facts / phase_series_thought / phase_scalar_facts — phase biography — Rust-only, no corresponding wat file

The exit wat files (volatility.wat, structure.wat, timing.wat) exist in the spec but are not wired into the running system. The Rust exit vocab has diverged from the GUIDE's exit vocabulary design. This is a significant spec-vs-implementation divergence.

Encoding Schemes#

From the GUIDE (lines 1376–1395):

"The encoding scheme IS the bounding strategy."

Three schemes:

Scheme Use case Examples
Linear { scale } Naturally bounded scalars. Bounds are in the math. Bollinger position [-1,1], RSI [0,1]
Log Unbounded positive scalars. Log compresses naturally. ATR ratio, volume ratio, band widths
Circular { period } Periodic scalars. The value wraps. minute (60), hour (24), day-of-week (7)

From the GUIDE (lines 1393–1395):

"Some facts are bounded. Some aren't. That's honest. The log doesn't bound — it compresses. The circular doesn't bound — it wraps. Only linear needs bounds, and linear's bounds come from the math."

Key GUIDE Principles#

From GUIDE lines 1396–1405:

"The vocabulary owns the encode AND the decode — it put the value on the scalar, it can take it back. That's why scalar accumulators work."

"No zones. No categories. Only scalars. 'Overbought' is a human label on a continuous value — a magic number wearing a name. WHO decided 70 was the boundary? The vocabulary emits 'RSI is at 0.73.' The discriminant learns where the boundaries are."

From GUIDE lines 1346–1356:

"Every relationship is a signed scalar. Not 'close is above SMA20' — the relative distance, with sign. [...] The sign IS the direction. The magnitude IS the distance. No 'above' atom. No 'below' atom. No boolean. Just the number."
