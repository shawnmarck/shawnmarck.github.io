Four-Step Loop — observe, predict, execute, learn EXPLAIN

TL;DR

Every candle, the system:

  1. Observe — Encode the market into thought vectors via vocabulary lenses.
  2. Predict — <a class="wikilink" href="pg-market-observer.html">MarketObserver</a> classifies direction (Up/Down); <a class="wikilink" href="pg-exit-observer.html">PositionObserver</a> estimates trail/stop distances.
  3. Execute — The <a class="wikilink" href="pg-broker.html">Broker</a> creates <a class="wikilink" href="pg-paper-trade.html">paper trades</a> that track live prices against predictions. Papers resolve as Grace (trail crossed, prediction confirmed) or Violence (stop hit, prediction wrong).
  4. Learn — Resolutions propagate back: observers update their reckoners and noise subspaces; the broker updates scalar accumulators, dollar-P&L EMAs, and its expected-value gate.

Values flow up — brokers return propagation facts, and the Post routes them to observers via mailbox channels. No shared mutation during parallel phases.

The Holon trading system runs a continuous four-step loop each candle: observe (encode market state), predict (direction + distances), execute (paper trades), and learn (propagate outcomes back). This page traces the loop end-to-end across <a class="wikilink" href="pg-market-observer.html">MarketObserver</a>, <a class="wikilink" href="pg-exit-observer.html">PositionObserver</a>, <a class="wikilink" href="pg-broker.html">Broker</a>, and <a class="wikilink" href="pg-paper-trade.html">PaperEntry</a>.

Step 1 — Observe

Each observer encodes the current candle into a high-dimensional thought vector using its vocabulary lens.

  • <a class="wikilink" href="pg-market-observer.html">MarketObserver</a> uses a MarketLens (e.g., DowTrend) to produce a raw_thought. A <a class="wikilink" href="T0-010-subspace.html">noise subspace</a> (8 principal components) learns the background distribution; the residual — anomalous_component() — is the signal. "The residual IS the signal." <a class="wikilink" href="T0-013-market-observer-rust.html">§T0-013</a>

  • <a class="wikilink" href="pg-exit-observer.html">PositionObserver</a> uses a PositionLens (Core with 5 trade atoms or Full with 13) to produce a position_thought. Also has a noise subspace, despite its own docstring claiming otherwise. <a class="wikilink" href="T0-014-exit-observer-rust.html">§T0-014</a>

Key invariant (Proposal 026): Position observer queries and learns on position_thought only — never composed thought. <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>

Step 2 — Predict

Direction (MarketObserver)

observe() calls reckoner.predict() on the noise-stripped vector. Returns a discrete classification (Up/Down) with conviction and an accuracy edge:

"prediction and learning are aligned — both operate on the anomaly, not the raw thought." <a class="wikilink" href="T0-013-market-observer-rust.html">§T0-013</a>

Distances (PositionObserver)

reckoner_distances(position_thought) queries two independent continuous <a class="wikilink" href="pg-reckoner.html">reckoners</a> — one for trailing-stop distance, one for safety-stop distance. Returns None if either has zero experience. <a class="wikilink" href="T0-014-exit-observer-rust.html">§T0-014</a>

The Distance Cascade (Broker)

The <a class="wikilink" href="pg-broker.html">Broker</a> owns a three-tier fallback for when the position observer cannot yet predict:

Tier Source Condition
1 Position observer reckoner Some(dists)
2 Broker-owned <a class="wikilink" href="T0-007-accumulator.html">scalar accumulators</a> Trail/stop accum have observations
3 Constructor-provided default_distances Cold-start fallback

"The broker owns the full cascade because it owns the scalar accumulators." <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>

Step 3 — Execute (Paper Trades)

Paper Creation

Every candle, for every broker, register_paper() creates a new <a class="wikilink" href="pg-paper-trade.html">PaperEntry</a> with:

  • The composed thought (market + position)
  • The raw market thought and position thought separately
  • The predicted direction and the (cascaded) trail/stop distances <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>

Paper Lifecycle

Each paper tracks live prices against its two triggers:

Trigger Name Meaning
Trailing stop crossed Grace Prediction confirmed — price moved in the predicted direction beyond the trail threshold
Fixed stop hit Violence Prediction wrong — price moved against beyond the stop threshold

Stop is checked before Grace on each tick. If both would fire simultaneously, Violence wins. <a class="wikilink" href="T0-015-paper-entry-rust.html">§T0-015</a>

flowchart TD
    created["Created"] -->|"Stop fires"| violence["Violence\n(resolved, never signaled)"]
    created -->|"Trail crosses"| runner["signaled = true\nbecomes a Runner"]
    runner -->|"Trailing stop fires"| grace["resolved = true\n(Grace)"]
    runner -->|"Price continues favorably"| running["keeps running"]
    created -->|"Neither fires"| open["stays open"]


Runner History


When a paper signals Grace (becomes a runner), the broker creates a RunnerHistory and accumulates (thought, distances, price) per candle. At runner resolution, compute_position_batch() uses a suffix-extremum pass to produce deferred training data — sparse samples where the optimal trail distance changed by >10%. <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>

Broker Propagation


propagate() is called by the <a class="wikilink" href="pg-four-step-loop.html">Post</a> with each resolution. It does three things: <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>


    

  1. Track record — Updates cumulative_grace, cumulative_violence, trade_count.
    2. 

  2. Scalar accumulators learn — Feeds optimal (hindsight) distances to trail/stop accumulators, weighted by outcome.
    3. 

  3. Dollar P&L + EMA — Computes net P&L against a $10,000 reference capital including swap fees. Updates avg_grace_net and avg_violence_net with EMA alpha = 0.038 (~50-paper half-life). Recomputes expected_value = grace_rate × avg_grace_net + (1 - grace_rate) × avg_violence_net.
    4. 



Returns PropagationFacts — the Post routes these to the correct observers. "Values up — the broker does not talk to observers directly." <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>


MarketObserver Learning


resolve() is called with a labeled direction and weight: <a class="wikilink" href="T0-013-market-observer-rust.html">§T0-013</a>


    

  1. Map direction to label index (Up→0, Down→1).
    2. 

  2. Self-grade: compare last_prediction against actual direction.
    3. 

  3. reckoner.observe(thought, label, weight) — feed labeled experience. Per Proposal 024, the thought fed is the anomaly, not the composed thought.
    4. 

  4. Update the conviction→accuracy curve.
    5. 

  5. Engram gate check: accumulate wins/total, evaluate whether the observer has proven itself using good_state_subspace.
    6. 





"Its own teacher. The market is the judge." <a class="wikilink" href="T0-013-market-observer-rust.html">§T0-013</a>




PositionObserver Learning


observe_distances(position_thought, optimal_distances) feeds hindsight-optimal distances: <a class="wikilink" href="T0-014-exit-observer-rust.html">§T0-014</a>


    

  1. Record pre-learning prediction error (diagnostic, read-only).
    2. 

  2. trail_reckoner.observe_scalar(position_thought, optimal.trail, 1.0) and stop_reckoner.observe_scalar(position_thought, optimal.stop, 1.0).
    3. 

  3. Weight is always 1.0 — each observation counts once.
    4. 



The "optimal" distances come from simulation sweeps against actual price histories. <a class="wikilink" href="T0-014-exit-observer-rust.html">§T0-014</a>


The Gate


gate_open() controls whether a broker can propose funded trades: <a class="wikilink" href="T0-018-broker-rust.html">§T0-018</a>


    

  • Cold start (<50 of either outcome): always open — not enough data to judge.
    • 

  • Warm (≥50 of both): open only when expected_value > 0. Brokers with negative EV are suppressed.
    •
