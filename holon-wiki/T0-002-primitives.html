T0-002: Primitives — atom, bind, bundle, and the algebraic structure of holon-rs HOW

What It Is#

The holon-rs kernel defines two structures. The WAT spec (wat/core/primitives.wat) declares them as two algebraic structures:

"Two algebraic structures. Nothing here is derivable from the others."

  1. Vector Algebra — four generators: atom, bind, bundle, cosine
  2. Reckoner Coalgebra — opaque state machine with co-generators for learning

The Rust file (holon-rs/src/kernel/primitives.rs) implements the vector operations portion of the Vector Algebra. It does not implement atom or any part of the Reckoner Coalgebra.

Vector Algebra (WAT spec — canonical)#

The WAT defines the Vector Algebra as:

"A commutative monoid under bundle, with bind as the group action and cosine as the measurement homomorphism."

atom#

"(atom name) → Vector" "Name a concept → deterministic bipolar vector." "Same seed + same name = same vector everywhere."

atom maps a name to a deterministic bipolar vector. It is the ground truth for concept identity. The Rust primitives.rs does not implement atom. It is a declared primitive with no corresponding method in this file.

bind#

"(bind role filler) → Vector" "Compose two concepts into a relationship." "Self-inverse: (bind (bind A B) A) = B"

The Rust implementation agrees:

/// Bind two vectors (element-wise multiplication for bipolar).
///
/// Creates an association between two concepts. The result is
/// dissimilar to both inputs, but `unbind(bind(A, B), A) ≈ B`.
///
/// For bipolar vectors, this is element-wise multiplication.
pub fn bind(a: &Vector, b: &Vector) -> Vector {
    let mut out = Vector::zeros(a.dimensions());
    Self::bind_into(a, b, &mut out);
    out
}

The implementation is literally *o = x * y for each dimension — element-wise multiplication of bipolar ({-1, 0, +1}) values. The self-inverse property is tested:

#[test]
fn test_bind_self_inverse() {
    // ...
    let ab = Primitives::bind(&a, &b);
    let b_recovered = Primitives::bind(&ab, &a);
    assert_eq!(b, b_recovered);
}

Rust also exposes unbind as an explicit alias:

/// Unbind a key from a bound vector.
/// For bipolar vectors, unbind is identical to bind (self-inverse property):
/// `unbind(bind(A, B), A) = B`
pub fn unbind(bound: &Vector, key: &Vector) -> Vector {
    Self::bind(bound, key)
}

bundle#

"(bundle fact1 fact2 ...) → Vector" "Superimpose multiple relationships into one thought." "Associative, commutative, identity = zero vector."

The Rust implementation uses element-wise majority vote (sum then sign-threshold):

/// Bundle multiple vectors (element-wise majority vote).
/// Creates a superposition that is similar to ALL input vectors.
pub fn bundle(vectors: &[&Vector]) -> Vector {
    // sums each dimension, then: >0 → +1, <0 → -1, ==0 → 0
}

Divergence noted: The WAT says "identity = zero vector," implying bundling nothing yields zero. The Rust bundle panics on an empty input:

if vectors.is_empty() {
    panic!("Cannot bundle empty vector list");
}

Associativity and commutativity hold in practice because majority vote over signed integers is order-independent.

cosine#

"(cosine thought direction) → Float ; [-1.0, +1.0]" "Measure similarity between two thoughts." "The only measurement. One number."

Rust's cosine similarity is not in primitives.rs. It lives in crate::similarity::Similarity::cosine. The Rust file calls it extensively from tests and from helper methods (cleanup, invert, topk_similar, etc.) but does not define it here.

Reckoner Coalgebra (WAT spec — canonical)#

The WAT declares a second structure: the Reckoner Coalgebra. The spec says:

"Opaque state. The co-generators produce all observable behavior. Removing any one makes the external characterization incomplete."

Generators declared:

Operation Signature Role
reckoner (name dims refit-interval) → Reckoner Create learning primitive
register (reckoner name) → Label Register label, get symbol handle
observe (reckoner thought label weight) Accumulate observation (state transition)
predict (reckoner thought) → Prediction Query (observation function)
decay (reckoner rate) Fade old observations (aging function)
resolve (reckoner conviction correct) Record prediction outcome
recalib-count (reckoner) → Integer Monotonically increasing rebuild count
discriminant (reckoner label) → Vector \| None Learned separation between prototypes
curve (reckoner) → (amplitude, exponent) Conviction-accuracy curve fit

The conviction-accuracy curve formula is given explicitly:

"accuracy = (1/N) + a × exp(b × conviction), N = number of labels." "At zero conviction, accuracy = 1/N (random chance among N labels)." "The curve IS the proof. Monotonically increasing = real signal."

None of the Reckoner Coalgebra is implemented in primitives.rs. This file is purely vector operations. The reckoner lives elsewhere in the kernel.

Rust-Only Operations (not in WAT spec)#

The Rust primitives.rs defines many operations absent from the WAT spec. These are the implementation's extended algebra:

negate (three methods)#

pub enum NegateMethod { Subtract, Zero, Invert }
  • Subtract (default): clamp superposition - component to [-1, +1]
  • Zero: zero out dimensions where component is non-zero
  • Invert: invert component and add (bundle with negated component)

The WAT does not define negate. There is also negate_f64 using orthogonal projection (result = a - (dot(a,b)/dot(b,b)) * b).

permute#

/// Circular shift (permutation) of vector dimensions.
/// Used for encoding sequential position.
pub fn permute(vec: &Vector, k: i32) -> Vector

Element-wise circular shift. The module doc lists permute as a core VSA primitive, but the WAT spec does not include it in its four generators.

weighted_bundle#

pub fn weighted_bundle(vectors: &[&Vector], weights: &[f64]) -> Vector

Same majority-vote semantics but with per-vector weight scaling before thresholding.

f64 Pipeline#

Rust maintains a parallel pipeline in f64 space for "scalar extraction from accumulated prototypes":

"The i8 pipeline is for encoding and prediction. The f64 pipeline is for extraction and analysis."

i8 method f64 equivalent Difference
bind bind_f64 Identical (element-wise multiply)
bundle bundle_f64 No thresholding — raw sum
negate (Subtract) negate_f64 Orthogonal projection instead of clamped subtraction
— cosine_f64 Standard dot-product cosine

Key Properties#

Bind is self-inverse (exact, not approximate)#

Both the WAT spec and Rust tests confirm exact recovery for bipolar vectors:

  • WAT: "(bind (bind A B) A) = B"
  • Rust: assert_eq!(b, b_recovered) in test_bind_self_inverse

This holds because bipolar element-wise multiplication is its own inverse: (a*b)*a = a²*b = b when a ∈ {-1, +1}.

Bundle is majority vote (not sum)#

Bundle sums dimension values into i32, then thresholds: positive → +1, negative → -1, zero → 0. This is not a plain sum; it is a sign function applied to the sum.

Bundle identity divergence#

  • WAT: "identity = zero vector"
  • Rust: panics on empty input

The Rust implementation does not provide a bundling identity. An empty bundle is undefined behavior, not the zero vector.

Cosine is the only measurement#

  • WAT: "The only measurement. One number."
  • Rust: cosine is the sole similarity metric used throughout, even though it is defined in a separate module.

Enums (Rust-only)#

pub enum AttendMode   { Hard, Soft, Amplify }
pub enum GateMode     { Positive, Negative, NonZero }
pub enum SegmentMethod { Diff, Prototype }
pub enum NegateMethod { Subtract, Zero, Invert }

None of these appear in the WAT spec.
