Conviction — the strength of a prediction EXPLAIN

TL;DR#

Conviction is the absolute value of the winning cosine similarity in a Reckoner's discrete prediction. It measures how strongly the reckoner leans toward one classification direction, on a scale of 0.0 (no preference) to 1.0 (maximum alignment). It is the primary input to the reckoner's self-evaluation mechanism — the conviction–accuracy curve — which validates whether the reckoner's confidence is well-calibrated.

Composite wiki page. Every claim below is traced to verified source documents.

Definition#

In discrete mode, a Reckoner computes the cosine similarity between the input thought and each label's learned discriminant direction. The label with the highest raw cosine becomes the predicted direction. The conviction is the absolute value of that cosine:

conviction = |best_raw_cosine|

The Prediction struct carries conviction as a first-class field:

pub struct Prediction {
    pub scores: Vec<LabelScore>,      // cosine per label, ordered descending
    pub direction: Option<Label>,     // label with highest raw cosine
    pub conviction: f64,              // |max cosine|
    pub raw_cos: f64,                 // raw signed cosine of winning direction
}

— T0-008 §3.2

Conviction is always non-negative [0.0, 1.0]. A conviction of 0.0 means the thought is orthogonal to all discriminant directions (the reckoner has no opinion). A conviction of 1.0 means perfect alignment with the winning discriminant.

Computation#

The prediction path in Reckoner::predict():

  1. For each label, compute cosine similarity between the (mean-stripped) thought and that label's discriminant vector.
  2. Sort all scores by raw cosine descending.
  3. The winning label = highest raw cosine.
  4. conviction = |winning_raw_cosine|.

An important subtlety: scores are sorted by raw cosine (not absolute), but conviction is the absolute value of the winner. This means a strongly negative cosine also produces high conviction — the reckoner is equally sure it is not that direction. As T0-008 notes:

"Scores are sorted by raw cosine descending … but conviction is set to best.cosine.abs(). This means the winning label is the one with the highest raw cosine, and conviction measures magnitude." — T0-008 §3.2

The conviction–accuracy curve#

Conviction's deeper purpose is self-evaluation. Through the resolve(conviction, correct) method, the reckoner records a sliding window of (conviction, was_it_right?) pairs. Once enough data accumulates (≥500 resolved predictions), curve() fits the empirical relationship:

accuracy = (1/N) + a × exp(b × conviction)

where N is the number of labels and 1/N is random-chance baseline. The WAT spec states:

"At zero conviction, accuracy = 1/N (random chance among N labels). The curve IS the proof. Monotonically increasing = real signal." — primitives.wat, quoted in T0-008 §3.6

A positive b coefficient means higher conviction genuinely predicts higher accuracy — the reckoner knows what it knows. The fitted curve enables accuracy_at(conviction), letting downstream systems ask: "If this reckoner says conviction = 0.7, how often is it actually right?"

This is the curve-fitting input point: resolve() is the only way external code feeds conviction data back into the reckoner for self-calibration.

"By recording (conviction, correct) pairs and fitting accuracy = 1/N + a·exp(b·conviction), the system gets a self-calibration measure." — T0-008 §6.7

Relationship to cosine similarity#

Conviction is a direct application of cosine similarity — the singular measurement primitive in the holon system. The WAT spec declares:

"The only measurement. One number." — primitives.wat, quoted in T0-003

Cosine returns a value in [-1, 1] measuring the angle between two vectors, independent of magnitude. Conviction folds the sign away to produce a scalar confidence measure: alignment with the discriminant direction, regardless of polarity.

The cosine similarity in this context operates on the f64 pipeline (cosine_f64), since discriminants are normalized f64 vectors, not the i8 bipolar Vector type used elsewhere.

Role in the system#

Conviction appears in three contexts:

  1. Reckoner prediction output — as Prediction.conviction, the immediate confidence signal accompanying every discrete classification.
  2. Self-evaluation input — via resolve(conviction, correct), feeding the sliding window that trains the conviction–accuracy curve.
  3. Calibrated downstream decisions — via accuracy_at(conviction), which lets application code use the reckoner's self-assessed reliability rather than raw cosine.

The primers frame the Reckoner (and thus conviction) within the memory layer:

"Where OnlineSubspace learns what normal looks like (anomaly detection), Reckoner learns what separates classes or predicts values." — Primer 003, quoted in T-1-000

Conviction is how that separation expresses its confidence.
