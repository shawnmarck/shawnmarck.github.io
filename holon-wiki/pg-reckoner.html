Reckoner — the geometric learning primitive EXPLAIN

One primitive. Two readout modes. Same accumulation. Same decay. Same recalibration.

Construction


Reckoner::new(name, dims, recalib_interval, config) -> Reckoner
  • dims: Must match the dimensionality of all input Vector objects.
  • recalib_interval: Every N observe() calls, discriminants are rebuilt automatically (discrete only).
  • config: ReckConfig::Discrete(label_names) or ReckConfig::Continuous { default_value, buckets }.

Mode is immutable for the lifetime of the instance — chosen at construction, never switched.

Discrete initialization

Creates one Accumulator per label (all zeroed), discriminants = vec![None; N], and initializes curve-fitting state (resolved_cap = 5000, min_resolved = 500, n_bins = 20).

Continuous initialization

Creates K ScalarBucket instances, each with an Accumulator and center: 0.0. Range starts undiscovered. The spec is explicit:

"Range is discovered from observations. Decay + rebalance = breathing range. No magic min/max. The data teaches the range. K is the only parameter." — reckoner.rs

Discrete Mode

observe(vec, label, weight) — State transition

  1. Bounds-check label index; silently returns if out of range.
  2. Increment update counter.
  3. If updates % recalib_interval == 0, run recalibrate().
  4. Call accumulators[idx].add_weighted(vec, weight) — adds the vector weighted into the label's Accumulator.

The Accumulator is frequency-preserving: unlike bundling (which is idempotent), accumulation preserves the fact that a pattern seen 100 times contributes 100× more than one seen once (accumulator.rs).

predict(vec) → Prediction

  1. If all discriminants are None, return default (no knowledge yet).
  2. For each label: compute cosine similarity between the (optionally mean-stripped) input and the label's discriminant vector.

pub struct Prediction {
    pub scores: Vec,      // cosine per label, sorted descending
    pub direction: Option

recalibrate() — Automatic discriminant rebuild

Triggered every recalib_interval observations. Extracts normalized prototypes from all Accumulators, then computes per-label discriminants:

"discriminant(label) = prototype(label) – prototype(other)." — primitives.wat

For binary classification, this simplifies to prototype(A) – prototype(B), the classic Fisher-like linear discriminant. Discriminants are normalized to unit length.

decay(factor) — Aging

Multiplicative on sums: sums *= factor on every Accumulator. The observation count is never decremented — it tracks history, not current weight.

"Useful for time-weighted baselines where recent patterns should have more influence." — accumulator.rs

Curve fitting — Self-evaluation

The curve answers: "When this reckoner says it's confident, is it actually right?"

  1. Record (conviction, correct) pairs via resolve(conviction, correct). Sliding window of 5000 entries.
  2. curve() bins resolved entries, fits accuracy = 1/N + a × exp(b × conviction) via linear regression on ln(acc – 1/N) vs conviction.
"At zero conviction, accuracy = 1/N (random chance among N labels). The curve IS the proof. Monotonically increasing = real signal." — primitives.wat

A positive b means higher conviction correlates with higher accuracy — the reckoner knows what it knows.

Continuous Mode

observe_scalar(vec, value, weight) — State transition

  1. Implicit decay: every observation decays all buckets by 0.999. "Every observation is a tick of time" (reckoner.rs).
  2. First observation: seeds a tiny range (±0.001), distributes K bucket centers uniformly.
  3. Range expansion: out-of-range values expand the range and trigger rebalance_continuous().
  4. Bucket assignment: idx = floor((value – range_min) / width), add weighted vector to bucket's Accumulator.
  5. Periodic rebalance: every 100 observations, contract range to where alive weight lives.

rebalance_continuous() — The breathing mechanism

  1. Contract range to alive bucket centers ± 10% margin.
  2. Allocate K fresh Accumulators with new centers.
  3. Pour old data into nearest new buckets via merge().

This is how the continuous reckoner adapts its range to data without hardcoded bounds.

query(vec) → f64

  1. Compute cosine between the input and each bucket's raw sums.
  2. Keep only positive-cosine buckets.
  3. Soft-weight top-3: result = Σ(cos_i × center_i) / Σ(cos_i).
"Interpolate, don't argmax." — reckoner.rs
"The prototype direction = 'thoughts that produced values in this range looked like THIS.' Cosine measures match quality regardless of accumulated mass. The center carries the value." — reckoner.rs

Key Design Properties

Accumulation, not storage

The Reckoner never stores individual observations. It uses Accumulators that maintain running weighted sums. The more you observe a pattern, the stronger its direction — but you never pay O(n) for n observations at query time.

Cosine as the sole measurement

All similarity computation uses Cosine Similarity. The WAT specification is explicit:

"(cosine thought direction) → Float ; [-1.0, +1.0]. The only measurement. One number." — primitives.wat

Frequency preservation vs. bundling

bundle([a, a, a, a, a]) = a (idempotent). accumulator.add(a) × 5 preserves that a was seen 5 times (sum vector has 5× magnitude). This distinction is critical: common patterns dominate the baseline, so rare anomalies have low similarity (accumulator.rs).

Decay is multiplicative on sums, not on count

decay(factor) multiplies sums by factor. Count stays — it tracks total observations ever, not current weight. The sums decay to near-zero but the count is history.

Continuous range is data-driven

No hardcoded value range. The first observation seeds a tiny range. Out-of-range observations expand it. Every 100 observations, the range contracts to where alive weight lives. The data teaches the range.

Recalibration is periodic, not event-driven

In discrete mode, recalibrate() fires every recalib_interval observations (modulo check). It is a batch operation: extract all prototypes, compute discriminants, write them back. No incremental update.

Computational complexity

  • Discrete predict: O(L × D) — L labels, D dimensions.
  • Continuous query: O(K × D) — K buckets, D dimensions.
  • Both are constant in observation count — the Accumulator compresses everything into a single D-dimensional sum vector.

Relationship to other primitives

The Reckoner is the Algebra 2 coalgebra in the WAT specification (primer 005):

"Five core operations, four accessors. The accumulators are private — the coalgebra is defined by its interface, not its representation."

Its learning mechanism is built entirely from Accumulator (frequency-preserving weighted sums) and Cosine Similarity (the sole measurement). It operates on thought vectors produced by the encoding stack described in the primers — atomize → bind → bundle → thought vector.
