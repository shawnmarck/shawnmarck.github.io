Curve — the conviction-to-accuracy mapping

TL;DR: The curve is a self-calibration mechanism inside every discrete Reckoner. It records (conviction, correct) pairs, fits the exponential model accuracy = 1/N + a·exp(b·conviction), and answers the question: "When this reckoner says it's confident, is it actually right?" A positive b means the reckoner knows what it knows.

What it IS

The curve maps conviction (the absolute cosine similarity of the winning discriminant) to empirical accuracy (fraction of correct predictions). It is an introspection tool — the reckoner's proof that its confidence is calibrated.

The WAT spec declares:

"accuracy = (1/N) + a × exp(b × conviction), N = number of labels. At zero conviction, accuracy = 1/N (random chance among N labels). The curve IS the proof. Monotonically increasing = real signal."
— primitives.wat

Why it exists

How data is collected

resolve(conviction, correct) — recording outcomes


pub fn resolve(&mut self, conviction: f64, correct: bool)

Discrete mode only. Pushes (conviction, correct) onto a bounded sliding window (resolved_cap = 5000 by default). When the window overflows, the oldest entry is evicted. This is a FIFO queue of bounded size.

Source: reckoner.rs — "Pushes (conviction, correct) onto the resolved vec. If resolved.len() > resolved_cap, the oldest entry is removed." (T0-008, §3.5)

Where resolve is called

In the Broker domain, resolutions flow through propagate() → PropagationFacts → the post routes outcomes back to the Reckoner. The broker itself does not call resolve — it returns structured Resolution objects that carry the outcome upstream.

Source: broker.rs — "propagate() returns PropagationFacts — the broker does not touch observer state." (T0-018, §12.2)

Fitting procedure

curve() — fit the model


pub fn curve(&mut self) -> Option<(f64, f64)>

Returns Some((a, b)) on success, None if prerequisites aren't met.

Prerequisites:

ConditionDefault threshold
n_labels >= 2N/A
resolved.len() >= min_resolved500 entries
bin_size >= 10(derived: resolved.len() / n_bins)

Steps:

  1. Compute base = 1.0 / N (random chance floor).
  2. Sort all resolved entries by conviction ascending.
  3. Divide into n_bins (default: 20) equal bins. Last bin absorbs the remainder.
  4. For each bin, compute mean_c (mean conviction) and acc (fraction correct).
  5. Filter: keep only bins where acc > base + 0.005 (accuracy meaningfully exceeds chance).
  6. Linearize: for each kept bin, compute y = ln(acc - base). This transforms the model into y = ln(a) + b·conviction — a straight line.
  7. Require at least 3 valid points.
  8. Ordinary least-squares linear regression on (conviction, ln(acc - base)):
  • b = (n·Σxy − Σx·Σy) / (n·Σx² − (Σx)²)
  • ln_a = (Σy − b·Σx) / n
  • a = exp(ln_a)
  1. Cache curve_a, curve_b, set curve_valid = true.
  2. Return Some((a, b)).

Source: T0-008, §3.6 — complete algorithm traced against reckoner.rs.

Reading the curve

accuracy_at(conviction) — predict accuracy


pub fn accuracy_at(&self, conviction: f64) -> Option

Formula: min(base + a·exp(b·conviction), 0.99) — capped at 99%.

curve_params() and curve_valid()

  • curve_params() → Option<(f64, f64)> — returns cached (a, b) if fitted.
  • curve_valid() → bool — whether a curve has been successfully fitted.

Interpretation

What b means

b valueInterpretation
b > 0Higher conviction → higher accuracy. The reckoner is calibrated. Signal is real.
b ≈ 0Conviction carries no information about accuracy. The reckoner is guessing at all confidence levels.
b < 0Higher conviction → lower accuracy. The discriminants are inverted or misleading.

What a means

a controls the spread between the chance floor (1/N) and the asymptotic ceiling (1/N + a). A larger a means the reckoner has a wider accuracy range — more room for confidence to discriminate good predictions from bad ones.

The WAT spec's claim

"Monotonically increasing = real signal."
— primitives.wat

A monotonic increase means b > 0: the higher the conviction, the more often the reckoner is right. This is the gold standard — the reckoner's internal confidence aligns with external reality.

Configuration

Configurable via the chainable builder:

ParameterDefaultRole
resolved_cap5000Sliding window size for (conviction, correct) history
min_resolved500Minimum history before curve fitting is attempted
n_bins20Number of bins for the binned regression

This is discrete-only. The continuous Reckoner mode has no curve — it doesn't produce conviction values.

Source: T0-008, §2 — ".with_curve_params(resolved_cap, min_resolved, n_bins) -> Self // discrete only, no-op for continuous"

Position in the system

flowchart TD
    mkt_obs["Market Observer\n(discrete Reckoner)"] -->|"predict(thought)"| prediction["Prediction\n{ conviction, direction }"]
    mkt_obs -->|"resolve(conviction, correct)"| feeds["feeds the curve"]
    mkt_obs -->|"curve()"| curve["(a, b)"]
    curve -->|"accuracy_at(conviction)"| broker["Broker uses this\nto weight proposals\n(higher predicted accuracy\n→ stronger signal)"]

The curve closes the feedback loop: prediction → conviction → outcome → calibration → better trust weighting.

Key properties

  1. Requires discrete mode. Continuous reckoners have no curve — no conviction, no classification. (T0-008, §3.5)
  2. Bounded memory. The sliding window caps at resolved_cap (default 5000). Old data is evicted FIFO. (T0-008, §3.5)
  3. Minimum data threshold. Won't produce a curve until min_resolved (default 500) outcomes have been recorded. Prevents spurious fits. (T0-008, §3.6)
  4. Bins, not raw points. Binning smooths noise and ensures the regression isn't dominated by conviction values with many observations. (T0-008, §3.6)
  5. Chance floor is structural. base = 1/N means the model is grounded in the combinatorics of N-ary choice. A binary reckoner (N=2) has a 50% floor; a 10-label reckoner has a 10% floor. (T0-008, §3.6)
  6. Capped at 99%. accuracy_at() never returns above 0.99 — no model should promise certainty. (T0-008, §3.7)
  7. Validates the reckoner's own confidence. "By recording (conviction, correct) pairs and fitting accuracy = 1/N + a·exp(b·conviction), the system gets a self-calibration measure." (T0-008, §6.7)
