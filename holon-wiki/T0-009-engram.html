T0-009: Engram — Observation Storage and Retrieval HOW

What It IS

An Engram is "a named, serializable snapshot of a trained OnlineSubspace." It records what a learned pattern "looks like" at the moment of training and is immutable once created.

An EngramLibrary is a HashMap-backed collection of named engrams that provides three matching strategies against probes:

  1. Two-tier matching (match_vec): eigenvalue pre-filter → full residual scoring
  2. Spectrum matching (match_spectrum): cosine similarity on eigenvalue signatures
  3. Alignment matching (match_alignment): directional alignment between subspaces via principal angles

How It Works

Engram Creation

When EngramLibrary::add() mints an engram from a trained OnlineSubspace, it:

  1. Extracts the subspace's eigenvalue vector.
  2. L2-normalizes it to produce an eigenvalue_signature — "so that engrams of different sizes can be compared on a level playing field." (Normalization guard: if eig_norm ≤ 1e-10, the raw eigenvalues are used as-is.)
  3. Calls subspace.snapshot() to capture the full SubspaceSnapshot (mean, component matrix, n, threshold state).
  4. Stores optional surprise_profile (per-feature surprise scores) and arbitrary metadata (JSON).

Subspace Cache

Engrams do not serialize a live OnlineSubspace. The field subspace_cache is #[serde(skip)]. When engram.subspace() is called, it lazily reconstructs the OnlineSubspace from the stored snapshot via OnlineSubspace::from_snapshot().

Two-Tier Matching (match_vec)

Takes a raw &[f64] probe, top_k, and prefilter_k:

  1. Eigenvalue pre-filter (O(k·n)): ranks engrams by eigenvalue energy (sum of squared signature values), takes the top prefilter_k candidates. Skipped entirely if len(engrams) ≤ prefilter_k.
  2. Full residual (O(k·dim)): scores candidates by computing engram.residual(x) on each — the residual norm after projecting the probe onto the engram's learned manifold. Returns Vec<(name, residual)> sorted ascending (lower = better fit).

Spectrum Matching (match_spectrum)

Takes an eigenvalue slice and top_k. Computes cosine similarity between the probe's eigenvalue spectrum and each engram's stored L2-normalized eigenvalue_signature. "Use for rough pre-screening." Returns sorted descending (higher = better).

Alignment Matching (match_alignment)

Takes a probe OnlineSubspace and top_k. Calls subspace_alignment(engram.subspace(), 0) for each engram — measuring whether variance lives in the same part of the vector space. Returns Vec<(name, alignment_score)> sorted descending; "alignment is in [0, 1]: 1.0 = same directions, 0.0 = orthogonal."

Persistence

EngramLibrary implements JSON round-trip via save() / load(). The test test_serialization_round_trip confirms that match results are bitwise identical after save→load.
