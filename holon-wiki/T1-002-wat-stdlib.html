The wat Standard Library — built-in operations DESCRIBE

Verified leaf: T1-002 Sources: wat/std/vectors.wat (44 lines), wat/std/memory.wat (28 lines), wat/std/scalars.wat (15 lines), wat/std/fields.wat (11 lines), wat/std/statistics.wat (41 lines)

Overview#

The standard library provides "derived forms built from the corelib." These are convenience operations, not core algebra primitives. The source categorizes them as "stdlib atoms" required via (require common).

std/scalars — scalar encoding into vector space#

"Maps continuous values into the algebra."

Three encoding functions, each mapping a continuous value to a Vector:

Function Signature Purpose
encode-log (encode-log value) → Vector "Equal ratios = equal similarity." Use for orders of magnitude (ATR, volume, prices).
encode-linear (encode-linear value scale) → Vector "Proportional mapping in [0, scale]." Use for fractions, ratios, [0,1] values.
encode-circular (encode-circular value period) → Vector "Endpoint wraps to start." Use for time-of-day, day-of-week, angles.

std/vectors — derived vector operations#

"All derivable from the four algebra primitives (atom, bind, bundle, cosine). Stdlib convenience, not core."

Transformation operations#

Function Signature Description
permute (permute vector shift) → Vector "Shift vector elements — orthogonal encoding of ordered alternatives." (permute A 0) = A. (permute A 1) is quasi-orthogonal to A.
difference (difference before after) → Vector "Structural change between two states." "What's new in after that wasn't in before."
negate (negate superposition component) → Vector "Remove a component from a superposition."
amplify (amplify superposition component strength) → Vector "Boost component presence in a superposition."

Pattern operations#

Function Signature Description
prototype (prototype vectors threshold) → Vector "Extract consensus from examples."
cleanup (cleanup noisy codebook) → Vector "Snap noisy observation to nearest known pattern."
attend (attend query memory strength mode) → Vector "Soft attention over memory." Mode: :hard or :soft.

Geometric operations#

| Function | Signature | Description | |----------|-----------|-------------| | coherence | (coherence vectors) → Float | "Mean pairwise similarity of a vector set." | | l2-norm | (l2-norm vector) → Float | "Magnitude of a vector. sqrt(sum(x_i^2))." | | l2-normalize | (l2-normalize vector) → Vector | "Project onto the unit sphere. vector / ||vector||." Returns D-dimensional unit vector. | | zeros | (zeros) → Vector | "Zero vector at the current dimensionality." | | blend | (blend a b ratio) → Vector | "Weighted interpolation between two vectors." ratio=0.0 → a, ratio=1.0 → b, ratio=0.5 → midpoint. |

Note: LANGUAGE.md also lists zeros as a host language collection form with two behaviors — nullary (zero vector at dims) and (zeros n) (zero-filled array of size n). The stdlib form is the nullary vector-space version.

std/memory — learning beyond the journal#

"The journal (core) is Template 1: prediction via discriminant. These are Template 2: reaction via learned manifold."

Implementation: "CCIPCA: incremental PCA without storing the full dataset."

Constructor#

Function Signature Description
online-subspace (online-subspace dims n-components) → Subspace "Learns what 'normal' looks like from a stream."

Operations#

Function Signature Description
update (update subspace vector) "Feed an observation to the subspace."
sample-count (sample-count subspace) → Integer "How many observations has the subspace seen?"
project (project subspace vector) → Vector "Project vector onto the learned manifold (reconstruction)." "Returns D-dimensional vector: the part the subspace CAN explain."
anomalous-component (anomalous-component subspace vector) → Vector "The part the subspace CANNOT explain: vector minus reconstruction." "Returns D-dimensional vector (same dimension as input)."
residual (residual subspace vector) → Float "How far is this observation from the learned manifold?"
threshold (threshold subspace) → Float "Self-calibrating anomaly threshold."

Note: LANGUAGE.md lists the constructor as (online-subspace dims k) while memory.wat specifies (online-subspace dims n-components). The stdlib source uses n-components.

std/statistics — numeric helpers for the pre-algebra layer#

"Pure arithmetic over lists. These produce Floats that feed encode-linear or fact/scalar. They do not touch the vector algebra."

"Tempered: multi-moment computation shares a single pass where possible."

All functions accept a list xs and return a Float (except moments which returns a list of three Floats).

Function Signature Description
mean (mean xs) → Float Returns 0.0 for empty list. Otherwise fold + 0.0 xs / len xs.
variance (variance xs) → Float "Population variance. One pass for mean, one pass for deviations." Returns 0.0 for empty list.
stddev (stddev xs) → Float (sqrt (variance xs)).
moments (moments xs) → (Float Float Float) "Compute mean, variance, and stddev in two passes (not four)." Returns (mean, variance, stddev) as a list. Returns (0.0 0.0 0.0) for empty list.
skewness (skewness xs) → Float "Population skewness. Uses moments to avoid redundant passes." Returns 0.0 when stddev ≤ 0.0.

std/fields — RETIRED#

This module is entirely retired. The file states:

"(field struct-name field-name computation) has been retired. See proposal 014 (protocols). defprotocol + satisfies subsume it."

The rationale: "Indicators aren't properties of candles — they're independent fold steps with their own state. The protocol declares the interface. The struct holds the state. The step function advances it."

"This file is kept for historical reference. Do not use (field ...)."

What the stdlib is not#

The stdlib does not define new algebra primitives. Every operation in vectors.wat is described as "derivable from the four algebra primitives." The statistics module operates on plain lists of numbers, not vectors. Memory adds a second learning template (manifold learning) complementary to the reckoner's discriminant-based prediction.

The fields.wat module documents a retired feature — it defines no active operations.
