Thought Space — the vector geometry where meaning lives EXPLAIN

TL;DR: Thought space is a high-dimensional vector space (typically 4,096–16,384 dimensions) with bipolar elements {-1, 0, 1}. Meaning lives in geometry: proximity = similarity, orthogonality = unrelatedness, opposition = anti-parallel direction. Every concept, document, and pattern the system handles becomes a single point in this space, constructed from three operations — bind, bundle, and cosine similarity — and queryable by angle alone.

The geometry

Thought space is built on Vector Symbolic Architectures (VSA), specifically the MAP variant. The vector struct — a Vec with elements constrained to {-1, 0, 1} — is the atomic unit.

"A very large vector of numbers — thousands of dimensions of values in {-1, 0, 1}."
— Primer 000

The space has a key statistical property: near-orthogonality.

"Random vectors are nearly orthogonal. Two random bipolar vectors have an expected cosine similarity of 0, with standard deviation ~1/√d where d is dimensionality."
— Primer 000

The literature (Kanerva) recommends at least 10,000 dimensions for the guarantees to hold comfortably. Holon experiments ran at 16,384 and 4,096.

Coordinates and determinism

Every point in thought space is generated deterministically from its inputs:

"SHA-256(atom_string) XOR global_seed → deterministic 4096-dimensional vector. Same atom, same language implementation, any machine, any process: identical vector."
— Primer 001

The seed and dimensionality define a coordinate system that exists nowhere physically but is instantiated identically everywhere:

"The seed and dimensionality define a coordinate system that exists nowhere physically but is instantiated identically everywhere. ... The space is the network."
— Primer 004

This determinism is load-bearing — without identical atom vectors on every node, engrams become geometrically incoherent.

Encoding: how data enters the space

Structured data (JSON, events, documents) becomes a single point via the encoding stack:

  1. Atomization — map each field value to a deterministic vector
  2. Binding — encode role-filler relationships (field name × value)
  3. Bundling — superpose bound pairs into a single document vector

Three scalar encoding paths exist:

  • String atomization — opaque identifier → unique hypervector, no proximity implied
  • Magnitude-aware ($linear, $log) — interpolation between basis vectors, so proximity reflects numeric closeness
  • Temporal ($time) — cyclical decomposition into circular components
"The document vector is simultaneously a hash function and a queryable fingerprint."
— Primer 001

Measurement: one number

Cosine similarity is the canonical measurement — "the only measurement" in the algebraic specification:

"(cosine thought direction) → Float ; [-1.0, +1.0]"
— primitives.wat

Semantics of the return value:

ScoreMeaning
1.0Identical (parallel)
0.0Orthogonal (unrelated)
-1.0Opposite (anti-parallel)
— similarity.rs

The Rust codebase exposes additional metrics (Dot, Euclidean, Manhattan, Hamming, Overlap, Agreement, Chebyshev) for practical flexibility, but cosine remains the canonical primitive.

Algebra: operations within the space

Thought space has three VSA base operations from which everything is built:

OperationMechanismPurpose
bindElement-wise multiplicationEncode role-filler relationships
bundleElement-wise majority voteSuperpose into composite vectors
cosineAngle measurementQuery, probe, search — same operation at every level

Binding is self-inverse for bipolar vectors: bind(bind(A, B), A) = B, enabling decomposition and explainability.

Beyond these, the holon algebra extends the space with prototype, negate, amplify, difference, analogy, attend, project, reject, resonance, cleanup, coherence, and more — plus streaming operations like accumulate/decay.

Memory: learning patterns in the space

Streams of encoded vectors occupy structured regions of thought space. The memory layer learns those structures:

  • OnlineSubspace — incremental manifold learning via CCIPCA. Captures what normal looks like; residual becomes the anomaly signal.
  • StripedSubspace — distributes bindings across independent sub-subspaces to eliminate crosstalk for high-cardinality data.
  • Engram — a named, serializable snapshot of a trained subspace. The engram is the learned capability.
  • Reckoner — discriminant learner: discrete classification + continuous regression in vector space.

The dual-signal principle pervades: every successful discrimination requires both magnitude (aggregate residual) and direction (residual profile).

Properties

  1. Geometry encodes structure, not semantics. The algebra doesn't know what data means — it encodes structural relationships. Similarity falls out of the cosine.
  1. Explainability is algebraic, not post-hoc. unbind(anomalous_component, role_vec) decomposes anomalies into per-field attributions because the encoding uses reversible operations.
  1. Capacity is finite and real. Near-orthogonality has limits (~O(dim/log(dim))). Striping solves practical crosstalk. Accumulators saturate; capacity and purity diagnose when.
  1. Composition over individual operations. "The interesting applications come from composition rather than individual operations. The library is a vocabulary; what you build with it is up to you." — Primer 002
  1. Stability across time. "Engrams minted today are valid indefinitely — the geometric relationships they encode don't expire." — Primer 004

Implementation notes

The vector struct stores data as Vec for memory efficiency. No bipolar invariant is enforced at rest — only the from_f64 constructor guarantees {-1, 0, 1}. Normalization (normalized()) returns Vec, escaping the bipolar domain. A SIMD path (via simsimd) provides "up to 200x faster similarity computations on supported hardware."
