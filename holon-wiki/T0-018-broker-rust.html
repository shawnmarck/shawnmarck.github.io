7.2 market_idx() / position_idx() — Grid Derivation (Lines 217–224)


pub fn market_idx(&self) -> usize {
    self.slot_idx / self.position_count    // integer division
}
pub fn position_idx(&self) -> usize {
    self.slot_idx % self.position_count
}

Semantics: The N×M broker grid maps flat slot indices to (market, position) pairs. With position_count = M:

  • slot_idx = market_idx * M + position_idx
  • Therefore: market_idx = slot_idx / M, position_idx = slot_idx % M

Test confirmation (line 615–626): slot_idx=5, position_count=3 → market_idx=1, position_idx=2.

7.3 gate_open() — The Gate (Lines 229–232)


pub fn gate_open(&self) -> bool {
    let cold_start = self.grace_count < 50 || self.violence_count < 50;
    cold_start || self.expected_value > 0.0
}

Two-phase gate:

  1. Cold start: If either outcome count is below 50, the gate is always open. The broker hasn't seen enough data to make a statistical judgment.
  2. Warm: After ≥50 of both outcomes, the gate opens only when expected_value > 0.0. A broker with negative expected value is suppressed — it cannot propose funded trades.

7.6 close_all_runners() — Direction Flip Handler (Lines 286–315)


pub fn close_all_runners(&mut self, _current_price: Price) -> Vec

Called when: The market observer's active direction flips (e.g., was predicting Up, now predicts Down).

Behavior: Iterates all papers. Force-resolves any that are signaled && !resolved (active runners) with Outcome::Grace. Non-runner papers that haven't resolved survive the flip (they'll hit stop naturally). Computes optimal distances via simulation::compute_optimal_distances() on the paper's full price history. Computes the position batch from accumulated runner history if available.

Note: The _current_price parameter is accepted but unused (prefixed with underscore). The resolution uses the paper's own tracked data.

7.7 tick_papers() — The Core Tick (Lines 320–406)


pub fn tick_papers(&mut self, current_price: Price) -> (Vec, Vec)

Returns two vectors:

  1. market_signals — for market observer learning (Grace signal or Violence stop)
  2. runner_resolutions — for position observer + broker learning (runner finished)

Algorithm (per paper):


1. Snapshot was_signaled, was_resolved
2. Call paper.tick(current_price)
3. IF just signaled (trail crossed for first time):
   - Emit Grace market_signal
   - Create empty RunnerHistory for this paper_id
4. IF active runner (signaled && !resolved):
   - Accumulate composed_thought, distances, price into RunnerHistory
5. IF just resolved:
   - IF !signaled: Emit Violence market_signal
   - IF signaled (runner finished):
     - Compute position_batch from RunnerHistory
     - Emit runner_resolution (Grace)
   - Update self-assessment running averages:
     - resolution_count += 1
     - avg_paper_duration: Welford-style running average
     - avg_excursion: Welford-style running average
     - last_trail = paper.distances.trail
     - last_stop = paper.distances.stop
6. Remove resolved papers, keep the rest

Resolution categories:

PathSignalResolutionGoes toPosition batch?
Stop fires before trail crosses—Violencemarket_signalsNo
Trail crosses (first time)Grace—market_signalsNo (runner begins)
Runner trail fires—Gracerunner_resolutionsYes

Step 3 — Dollar P&L and EMA (Lines 441–470)

Reference capital: 10_000.0 (constant).

Grace P&L:


residue_usd = weight × 10_000        // weight IS excursion for Grace
entry_fee   = 10_000 × swap_fee
exit_fee    = (10_000 + residue_usd) × swap_fee
net         = residue_usd - entry_fee - exit_fee

Violence P&L:


loss_usd    = weight × 10_000        // weight IS stop_distance for Violence
entry_fee   = 10_000 × swap_fee
exit_fee    = (10_000 - loss_usd) × swap_fee
net         = -(loss_usd + entry_fee + exit_fee)

EMA update:


let alpha = 0.038; // half-life ~50 papers
self.avg_grace_net   = (1 - alpha) × avg_grace_net   + alpha × net;  // for Grace
self.avg_violence_net = (1 - alpha) × avg_violence_net + alpha × net;  // for Violence

Expected value:


let gr = grace_count / trade_count;
self.expected_value = gr × avg_grace_net + (1 - gr) × avg_violence_net;

Return: PropagationFacts containing which observers should learn and with what data.

8.1 approximate_optimal_distances() (Lines 493–508)


fn approximate_optimal_distances(entry: f64, extreme: f64, prediction: Direction) -> Distances

Used for: Market signal resolutions (Grace signal, Violence stop). Quick approximation without full simulation.

Logic:


excursion = (extreme - entry) / entry     for Up
excursion = (entry - extreme) / entry     for Down
excursion = max(excursion, 0.001)         // floor at 0.1%
trail     = (excursion × 0.5).clamp(0.001, 0.10)  // 50% of excursion, clamped
stop      = 0.001                                  // bootstrap — always near-zero

Comment (line 503–505): "Stop is learned independently — near-zero bootstrap. The simulation (compute_optimal_distances) teaches the real value. This approximation doesn't have enough information to derive stop."

8.2 compute_position_batch() (Lines 513–572)


fn compute_position_batch(
    history: &RunnerHistory,
    prediction: Direction,
    swap_fee: f64,
) -> Vec<(Vector, Distances, Distances, f64)>

Purpose: Deferred batch training data for the position observer from a completed runner's full candle-by-candle history.

Algorithm (O(n)):

  1. Compute optimal stop for the whole runner via simulation::compute_optimal_distances(&history.prices, prediction, swap_fee). Stop is a property of the full trade, not per-candle.
  1. Suffix extremum pass — O(n) backward scan:
  • For Up: suffix_ext[i] = max(prices[i], suffix_ext[i+1])
  • For Down: suffix_ext[i] = min(prices[i], suffix_ext[i+1])
  1. Per-candle optimal trail computation:
    2. 
   optimal_trail = (suffix_ext[k] - price_k) / price_k     for Up
   optimal_trail = (price_k - suffix_ext[k]) / price_k     for Down
   optimal_trail = clamp(optimal_trail, 0.001, 0.10)
  1. Sparse sampling — only when optimal changed meaningfully:
    2. 
   let changed = last_optimal_trail.is_nan()
       || ((optimal_trail - last_optimal_trail).abs() / last_optimal_trail.max(0.001)) > 0.10;

    Only emit a training sample when the optimal trail changed by >10% relative. If the answer is the same candle over candle, there's nothing to learn.

  1. Weight: weight = optimal_trail (the residue this optimal distance would capture).

Batch entry: (thought_at_candle, optimal_distances_at_candle, actual_distances_at_candle, weight).

12. Findings

  1. Complete. All 24 fields documented. All 9 methods (including 2 free functions) documented. Every code path traced.
  2. No hidden mutation. propagate() returns PropagationFacts — the broker does not touch observer state. tick_papers() returns Vec — the broker does not call propagate itself.
  3. Runner history is correctly scoped. Created at Grace signal, accumulated per-candle, consumed and removed at runner resolution. The HashMap key is paper_id, preventing stale data.
  4. The gate has two distinct modes with a clear boundary at 50 resolved trades of each outcome.
  5. The distance cascade has exactly three tiers with clean fallback semantics.
  6. P&L computation includes swap fees on both entry and exit, against a fixed $10,000 reference.
  7. close_all_runners accepts _current_price unused — all resolution data comes from the paper's own tracked price history.
  8. 15 tests cover construction, grid math, gate logic, paper lifecycle (no-resolution, violence, grace→runner), propagation accounting, dollar P&L (both directions), EMA convergence, accumulator naming, and optimal distance approximation (both directions + clamping). No test for compute_position_batch or close_all_runners.
