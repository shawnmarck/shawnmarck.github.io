T0-018: Broker (Rust) — Context-Free Verification WHAT

Source: src/domain/broker.rs (170 lines, 91 production + 79 test) + src/programs/app/broker_program.rs (353 lines) Verdict: Accountable. Complete. Gate 4 wired — Hold/Exit reckoner from anomaly. Quoted below.

1. Module Purpose — broker.rs (Lines 1–3)#

/// broker.rs — The accountability primitive. Binds one market observer +
/// one regime observer. The broker owns the game — gate 4, anxiety
/// atoms, exit/hold decisions. The treasury owns papers and capital.

What it is: The broker binds one market observer to one regime observer. It is the accountability unit. It owns outcome counts, a grace-rate gate, a noise subspace for anomaly detection, and a gate reckoner (Hold/Exit). It does NOT own paper trades, accumulators, or dollar P&L — the treasury owns papers and capital.

4.2 market_idx() / regime_idx() — Grid Derivation (Lines 66–73)#

pub fn market_idx(&self) -> usize {
    self.slot_idx / self.regime_count
}
pub fn regime_idx(&self) -> usize {
    self.slot_idx % self.regime_count
}

Semantics: Integer division and modulo. With regime_count = M:

  • slot_idx = market_idx * M + regime_idx
  • Therefore: market_idx = slot_idx / M, regime_idx = slot_idx % M

Test confirmation: slot_idx=5, regime_count=3 → market_idx=1, regime_idx=2.

4.3 gate_open() — The Gate (Lines 77–80)#

pub fn gate_open(&self) -> bool {
    let cold_start = self.grace_count < 50 || self.violence_count < 50;
    cold_start || self.expected_value > 0.5
}

Two-phase gate:

  1. Cold start: If either outcome count is below 50, the gate is always open.
  2. Warm: After ≥50 of both outcomes, the gate opens only when expected_value > 0.5.

Key detail: expected_value = 0.5 is NOT sufficient — threshold is > 0.5, not >= 0.5.

4.4 record_outcome() — Outcome Recording (Lines 83–91)#

pub fn record_outcome(&mut self, outcome: Outcome) {
    match outcome {
        Outcome::Grace => self.grace_count += 1,
        Outcome::Violence => self.violence_count += 1,
    }
    self.trade_count += 1;
    self.expected_value = self.grace_count as f64 / self.trade_count as f64;
}

Simple grace rate. No EMA, no dollar P&L.

6. Broker Program — broker_program.rs (353 lines)#

6.1 Module Purpose (Lines 1–7)#

/// broker_program.rs — the broker thread body.
///
/// Receives MarketRegimeChains through a queue.
/// Submits papers to the treasury, discovers outcomes by reading.
/// Encodes anxiety atoms from active position receipts.
/// Gate 4: the Hold/Exit reckoner. Learns from anxiety. Proposes exits.
/// On shutdown it returns the broker. The accounting comes home.

6.2 Key Changes from Previous Version (Proposal 056)#

  • Input changed from MarketPositionChain → MarketRegimeChain
  • Gate 4 is now wired: Hold/Exit reckoner learns from composed thoughts
  • Portfolio rhythms: broker tracks a window of PortfolioSnapshot values (avg_age, avg_tp, avg_unrealized, grace_rate, active_count) and encodes them as indicator rhythm ASTs
  • Broker composes a full thought: market rhythms + regime rhythms + portfolio rhythms + phase rhythm + time facts
  • One encode. One noise subspace. One gate reckoner.

6.3 broker_thought_ast() — Thought Composition (Lines 67–100)#

Builds the broker's complete thought AST each candle:

market rhythms (the market observer's full AST)
+ regime rhythms (each one an indicator rhythm AST)
+ portfolio rhythms (5 portfolio indicators as rhythm streams)
+ phase rhythm (bundled bigrams of trigrams with structural deltas)
+ time facts (hour, day-of-week — top-level, not rhythms)

All bundled together into one ThoughtAST::Bundle. This is encoded once. The noise subspace strips background. The gate reckoner predicts Hold/Exit from the anomaly.

6.4 PortfolioSnapshot — Broker's Internal State Stream#

Field Type Meaning
avg_age f64 Average paper age across active receipts
avg_tp f64 Average time-pressure across active receipts
avg_unrealized f64 Average unrealized-residue across active receipts
grace_rate f64 Current grace rate (grace_count / trade_count)
active_count f64 Number of active papers

Each becomes an indicator rhythm AST with appropriate bounds. Five rhythms total.

6.5 Gate 4 — Now Wired (Proposal 056)#

The gate reckoner is no longer "to be built." The broker_program:

  1. Composes the full thought AST (market + regime + portfolio + phase + time)
  2. Encodes via cache
  3. Feeds to broker.noise_subspace to strip background → anomaly
  4. Predicts via broker.gate_reckoner.predict(anomaly) → Hold or Exit
  5. If Exit and gate is open → proposes exit to treasury
  6. On resolution: broker.gate_reckoner.observe(anomaly, label, weight) — the reckoner learns

Label mapping: index 0 = Hold (paper should live), index 1 = Exit (get out).

8. Key Design Decisions#

Decision Detail
Grace rate, not EMA expected_value = grace_count / trade_count. Simple fraction.
Gate threshold > 0.5 Strict majority required. 0.5 exactly → gate closed.
Cold start = 50 Either outcome count < 50 → always open.
Treasury owns positions Broker submits papers and reads state.
Noise subspace on broker Learns normal composed thoughts. Anomaly drives gate 4. Proposal 056.
Gate reckoner Hold/Exit Discrete 2-label classifier. Label 0 = Hold, Label 1 = Exit. Proposal 056.
Full thought composition Market + regime + portfolio + phase + time. One encode. One noise strip. One gate.
Portfolio as rhythm streams 5 portfolio indicators encoded as indicator rhythms, not static atoms.
regime_count not position_count The second dimension is regime observers, not position observers.
