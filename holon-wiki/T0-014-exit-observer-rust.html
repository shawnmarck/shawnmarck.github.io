T0-014 — RegimeObserver (Rust) — Verification WHAT

Source: holon-lab-trading/src/domain/regime_observer.rs (34 lines: 17 production, 17 test) Context: None. Read only what the code says. Previous name: PositionObserver (renamed in commit 5bf46ca)

1. What It Is#

A RegimeObserver. The module docstring (lines 1–3) says:

"Thought middleware. Composes market thoughts with position-specific facts through a lens. Does not learn. Does not predict. Quality is measured through the BROKER's curve."

It has one field. It is a struct, not a trait. The previous version had 6 fields, 291 lines, reckoners, noise subspace, and learning methods. All stripped. This is now a pure lens carrier.

2. Fields#

# Field Type Comment
1 lens RegimeLens "Which judgment vocabulary." Enum: Core or Full.

Note: The lens enum was renamed from PositionLens to RegimeLens (commit 5bf46ca: gaze: position observer → regime observer — complete rename). The two variants (Core, Full) are unchanged.

3. Methods#

new(lens: RegimeLens) -> Self#

Constructor. Sets self.lens = lens. That is the entire implementation.

pub fn new(lens: RegimeLens) -> Self {
    Self { lens }
}

No other methods. No reckoner_distances. No observe_distances. No strip_noise. No recommended_distances.

4. The Program Thread#

The struct is a data carrier. The actual computation lives in regime_observer_program.rs (157 lines). Key behaviors:

4.1 RegimeSlot#

Field Type Role
input_rx QueueReceiver<MarketChain> Receives from one market observer
output_tx QueueSender<MarketRegimeChain> Sends to exactly one broker

4.2 regime_lens_facts#

Called once per candle with (lens, candle, scales). Returns Vec<ThoughtAST>:

  • Core: exit-regime (8 facts) + exit-time (2 facts) = 10 facts.
  • Full: exit-regime (8) + exit-time (2) + phase-current (1) + phase-series (1) + phase-scalars (1) = 13 facts.

Computed on the first slot's candle, then cloned for all subsequent slots in the same round.

4.3 Regime Rhythm Encoding (Proposal 056)#

The regime observer now also builds indicator rhythm ASTs from the candle window using regime_rhythm_specs(lens). These rhythm ASTs encode time-series patterns in regime indicators (kama-er, choppiness, entropy-rate, etc.) as bundled bigrams of trigrams — the same pattern used by market observers. The rhythms are included in the MarketRegimeChain alongside the static lens facts.

4.4 Downstream Composition#

Produces a MarketRegimeChain with all MarketChain fields forwarded plus regime_facts: Vec<ThoughtAST>. No encoding — sends raw AST downstream. The broker encodes when it composes the full thought.

5. MarketRegimeChain (chain.rs, 38 lines)#

# Field Type Source
1 candle Candle Forwarded
2 window Arc<Vec<Candle>> Forwarded
3 encode_count usize Forwarded
4 market_raw Vector Forwarded
5 market_anomaly Vector Forwarded
6 market_ast ThoughtAST Forwarded
7 market_prediction Prediction From MarketChain.prediction
8 market_edge f64 From MarketChain.edge
9 regime_facts Vec<ThoughtAST> Produced by regime observer

Previous name: MarketPositionChain (renamed along with the observer). The docstring confirms: "No encoding. No vectors. No distances. The broker encodes."

6. Tests#

2 tests in #[cfg(test)] mod tests:

# Test What it verifies
1 test_regime_observer_new Constructs with RegimeLens::Core, asserts obs.lens == Core
2 test_regime_observer_lens Constructs with RegimeLens::Full, asserts obs.lens == Full

Both are trivially simple — the struct has one field and one constructor.

7. Discrepancies#

None. The docstring accurately describes the struct: "Does not learn. Does not predict."

8. Summary#

RegimeObserver is a pure lens carrier: 1 field, 1 constructor, 34 total lines. All learning, prediction, and encoding logic has been stripped. The domain struct is a data tag — "which vocabulary does this observer use?" The program thread (regime_observer_program.rs) does the actual work: computing lens-specific facts, building regime rhythm ASTs, and composing MarketRegimeChain values with raw ThoughtAST regime facts for downstream brokers.

Name change: PositionObserver → RegimeObserver (commit 5bf46ca). All references to PositionLens → RegimeLens. All references to MarketPositionChain → MarketRegimeChain. All references to position_facts → regime_facts.
