T-1-005 — Lab Circuit (CIRCUIT.md) WHY

TL;DR: CIRCUIT.md is the architectural wiring diagram for the holon-lab-trading system. It defines 11 sub-circuits (each rendered as a Mermaid graph) that together form a complete trading machine: raw candles enter, thoughts are encoded, predictions are made, proposals are funded as trades, outcomes propagate back as learning, and stops breathe every candle. The core equation is f(state, candle) → state — one tick of the clock.

Source#

wat/CIRCUIT.md — 456 lines, ~19KB. Renders natively on GitHub via Mermaid.

Overview#

The document describes the machine as signal flow diagrams for humans. Each circuit is a Mermaid graph TD (top-down) definition. Signals flow down (candle → thought → proposal); outcomes flow back up (settlement → propagation → observers). The entire system is a loop.

The 11 Sub-Circuits#

1. The Full Enterprise (top-level)#

The master diagram. One Post per asset pair. Data flows:

  • RawCandle → IndicatorBank → Candle (enriched with 100+ indicators)
  • Candle → MarketObserver ×N (predicts direction — Up/Down)
  • Candle → ExitObserver ×M (predicts distances — trail, stop)
  • MarketObservers + ExitObservers → Broker ×N×M (composes predictions, produces proposals)
  • Brokers → Treasury → Enterprise (proposals funded, outcomes computed)
  • Enterprise → Posts (propagation back down — learning signals)

Key architectural constraint: dashed arrows denote tool usage (Vocabulary, ThoughtEncoder), not data flow. Observers call these tools; they are not upstream producers.

2. The Encoding Circuit#

RawCandle in, Vector out. Two layers of laziness:

  • IncrementalBundle — diffs against last candle's facts; unchanged facts incur zero work (their sum contribution persists)
  • Composition cache — avoids recomputing unchanged sub-trees at the encode step

The result is a thought vector, bit-identical to a full (non-incremental) bundle.

3. The Learning Circuit#

The feedback loop. A Reckoner accumulates observations. Grace (correct prediction) strengthens the discriminant; Violence (incorrect) weakens it. The loop is the learning — every tick sharpens the next prediction.

4. The Paper Circuit#

The fast learning stream. Every candle, every broker, no real capital. Papers play both sides (buy and sell) simultaneously. When a side's trailing stop fires, the paper resolves. Direction determined by which side fired first. Papers are how the machine learns before it trades.

5. The Funding Circuit#

The capital lifecycle. States:

  • available → reserved (proposal funded)
  • :active (trade live)
  • :runner (trail passes break-even — no swap, same trail, zero effective risk)
  • :settled-grace (trail fires, exit > principal — residue is permanent gain)
  • :settled-violence (safety-stop fires OR trail fires with exit ≤ principal — loss bounded by reservation)

Exactly two swaps per trade lifecycle (entry + exit). Runner transition is not a swap.

6. The Breathing Stops Circuit#

Step 3c of the four-step loop. Every candle, stops adapt. Reuses the market-thoughts-cache from step 2 — no fresh encoding. The exit observer's reckoner predicts optimal distances for the current market context. Distances convert to price levels. The trailing stop moves. The trade breathes. This is described as the value extraction mechanism — continuous adaptation, not set-and-forget.

7. The Cascade Circuit#

Three levels of distance knowledge, from specific to general:

  1. Reckoner (K=10 bucketed) — contextual: "for THIS thought, what distance?" Cosine against K prototypes, soft-weight top-3, interpolate. O(K×D).
  2. ScalarAccumulator (global) — if reckoner inexperienced: "what does Grace prefer for this pair overall?"
  3. Default crutch — if accumulator empty: the hardcoded default from construction.

8. The Propagation Circuit#

The signal that teaches. TreasurySettlement → Enterprise computes direction + optimal distances → post-propagate → broker fans out learning signals to:

  • Broker's own reckoner (Grace/Violence)
  • MarketObserver (Direction)
  • ExitObserver (optimal Distances)
  • ScalarAccumulators (values)

Weighted — large Grace teaches harder than marginal Grace. Everyone learns from one resolution.

9. The Binary Circuit#

The outer loop. 30+ threads:

  • 6 observer threads (bounded 1 in/out, unbounded learn)
  • 24 broker threads (bounded 1 in/out, unbounded learn)
  • 1 EncoderService thread (LRU cache, select loop)
  • 1 LogService thread (SQLite, batch commits)

Per-candle fold: Step 1 (settle+propagate) → Step 2 (tick+collect) → Step 3 (paper resolutions + exit learning) → Step 3c (trigger updates) → Step 4 (fund proposals, diagnostics). Shutdown: drop send ends → threads exit recv loops → join returns state.

10. The Moment Circuit#

Decoupling prediction from learning. Two independent timelines:

  • The Moment (every candle, constant time) — encode, predict (one cosine), set stops, register paper, tick papers, queue resolutions for learning
  • The Past (separate schedule, deferred) — drain learn queue, update reckoners, discriminant converges

The reckoner is a CRDT. The discriminant is a commutative monoid. Order of observation doesn't change the destination. Warmup is synchronous for the first N candles, then async.

11. The Pipe Circuit#

CSP (communicating sequential processes). Concrete layout:

  • Main thread = heartbeat (drives the fold)
  • Solid arrows = data flow via bounded(1) channels (lockstep, no locks/mutexes)
  • Dashed arrows = learning signals via unbounded channels (fire-and-forget, CRDT convergence)
  • Each consumer drains at most MAX_DRAIN=5 learn signals per candle
  • Log service batches 100 rows per commit

Key Architectural Principles#

Principle Meaning
Values up, not queues down Side-effects returned as values, never queued via parameters
Bounded(1) IS the synchronization No mutexes, no shared state — channel capacity enforces lockstep
The binary does not think It drives the fold and writes what happened
One encoding path Encoding at prediction and resolution are identical
The moment acts, the past teaches Prediction and learning are decoupled, both converge

Notes#

  • The document uses "ExitObserver" throughout; per the project's CLAUDE.md, this was renamed to "Position Observer" in Proposal 050, but CIRCUIT.md has not been updated to reflect this rename.
  • The "misses" pattern (cache misses returned as values, inserted between candles) is consistent throughout all circuits.
  • All circuits compose into one equation: f(state, candle) → state.
