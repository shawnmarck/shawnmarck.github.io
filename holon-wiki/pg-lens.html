Lens — the vocabulary subset selector. EXPLAIN

TL;DR

A lens selects which vocabulary modules fire for a given observer. The system has two lens types:

Lens TypeEnumVariantsPurpose
MarketLensMarketLens11 (3 schools × 3 specialists + 2 generalists)Which market-data vocab atoms an observer attends to
PositionLensPositionLens2 (Core, Full)Which trade-state vocab atoms a position observer uses

Every market lens includes shared/time (5 temporal atoms). Specialist lenses target 3–5 modules (22–39 atoms). Generalist lenses are the union of their school's specialists (48–56 atoms). Position lenses encode exit context (10–13 vocab atoms) with trade atoms arriving separately through the trade pipe.

Two market modules exist but are unwired: ichimoku and stochastic (removed per Proposals 041+042). See vocabulary for the full atom catalog.

Overview

"Lens wiring. Maps MarketLens/PositionLens to vocabulary facts. Extracted from orchestration/post.rs. These are the live functions that the wat-vm programs call to wire vocab modules to observer lenses."
— src/domain/lens.rs lines 1–4
"Proposals 041+042: three schools (Dow, Pring, Wyckoff), 11 lenses."
— src/domain/lens.rs line 38

Lenses are the mechanism by which different observers in the same system attend to different subsets of the vocabulary. Rather than every observer seeing all ~100 available atoms, each lens wires only the relevant modules — controlling information flow and enabling school-specific reasoning.

Generalist Property

Each *Generalist is the union of its school's three specialist lenses (superset of modules).
— T0-012

Tests enforce this: DowGeneralist ≥ each Dow specialist and PringGeneralist ≥ each Pring specialist by fact count.

GUIDE vs Rust Divergence

The GUIDE describes a flat 6-variant MarketLens (:momentum, :structure, :volume, :narrative, :regime, :generalist) and a 4-variant exit lens (:volatility, :structure, :timing, :generalist) mapped to wat modules. The live Rust has completely replaced both with:

SystemGUIDE SaysRust Actually Does
MarketLens6 flat variants11 variants in 3 named schools
PositionLens4 exit variants → exit wat modules2 variants (Core, Full) → Rust-only exit modules
Exit vocabvolatility.wat, structure.wat, timing.watexit_regime.rs, exit_time.rs, exit_phase.rs — no corresponding wat files
Removed modulesNot mentionedichimoku, stochastic — modules exist but no lens calls them

The exit wat files exist in the spec but are not wired into the running system. See vocabulary for the full spec-vs-implementation details.

Removed Modules

Per src/domain/lens.rs comments:

"Proposals 041+042: ichimoku, stochastic removed from all lenses. Their modules still exist but are no longer called."
  • ichimoku — 6 atoms, module file exists, zero consumers
  • stochastic — 4 atoms, module file exists, zero consumers
