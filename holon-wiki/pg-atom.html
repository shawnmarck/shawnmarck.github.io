Atom — the irreducible unit of meaning EXPLAIN

#TL;DR

An atom is the smallest named unit of semantic content in the Holon system. It is a string identifier (e.g., rsi, kama-er, phase) that maps deterministically to a high-dimensional bipolar vector via the <a class="wikilink" href="T0-005-vector-manager.html">VectorManager</a>. Atoms are emitted by <a class="wikilink" href="pg-vocabulary.html">Vocabulary</a> modules, carry encoded scalar values through encoding schemes (Linear, Log, Circular), and serve as the fundamental inputs to the thought pipeline. There are approximately 99–100 unique atom names across all live vocabulary domains.

#What an atom IS

An atom is a name — nothing more, nothing less. It carries no intrinsic structure beyond its string identity and the encoding scheme that produces its scalar value. The GUIDE is explicit:

"No zones. No categories. Only scalars. 'Overbought' is a human label on a continuous value — a magic number wearing a name. WHO decided 70 was the boundary? The vocabulary emits 'RSI is at 0.73.' The discriminant learns where the boundaries are." (T2-004, GUIDE lines 1396–1405)

Every atom resolves to a vector through the <a class="wikilink" href="T0-005-vector-manager.html">VectorManager</a>'s deterministic hash pipeline: SHA-256(global_seed ∥ atom)[0..8] → ChaCha8Rng → bipolar vector in {-1, 0, 1} with uniform distribution. This is the foundational guarantee — the same atomic value always produces the same vector.

"This module provides the core guarantee of Holon: The same atomic value ALWAYS produces the SAME vector." (T0-005)

#Encoding schemes — how atoms carry values

Atoms are not static symbols. Each atom carries a scalar value produced by one of three encoding schemes. The encoding scheme IS the bounding strategy:

Scheme Behavior Use case Examples
Linear { scale } Scaled clamping. Bounds come from the math. Naturally bounded scalars Bollinger position [-1,1], RSI [0,1]
Log Logarithmic compression. Compresses, doesn't bound. Unbounded positive scalars ATR ratio, volume ratio
Circular { period } Modular wrapping. Wraps, doesn't bound. Periodic scalars minute (60), hour (24), day-of-week (7)

"The encoding scheme IS the bounding strategy." (T2-004, GUIDE line 1376)

"Some facts are bounded. Some aren't. That's honest. The log doesn't bound — it compresses. The circular doesn't bound — it wraps. Only linear needs bounds, and linear's bounds come from the math." (T2-004, GUIDE lines 1393–1395)

Every relationship is a signed scalar. Not "close is above SMA20" — the relative distance, with sign:

"The sign IS the direction. The magnitude IS the distance. No 'above' atom. No 'below' atom. No boolean. Just the number." (T2-004, GUIDE lines 1346–1356)

#Atom anatomy: name → vector → thought

  1. Name — The atom's string identity (e.g., "rsi", "kama-er", "phase").
  2. Scalar value — Produced by the vocabulary encoder using the atom's encoding scheme.
  3. Vector — Deterministically derived from the atom's name by the <a class="wikilink" href="T0-005-vector-manager.html">VectorManager</a>. The scalar value modulates this vector (via scalar accumulation) to produce a thought.
  4. Thought — The vector representation, ready for composition in the thought pipeline.

The <a class="wikilink" href="T0-005-vector-manager.html">VectorManager</a> stores two separate caches:

  • Atom cache — keyed by atom name string → vector
  • Position cache — keyed by i64 position with synthetic key "__pos__{n}", kept strictly separate so positional encoding never collides with semantic encoding.

"Position vectors are seeded with __pos__{position} to match the Python implementation's _position_to_seed()." (T0-005)

#Key properties

Property Detail
Determinism Same (dimensions, seed, atom) → identical vector, always.
Seed isolation Different seeds → different vectors for the same atom.
Atom isolation Different atoms → different vectors (with overwhelming probability).
Position isolation Positional encoding never collides with semantic atom encoding.
Conditional emission Some atoms only fire when a condition is met (e.g., divergence atoms emit 0–3 facts).
Name collision Same atom names in different modules (e.g., kama-er in market/regime and exit/regime) map to the same vector — they are the same atom.
Codebook persistence Atom-to-vector mappings can be exported/imported for cross-process consistency.

"Export the atom codebook as a serializable map. Returns atom names mapped to their raw i8 vector data. Useful for persisting the vector cache across process restarts or sharing identical mappings between distributed nodes." (T0-005)

#Design philosophy

Atoms embody the Holon principle of pure functions with no state:

"Pure functions. Something in, facts out. No state. Each domain thinks about different things. Market vocab thinks about candles. Exit vocab thinks about candles and conditions. Risk vocab (future) thinks about portfolio state. The input is whatever the domain needs to form its judgment." (T2-004, GUIDE lines 1244–1247)

And the vocabulary owns both encode and decode:

"The vocabulary owns the encode AND the decode — it put the value on the scalar, it can take it back. That's why scalar accumulators work." (T2-004, GUIDE lines 1396–1405)
