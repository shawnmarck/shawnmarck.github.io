Prologue: "The Idea That Wouldn't Leave"

Series: Prologue

Problem: Author spent years at AWS Shield unable to convince leadership that VSA-based anomaly detection could work for DDoS. VSA discovered via Clojure/conj talk. Left AWS to build full-time.

Decision: Solo after-hours project. Two site tracks: primers (fundamentals) and story posts (development arc). Python-first for LLM compatibility despite Clojure preference.

Concept produced: The project itself — Holon. "Build it in Python first, then translate."

Key quote: No explicit single quote, but the framing establishes the entire project as a bet on algebraic structure over statistical pattern matching.

Series 002-001: "Python: The Foundation"

Series: 002 (Python Foundation)

Problem: Original goal was a database for structural sub-document queries (DynamoDB/ES alternative), not anomaly detection. Day 1: full HTTP API with FAISS.

Decision: The encoding proved more powerful for pattern recognition than database queries. Pivot to anomaly detection. Python chosen as "Rosetta Stone" despite author's Clojure preference.

Concept produced: The primitive algebra: bind, bundle, prototype, difference, blend, amplify, negate. Proven across tasks, recipes, graph topology, text, and (failed) Sudoku.

Key quote: "Build it in Python first, then translate."

Series 002-002: "The NP Wall"

Series: 002 (Python Foundation)

Problem: Batch 004 attempted Sudoku (NP-hard) with VSA. Result: 14× slower than standard backtracking.

Decision: Accepted failure. Mapped the boundary: approximate geometric similarity cannot do exact combinatorial search. Everything on the "approximately correct" side is where the algebra works.

Concept produced: The NP-hard boundary precisely established. The primitives forged in the attempt (bind, bundle, etc.) were the real output, not the Sudoku solver.

Key quote: "Approximate geometric similarity cannot do exact combinatorial search."

What was learned: Token cost acknowledged as expensive. The failure was productive — the primitives became permanent.

Series 002-003: "Scale, Detection, and the Python Ceiling"

Series: 002 (Python Foundation)

Problem: Scaling from experiments to real detection workloads. Batches 006–012.

Decision: Multiple key decisions:

  • Batch 006: LLM memory augmentation — 82% context token reduction, 95% decision recall.
  • Batch 007: Qdrant integration, $time marker, 16k dimension selection (σ_cos ≈ 1/√d = 0.0079 at 16k, fitting 10,000+ near-orthogonal atoms).
  • Batch 008: Enterprise challenges — 100% routing accuracy, config drift detection (2747× ratio), 5 attack types at 3.88ms.
  • Batch 009: GPU rejected — transfer overhead made CPU faster.
  • Batch 010: Accumulator primitive breakthrough (frequency-preserving streaming).
  • Batch 011: 5M records in 7.5 min.
  • Batch 012: Zero-hardcode detection — 100% recall, 4% FP. Python ceiling: 3,100 pkt/sec. Rust port started Feb 6.

    • Concept produced: Accumulator primitive. 16k dimensions as default. Python ceiling established.

    Key quote: "GPU lost to CPU due to transfer overhead."

    What was learned: Two-implementation validation strategy (Python + Rust cross-checking) established.

Series 003-001: "The Rust Port"

Series: 003 (Rust & Production)

Problem: Python ceiling at 3,100 pkt/sec. Needed 10-15× speedup.

Decision: Feb 6: 2,554 lines across 13 files in one evening. Python as "Rosetta Stone" — 239 passing tests as behavioral spec. Flat facade pattern. SIMD from day 1. SHA256+ChaCha8 for deterministic atom hashing.

Concept produced: holon-rs. Two-implementation validation strategy (Python + Rust with independent test suites catching different bug classes).

Key quote: "Python as 'Rosetta Stone' — 239 passing tests as behavioral spec."

Series 003-002: "The Labs"

Series: 003 (Rust & Production)

Problem: Synthetic challenge data had taken the algebra as far as it could. Needed real packets on real interfaces for DDoS detection.

Decision: Built two labs in one weekend (Feb 7–8):

  • holon-lab-ddos: XDP sampling, AF_PACKET generator (raw sockets failed — kernel routing drops packets with non-local source IPs), macvlan hairpin, perf buffer, sendmmsg batching. 45k PPS validated.
  • holon-lab-baseline: WordPress + nginx + MariaDB on Docker, 23 source IPs via Squid + ipvlan, LLM-driven Playwright agents (20 user + 3 admin), 80% Chromium / 15% WebKit / 5% Firefox.

    • Concept produced: The four-repo development pattern (holon, holon-rs, holon-lab-ddos, veth-lab). Walkable trait with zero-allocation visitor. Batch 013 rate limiting in both languages simultaneously.

    Key quote: "This is the first challenge developed in parallel across both implementations from the start."

    What was learned: Raw sockets don't work for spoofed traffic — kernel routing drops them silently. AF_PACKET (layer 2) is the fix. Full-day LLM-assisted development is "genuinely taxing."

Series 003-003: "1.3 Million Packets Per Second"

Series: 003 (Rust & Production)

Problem: Integrate holon-rs into the DDoS lab sidecar. Validate detection at line rate.

Decision: Feb 9: veth lab — 2,873 lines, 20 files. Sidecar architecture: XDP samples packets → perf buffer → holon-rs encode → drift detection → BPF map rules → XDP enforce.

Concept produced:

  • Bitmask Rete: O(fields), capped at 64 rules.
  • BPF tail-call DFS: 1,000,000 rules at line rate with ~5 tail calls per packet.
  • p0f field fingerprinting for autonomous attack classification.
  • s-expression rule format (Clara-style LHS ⇒ RHS).

    • Key numbers: 1.3M PPS, 99.5% drop rate, 52ms detection latency.

    Key quote: "Per-packet cost is identical at 50K and 1M rules: ~5 tail calls either way."

    What was learned: The tree is self-optimizing — no rule ordering needed. Adding rules that share constraints shares tree nodes.

Series 003-004: "The Rule Engine"

Series: 003 (Rust & Production)

Problem: Rules needed richer predicates. Detection needed to extend beyond equality matching. Windows were too slow for continuous detection.

Decision: Feb 13–16. Four major additions:

  • EDN replaces JSON for rules (streaming parseable, Clara-idiomatic). "Biased choice" acknowledged honestly.
  • Range, mask, and L4 byte-match predicates added.
  • Real-time dashboard with live DAG visualization (surfaced two bugs: sawtooth flapping from unnamed rate-limit buckets, compound range predicates silently dropping constraints).
  • IPv4 header fingerprinting: 6 new dimensions (ip-id, ip-len, dscp, ecn, mf-bit, frag-offset) added to PacketSample. "The detection code didn't change. The Walkable implementation picked them up automatically."
  • Full L4 payload analysis: 2048 bytes, 32 windows of 64 bytes, autonomous byte-match rule derivation.
  • The decay model: continuous per-packet accumulation replacing fixed windows. Dual accumulator (direction with per-packet decay, rate with time-based decay). Hybrid trigger (200 packets OR 200ms).

    • Concept produced: The decay model (establishing 750ms as the honest detection baseline). EDN as rule language. Data-first design proven: "Adding signal doesn't touch behavior."

    Key quote: "The detection code didn't change. [...] That's what data-first means in practice: the abstraction boundary is at the data, not the logic."

    What was learned: Vector-derived rate estimation from batch 013 was rendered structurally inapplicable by the decay model. "Sometimes the better design makes the clever technique irrelevant." Dashboard surfaced bugs invisible in log files.

Series 003-005: "Engrams and the 765ms → 3ms Moment"

Series: 003 (Rust & Production)

Problem: 750ms detection latency from decay model was the honest baseline. Could remembered patterns be matched faster?

Decision: Feb 17–20. CCIPCA online subspace learning. Engram = durable trace of learned pattern (manifold snapshot). Two-tier matching: eigenvalue pre-filter + full residual scoring. Three-layer architecture refactor (kernel/memory/highlevel) in both Python and Rust.

Concept produced:

  • Engram: "programmatic neural memory" refined to "engram" (neuroscience term for physical memory trace).
  • Two-tier matching: eigenvalue pre-filter (fast, cheap) + full residual scoring (expensive, thorough).
  • Three-layer architecture: kernel/ (stateless math), memory/ (subspace, engram, library), highlevel/ (convenience wrappers).

    • Key numbers: 765ms baseline → 3ms on engram hit. 100% accuracy, variant resilience, eager activation.

    Key quote: "750ms to accumulate signal versus 3ms to match a single packet against a remembered pattern."

    What was learned: Experiments 014–018 validated the approach. Speculative drone autopilot application noted in commits.

Series 004-001: "The L7 Lab: Building the HTTP WAF"

Series: 004 (L7 WAF)

Problem: Move from L4 packet inspection (veth-lab) to L7 HTTP/TLS inspection. The same detection architecture, one layer up.

Decision: Feb 23–26. TLS-terminating proxy with lossless ClientHello capture (ReplayStream wrapper tees raw bytes before rustls processes them). Dual SubspaceDetector (TLS + REQ) with independent tuning. EDN rules with string keys. Per-IP token bucket rate limiting.

Concept produced:

  • Lossless ClientHello capture: full wire-order extension bytes preserved, not reduced to JA3/JA4 hashes. "We don't reduce anything. The full TlsContext gets encoded into the 4096-dimensional vector."
  • HTTP data model preserving duplicates: headers: Vec<(String, String)> not HashMap. Enables smuggling detection.
  • Adaptive ordered-vs-set TLS field selection: detect whether attacker is randomizing, choose ordered or set-based constraint accordingly.
  • Specificity ranking via derive(Ord): cross-layer > HTTP > TLS, more constraints > fewer.

    • What was learned / bugs found:

  • Per-IP rate limiting is a memory exhaustion DoS vector at scale. Documented honestly: "This is exactly the kind of thing that works in a demo and breaks in deployment."
  • 14-second stall from RwLock in drain loop. Fixed by batching writes.
  • "The LLM doesn't have 3 AM. It has training data, and the training data says token buckets are per-IP. The sorcerer still has to review the spellbook."

Series 004-002: "No Rules, No Signatures: The Expression Tree"

Series: 004 (L7 WAF)

Problem: Rules were flat structs with 13 hard-coded FieldDim variants. Couldn't express composition like (first (header "user-agent")) or (count (nth path-parts 2)). Concentration-based detection fails for high-cardinality content (random product IDs).

Decision: Feb 27–28. Complete rewrite of rule language as composable s-expressions over EDN. Three orthogonal concepts: domain accessors (26 dimensions), generic functions (first, last, nth, count, keys, vals, set, lower), operators (=, prefix, suffix, contains, regex, gt, lt, subset, superset). Expression tree compiler dimension-agnostic, O(depth) evaluation.

Concept produced:

  • Expression tree: Rete-spirit DAG compilation. 1M rules evaluated in under 3µs (hit) / 50ns (miss). "No production router or firewall I know of can host a million rules while maintaining sub-microsecond evaluation."
  • VSA surprise probing: bind(anomalous_component, role_vector) recovers per-field anomaly contribution. Hierarchical sweep: top-level → positional → nested.
  • Shape encoding: field lengths encoded alongside values. Catches attacks where content is random but structure is fixed.
  • SurpriseHistory: cross-tick consistency gate requiring anomalous fields to persist across 3+ consecutive ticks.
  • Engram resilience: parallel fast-path deployment AND fresh rule generation. Stored engram rules never inherited by new engrams.

    • Key numbers: 287 tests, 7/7 attack waves mitigated autonomously. 1M rules at 2.57µs hit / 50ns miss (6 dimensions).

    Key quote: "The tree is the check. Irrelevant branches are never reached. This is what makes the tree self-optimizing."

    What was learned / bugs found:

  • Bug A: is_redundant check rejecting surgical rules that were supersets of broader ones. Wrong for tree evaluation (tree picks most specific match). Fixed by removing subsumption check.
  • Bug B: Engram fast-path deploying stale rules for geometrically similar but semantically unrelated attacks. Fixed by parallel fresh rule generation alongside engram deployment.

Series 005-001: "The Spectral Firewall"

Series: 005 (Spectral Firewall)

Problem: The manifold firewall concept doc needed to become working code. Four layers of geometric anomaly detection.

Decision: Mar 1–3. Last Python experiment (batch 018, 16 experiments, 8,152 insertions). Validated four-layer architecture. Built in Rust in one commit (2,457 insertions, 19 files). Live Nikto scan against DVWA.

Concept produced:

  • Four-layer geometric WAF: Layer 3 rule tree (~50ns), Layer 0 residual score, Layer 1 deny threshold, Layer 2 async window spectrum. NaN → Deny (conservative default).
  • AES-256-GCM sealed denial tokens: encrypted verdict context in 403 response. Avg 1.6KB, URL-safe base64.
  • Dual-signal spectral pre-filter: eigenvalue spectrum (magnitude) × principal angle alignment (direction). 100% accuracy at 75% compute savings.
  • Normal manifold allow list with freeze at sample 49 (prevents attack poisoning of baseline).

    • Key numbers: 41µs p50 during DDoS. 10,121 Nikto requests denied, 0 exploitable vulnerabilities through proxy (17 without). 0% FPR on synthetic scenarios.

    Key quote: "Neither metric alone has been sufficient in any experiment. Combined, neither has failed."

    What was learned / honest caveats:

  • Attribution was "garbage" — all fields within a 1.2-point spread (47.5–48.7). The denial was correct; the explanation was useless.
  • Three problems remained: monolithic encoding (capacity ceiling), magic numbers (sigma_mult=3.5, deny_mult=2.0), single scanner diversity (Nikto only).

Series 005-002: "Self-Calibrating"

Series: 005 (Spectral Firewall)

Problem: Magic numbers calibrated by hand against one lab's traffic. Attribution crosstalk from ~100 bindings packed into one 4096-dimensional vector. Nikto-only testing.

Decision: Mar 4–6. Three problems, three solutions:

  1. Striped encoding: FNV-1a hash of fully-qualified domain name assigns bindings to stripes. 32 stripes × ~1.7 bindings per stripe. StripedSubspace into holon-rs core. Cosine similarity drilldown (not L2-norm — LLM had applied wrong VSA family's technique). Removed body_len (client-declared, not observed).
  2. Parameter sweep: K (deflation steps) dominates quality, not DIM. Winner: DIM=1024, K=32, STRIPES=32 → 997µs, 13× separation. Later re-optimized to K=20 after char_list.
  3. Self-calibrating boundaries: ResidualBuffer derives thresholds from observed traffic. deny_threshold = sqrt(buf_max × CCIPCA_threshold). Anomaly breadth gate (concentration, Shannon entropy, Gini) prevents poisoning. 20 LLM browser agents + 3 vulnerability scanners concurrently.

    4. Concept produced:

  • StripedSubspace: generic holon-rs primitive for crosstalk-free attribution.
  • Self-calibrating decision boundaries via geometric mean.
  • char_list encoding: character-level positional binding for fuzzy string matching. Spread WalkableValue variant.
  • Von Neumann automaton framing: engram as simultaneously description and program (data-is-policy).

    • Key numbers: 99.1% precision. 5,118 attack denials. Sub-millisecond after optimization.

    Key quote: "The LLM's knowledge is vast but not always grounded in your system. When the results don't make sense, challenge the method, not just the parameters."

    What was learned: First run of concurrent lab: 0 denials (attack traffic being learned as normal). Three cascading parameter failures. Run 2 with tightened params: 3,605 denials, 0 FP. Led to eliminating magic numbers entirely.

Series 005-003: "The Residual Profile"

Series: 005 (Spectral Firewall)

Problem: Geometric mean threshold had 99.1% precision but ~48 FP. Minority browsers (WebKit/Firefox) after Chrome-dominated warmup had similar residuals to Nikto scanners — same magnitude, different cause.

Decision: Mar 8. Two efforts:

  1. 21-strategy threshold sweep: simulation (21→5), live validation (5→1). Logarithmic mean (ccipca - buf_max) / ln(ccipca / buf_max) won: 1.9% FPR (43% fewer than geometric), fastest settling.
  2. Structural ratio experiment: classify fields as "structural" (TLS fingerprint) vs "content" (path, headers). Hypothesis: minority browsers would have high structural ratios. Result: inverted. Scanners had higher structural ratios (0.72–0.77) vs minority browsers (0.50–0.52). Reverted.

    3. Concept produced:

  • Residual profile: dual-signal deny gate. Magnitude + direction. ~3000× cheaper than primary baseline.
  • Logarithmic mean threshold strategy.
  • Systematic threshold evaluation methodology: simulation → live isolation funnel → temporal FP analysis.

    • Key numbers: 0.1% FPR. 4,351 attack denials. 1 browser FP (early convergence only, +42s). 0 late FPs. 17 adaptive learns during active concurrent attacks with zero poisoning.

    Key quote: "This is the fourth time this project has proved the same thing: magnitude and direction together. Always."

    What was learned: "The structural ratio didn't work because it was a single-signal approach — one ratio, one threshold, one number. The same pattern as every other failure in this project." The fix was to stop discarding what was already being computed.

Series 006-001: "The Thought Machine"

Series: 006 (Markets)

Problem: Prove the algebra works beyond network security. Markets chosen as proving ground — "DDoS detection and trading are structurally identical. A DDoS attack is an anomaly on a trend line. A market reversal is the same signal in a different stream."

Decision: Mar 15–24. Started with pixel obsession (viewport encoding: 48-candle chart as pixel grid, 10,000-dimensional vector). Two weeks of Python bashing, then frustration-driven pivot to Rust. trader → trader2 → trader3. v10 breakthrough: PELT change-point detection on 17 indicator streams producing 120+ compositional facts.

Concept produced:

  • Visual encoding: 50.5% accuracy (barely above random). Gap between win-win and win-loss cosine: 0.0004. Dead end.
  • Thought encoding: 57.1% accuracy. d'=0.734 separation. Regime-invariant (explained ratio stable at 53% across bull/bear/choppy).
  • PELT segment narrative: indicator × direction × log-magnitude × log-duration × position × chrono-anchor. Each fact a compositional binding.
  • Mean-stripping: subtract (buy+sell)/2 before cosine. Fixes prototype convergence cascade (cos(buy,sell) went 0.79→0.99 without it).
  • The Journal: 238 lines. Two accumulators, discriminant, cosine. Mean-stripped input. The complexity is in the encoding, not the learning.

    • Key quote: "Charts don't predict; interpretations of charts predict. Visual engrams are a dead end."

    What was learned / honest admissions:

  • The +228% P&L was a lie: "record_trade now uses outcome_pct (signed price return at first threshold crossing), not peak_abs_pct. Previous +228% was fiction." Honest number: +5.49%.
  • "I genuinely thought the project was going to join the pile of attempts that burned in a fiery death."
  • Every adaptation experiment failed. Fact pruning hurt by 2.3%. Weighted bundling created positive feedback loop. Higher dimensions showed no improvement.

Series 006-002: "The Conviction Curve"

Series: 006 (Markets)

Problem: 57.1% accuracy with zero selectivity — trading every candle. Is there a way to know when the system is right?

Decision: Mar 25–27. trader3 lands. The conviction flip (high confidence about continuation = trend extreme = reversal likely). q99 discovery at 59.7% (from explainability commit, not tuning). Exponential curve fit: accuracy = 0.50 + a × exp(b × conviction). One economic parameter (min_edge) derives everything. 652k candle validation. Vocabulary expansion to 107 atoms (62.1% at 100k).

Concept produced:

  • The conviction-accuracy curve: continuous, monotonic, exponential. Three phases: noise (50.3%), signal (54.5%), exponential (63%+).
  • Self-derived threshold from conviction distribution.
  • Vocabulary as model: 23 atoms added → 57% to 62%, same algorithm. "The discriminant discovers which thoughts predict. The human's job is to name the thoughts."
  • Self-supervised walk-forward: no train/test split. Every candle simultaneously training and test data.

    • Key numbers: 652,362 candles, 5,298 trades, 56.5% accuracy across 6 years. Best year: 2022 bear market at 60.3%. Worst: 2023 choppy recovery at 50.1%.

    Key quote: "The geometry doesn't care about the regime. It cares about the measurement basis."

Cross-Cutting Patterns

Recurrent Discovery: Magnitude + Direction

Proved four times across the project. Every successful discrimination used both signals. Every failure used only one.

  • Batch 017: cosine-to-centroid (mag) + reconstruction residual (dir)
