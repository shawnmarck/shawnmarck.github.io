Thought System — how candles become thoughts and thoughts become meaning EXPLAIN

TL;DR

A candle (market data) enters a vocabulary module (pure function, no state), which emits facts as ThoughtAST nodes. The ThoughtEncoder walks that AST recursively, converting each node into a bipolar vector via the VectorManager (atoms) or ScalarEncoder (numbers). Bindings compose relationships (bind), superpositions combine them (bundle), and cosine measures similarity. The result is a single high-dimensional thought vector — a point in meaning-space where "similar market conditions live near each other." The Reckoner Coalgebra then observes and learns from these thoughts over time.

1. The Pipeline: Candle → Facts → AST → Vector → Thought

flowchart TD
    candle["candle"] -->|"facts"| vocab["vocab module\n(pure fn)"]
    vocab -->|"Vec<ThoughtAST>"| encoder["ThoughtEncoder\n.encode()"]
    encoder -->|"Vector"| vector["Vector"]
    encoder -->|"across candles"| incremental["IncrementalBundle"]

Step 1: Vocabulary modules emit facts

Vocabulary modules are pure functions — something in, facts out, no state. Each domain (market, exit, shared) observes different aspects of a candle and emits a Vec.

"Pure functions. Something in, facts out. No state. Each domain thinks about different things." — T2-004

Each fact is a ThoughtAST node carrying an encoding scheme:

SchemeWhen to useExamples
Linear { scale }Naturally bounded scalarsRSI [0,1], Bollinger position [-1,1]
LogUnbounded positive scalarsATR ratio, volume ratio
Circular { period }Periodic scalarsminute (60), hour (24), day-of-week (7)
"The encoding scheme IS the bounding strategy." — T2-004

Every relationship is a signed scalar, not a boolean. No "above" or "below" atoms — just the number with sign and magnitude.

"No zones. No categories. Only scalars. 'Overbought' is a magic number wearing a name." — T2-004

Step 2: ThoughtAST — the thought expression tree

The ThoughtAST enum (T0-011) is the intermediate representation between vocabulary and vector:

VariantPurposeExample
Atom(String)Named symbol, no scalar"rsi"
Linear { value, scale }Bounded scalarLinear { value: 0.73, scale: 1.0 }
Log { value }Unbounded positive scalarLog { value: 1.5 }
Circular { value, period }Wrapping periodic scalarCircular { value: 14.0, period: 24.0 }
Bind(Box, Box)Two concepts composed into a relationshipBind(Atom("rsi"), Log(1.5))
Bundle(Vec)Superposition of multiple relationshipsBundle([fact1, fact2, fact3])
Sequential(Vec)Position-sensitive sequence[A, B] ≠ [B, A]

A typical market candle might produce ~60–80 individual facts across all vocabulary modules, bundled into a single thought vector.

collect_facts() recursively flattens nested Bundles/Sequentials, skipping bare Atoms (names without values are not facts).

Step 3: ThoughtEncoder walks the AST

The ThoughtEncoder (T0-011) is stateless — no cache. It holds only a ScalarEncoder and a VectorManager. Each encode() call re-evaluates the entire subtree:

AST variantEncoding action
Atom(name)VectorManager.get_vector(name) — SHA-256 → ChaCha8Rng → bipolar vector
Linear { value, scale }ScalarEncoder.encode(value, Linear { scale }) — rotational interpolation
Log { value }ScalarEncoder.encode_log(value) — log₁₀ then rotational
Circular { value, period }ScalarEncoder.encode(value, Circular { period }) — modulo wrap
Bind(left, right)Recursively encode both, then Primitives::bind(&l, &r)
Bundle(children)Encode all children, then Primitives::bundle(&refs)
Sequential(items)Encode each item, permute by position index, then bundle
"Production encoding goes through encoding::encode::encode() which checks the LRU. This struct exists for tests and IncrementalBundle." — T0-011

2. The Vector Algebra — how thoughts are composed

The foundation is a Vector Algebra defined in the WAT spec (T0-002) as:

"A commutative monoid under bundle, with bind as the group action and cosine as the measurement homomorphism."

atom — concept identity

"Name a concept → deterministic bipolar vector. Same seed + same name = same vector everywhere."

Implemented by VectorManager: SHA-256(global_seed ∥ name)[0..8] → ChaCha8Rng → {-1, 0, +1}^dim.

bind — compose relationships

Element-wise multiplication of bipolar vectors. Self-inverse: bind(bind(A, B), A) = B (exact for bipolar, not approximate).

This is how "rsi is 0.73" becomes a single vector: bind(atom("rsi"), encode_linear(0.73, 1.0)).

bundle — superimpose facts

Element-wise majority vote (sum then sign-threshold): >0 → +1, <0 → -1, ==0 → 0. Associative and commutative. This is how ~60 individual facts collapse into one thought vector.

Divergence noted: WAT says "identity = zero vector" but Rust panics on empty input.

cosine — measure similarity

The only measurement. Returns [-1.0, +1.0]. How the system determines that "this market state is similar to that one."

3. Scalar Encoding — mapping numbers into meaning-space

Scalar encoding ensures that similar values produce similar vectors (T0-004). Two independent systems exist:

Rotational encoding (ScalarEncoder)

Generates two random bipolar vectors (base, ortho) via ChaCha8Rng, then rotates between them:


frac = value / scale
angle = frac × 2π
result[i] = sign(base[i] × cos(angle) + ortho[i] × sin(angle))

Used by ThoughtEncoder for Linear, Log, and Circular nodes.

Positional encoding (Encoder/Walkable path)

Transformer-style sinusoidal encoding with geometrically spaced frequencies:


freq = 1.0 / scale^(i / dim)
if i is even: sin(position × freq)
if i is odd:  cos(position × freq)

Used by the Walkable/Encoder path for LinearFloat, LogFloat, TimeFloat scalars.

These produce different vectors for the same input — they are separate code paths.

Key encoding properties

  • Similarity gradient: Closer values → higher cosine. Verified: sim(0,10) > sim(0,50).
  • Circular wrap: Hour 23 is more similar to hour 0 than hour 12.
  • Log ratio preservation: Equal ratios → approximately equal similarity drops.
  • Quantization awareness: Bipolar thresholding causes quantization; accumulators that average multiple samples recover gradient behavior.

4. Walkable — zero-serialization encoding

The Walkable trait (T0-006) enables any Rust type to be directly encoded into vectors without JSON/EDN serialization. It defines a structural type system (WalkType) that the encoder recursively walks:

WalkTypeEncoding semantics
ScalarAtom lookup or special numeric encoding
MapRole-filler binding: each key bound to its value's vector, then all bundled
ListPosition-encoded: each element bound to its indexed path, then bundled
SetItems bundled directly, then bound with "set_indicator" role
SpreadFan-out: each element becomes an independent leaf binding
"The key insight is role-filler binding: we bind each value (filler) with its key (role) before bundling." — T0-006

This is an alternative path to ThoughtAST — both produce the same vector algebra operations, but Walkable works on native Rust types while ThoughtAST works on the explicit tree.

5. IncrementalBundle — efficiency across candles

For streaming data (one candle at a time), re-encoding ~60 facts every tick is expensive. IncrementalBundle (T0-011) maintains running Vec sums:

  1. First candle: Full encode — encode every fact, accumulate sums, store HashMap.
  2. Subsequent candles: Diff — remove vectors for facts that disappeared, encode only new/changed facts, update sums.
  3. Threshold: sign(sum) produces the bundle — proven bit-identical to a full bundle from scratch.
"round_to at vocab emission is load-bearing for the AST diff. Quantized floats compare reliably." — T0-011

Without round_to, floating-point drift would cause false "changed" detections, degrading to full recompute.

6. The Full Vocabulary Landscape

Three domains, scoped subfiles (T2-004):

~86+ atoms fire per candle across a typical lens configuration. Each atom becomes one ThoughtAST leaf. All leaves are bundled into one thought vector.

Known divergences

  • Exit domain: GUIDE describes volatility.wat, structure.wat, timing.wat — none are wired in live Rust. The Rust exit vocab uses separate regime/time/phase modules instead.
  • Ichimoku & stochastic: Removed from all lenses per Proposals 041+042. Modules exist but are not called.

8. What Makes a Thought Meaningful

A thought vector is meaningful because of the algebraic properties of its construction:

  1. Bind is self-inverse: You can decompose a thought to recover its components. bind(bound_thought, role_vector) ≈ filler_vector.
  2. Bundle preserves similarity: A bundled thought is similar (by cosine) to each of its constituent facts.
  3. Cosine is the only measurement: One number determines similarity. No thresholds, no categories in the encoding itself.
  4. The vocabulary owns encode AND decode: "It put the value on the scalar, it can take it back. That's why scalar accumulators work."
  5. The Reckoner learns boundaries: The vocabulary emits raw scalars. The Reckoner Coalgebra learns where boundaries are via observe, predict, and resolve — no human-chosen thresholds.
"The vocabulary emits 'RSI is at 0.73.' The discriminant learns where the boundaries are." — T2-004
