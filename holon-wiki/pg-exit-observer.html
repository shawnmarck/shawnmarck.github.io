Exit Observer — Perceiving When to Leave a Position

TL;DR: A PositionObserver (also called "exit observer") predicts two continuous values — trailing stop distance and safety stop distance — from a position thought vector using two independent continuous Reckoners. It learns from hindsight-optimal distances determined by simulation. Each observer wears a PositionLens that selects its trade-state vocabulary. It has a noise subspace for anomaly detection despite its own docstring claiming it doesn't. The broker owns the production cascade (reckoner → accumulator → default); the observer only exposes raw reckoner queries. M per trading system; N×M brokers total.

Prediction (reckoner_distances)

One public prediction method: reckoner_distances(position_thought: &Vector) -> Option.

  1. Check experience — if either reckoner has zero experience, return None.
  2. Query trail — trail_reckoner.query(position_thought) → trailing stop distance.
  3. Query stop — stop_reckoner.query(position_thought) → safety stop distance.
  4. Return — Some(Distances { trail, stop }).

Tier 1 only — raw reckoner output with no fallback. The broker owns the full three-tier cascade:

TierSourceOwner
1Reckoner query (this method)Position observer
2Scalar accumulators (per-broker)Broker
3default_distancesBroker

Per Proposal 026, queries operate on position_thought only — no composition with market thought at query time.

"The broker owns the full cascade because it owns the scalar accumulators. The reckoner answer comes from outside (the position observer's continuous reckoner), but tiers 2 and 3 are broker-owned." — T0-018

Learning (observe_distances)

One learning method: observe_distances(position_thought: &Vector, optimal: &Distances) -> (f64, f64, f64, f64, f64, f64).

  1. Pre-learning prediction — query each reckoner for what it would predict given current state. If no experience, use default.
  2. Observe — trail_reckoner.observe_scalar(position_thought, optimal.trail, 1.0) and stop_reckoner.observe_scalar(position_thought, optimal.stop, 1.0). Weight is always 1.0 — each observation counts once.
  3. Return — (trail_error, stop_error, predicted_trail, predicted_stop, optimal_trail, optimal_stop).

The "optimal" distances come from hindsight — the simulation module sweeps candidate trail/stop values against actual price histories to determine what would have been ideal.

How the Continuous Reckoner Learns

Each continuous Reckoner uses bucketed nearest-neighbor regression:

  • 10 buckets span the discovered value range. The first observation seeds a tiny range (±0.001); out-of-range values expand it; every 100 observations the range contracts to where alive weight lives ("breathing range").
  • Implicit decay — every observation decays all buckets by 0.999. "Every observation is a tick of time."
  • Query — cosine against bucket prototypes, soft-weight top-3, interpolate. O(K×D), constant in observation count.
  • Range is data-driven — "No magic min/max. The data teaches the range. K is the only parameter." — T0-008

Noise Stripping

strip_noise(thought: &Vector) -> Vector — converts to f64, runs noise_subspace.anomalous_component(), converts back. Returns what the subspace cannot explain. Same pattern as the market observer.

The Lens — What the Observer Sees

Each observer wears a PositionLens that determines its trade-state vocabulary (T0-005):

LensTrade AtomsDescription
Core5Excursion, retracement, age, peak-age, signaled. The consensus view.
Full13All Core atoms + 3 phase biography atoms. All three voices plus phase context.
"Core: lean — regime + time. 5 trade atoms downstream." — T0-005
"Full: rich — regime + time + phase series. 13 trade atoms downstream." — T0-005

N is the number of market observers (one per MarketLens variant); M is the number of exit observers (one per PositionLens variant). N×M brokers bind every market observer to every exit observer.

Discrepancies

The module docstring (lines 1–5) states: "Intentionally simpler than MarketObserver. No noise-subspace, no curve, no engram gating."

The struct does have a noise_subspace field and a strip_noise method. The docstring contradicts the code. All 7 tests confirm the noise subspace is initialized and functional.

Construction


PositionObserver::new(lens: PositionLens, dims: usize, recalib_interval: usize,
                      default_trail: f64, default_stop: f64)
  • Two continuous Reckoners: "trail-{lens}" and "stop-{lens}", each with 10 buckets.
  • noise_subspace: 8 principal components, initialized with (dims, 8).
  • Test constants: DIMS=4096, RECALIB=100, DEFAULT_TRAIL=0.02, DEFAULT_STOP=0.05.

Position in the Pipeline

The exit observer sits between market observers and brokers in the concurrent pipeline:

flowchart TD
    market_obs["Market Observer"] -->|"Topic\n(MarketChain)"| position_obs["Position Observer"]
    position_obs -->|"Queue\n(MarketPositionChain)"| broker["Broker"]
    broker -.->|"ObsLearn\n(Mailbox)"| market_obs
    broker -.->|"PositionLearn\n(Mailbox)"| position_obs
  • Input: MarketChain from each market observer (via Topic, fan-out 1→N).
  • Output: MarketPositionChain to each broker (via Queue).
  • Feedback: PositionLearn signals from brokers (via Mailbox, fan-in N→1).

Each arrow is a Queue. The fan-out uses Topic (1→N). The learn-signal aggregation uses Mailbox (N→1).
