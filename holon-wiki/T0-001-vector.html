Vector — the atomic data structure of holon-rs HOW

Source: holon-rs/src/kernel/vector.rs (132 lines) Verified: 2026-04-14

#What it IS

Vector is a struct containing a single field: a Vec<i8> called data. Each element is constrained to the set {-1, 0, 1} — a bipolar vector with a zero state.

"A high-dimensional vector with bipolar elements {-1, 0, 1}." "This is the core data structure for all VSA operations."

The source doc comment states the purpose plainly: "Holon uses bipolar vectors with elements in {-1, 0, 1}. Internally stored as i8 for memory efficiency."

The struct derives Clone and Debug. It implements PartialEq/Eq (exact element-wise equality) and Index/IndexMut<usize> for direct element access.

#How it works

#Construction

Constructor Input Behavior
Vector::zeros(dimensions) usize Returns a zero-filled vector of the given length.
Vector::from_data(data) Vec<i8> Wraps raw i8 data directly — no validation that elements are in {-1, 0, 1}.
Vector::from_f64(values) &[f64] Thresholds each value: positive → 1, negative → -1, zero → 0.

The from_f64 constructor is the only place bipolar discipline is enforced. from_data trusts the caller.

#Query operations

  • dimensions() → returns data.len()
  • data() / data_mut() → raw slice access
  • nnz() → count of non-zero elements
  • norm() → L2 norm (Euclidean): computed as sqrt(Σ v_i²) where each v_i is cast to i64 before squaring to avoid overflow
  • normalized() → returns Vec<f64> (not a Vector), each element divided by the L2 norm; returns a zero vector if the norm is below 1e-10
  • to_f64() → casts each element to f64

#Key properties

  1. No dimension check on construction. from_data and from_f64 accept any-length input. No fixed dimensionality is enforced at the type level.
  2. No bipolar invariant enforced at rest. Only from_f64 guarantees {-1, 0, 1}. from_data and data_mut() allow arbitrary i8 values to be injected.
  3. Equality is element-wise exact. PartialEq compares data directly.
  4. Normalization escapes the type. normalized() returns Vec<f64>, not Vector — it leaves the bipolar domain.
  5. Sparse-friendly API. nnz() is provided, suggesting vectors may often be mostly zeros.
