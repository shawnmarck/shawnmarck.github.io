Treasury — Bounded Capital Allocation EXPLAIN

TL;DR: The treasury is a capital reservation engine. It receives trade proposals from brokers, funds those that pass an edge gate, and settles them through stop-level triggers. Loss is bounded structurally: reserved capital cannot be double-spent, and the maximum loss per trade equals one swap's venue cost. The broker is the accountability gate — it only proposes when its expected dollar-P&L is positive.

Funding: fund-proposals#


Proposals arrive via submit-proposal, which pushes onto the proposals vector. fund-proposals evaluates all proposals, sorted by edge descending (best first), and greedily funds from the top.


Funding Gates (checked per proposal)#


    

  1. Edge gate: edge < venue-cost-rate → rejected. A proposal must clear the round-trip cost to be worth funding.
    2. 

  2. Capital gate: available-capital ≤ 0.0 → rejected. No capital, no trade.
    3. 



Venue cost rate: 2.0 × (swap-fee + slippage) — the open+close round-trip.


Funding Math#


reserve-amount = amount + (amount × venue-cost-rate)   // principal + buffer
actual-reserve = min(reserve-amount, available-capital) // clamped
trade-amount   = actual-reserve / (1.0 + venue-cost-rate)  // back-computed principal



Capital moves available → reserved by trade-amount. Entry price initialized to 0.0 — set later by the enterprise from the current price. Stop levels computed via distances-to-levels(distances, entry-price, side). Proposals vector is drained after processing.


Returns: Vec<LogEntry> — one per proposal (funded or rejected).




Greedily funds top-to-bottom. First proposal to deplete an asset's available capital blocks subsequent proposals for that asset.






Settlement: settle-triggered#


Checks all active trades against their stop-levels using current prices. Two trigger paths:


Path A — Safety Stop#


    

  • Condition: trade is :active; price crosses safety-stop in the adverse direction (buy: price ≤ safety-stop; sell: price ≥ safety-stop).
    • 

  • Exit value: amount × (1.0 - venue-cost-per-swap) — loses one swap's venue cost on exit.
    • 

  • Loss: amount - exit-value = amount × venue-cost-per-swap.
    • 

  • Outcome: :settled-violence, reason :violence.
    • 



Path B — Trailing Stop#


    

  • Condition: trade is :active or :runner; price crosses trail-stop in the adverse direction.
    • 

  • Exit value: amount × (price / entry-price) × (1.0 - venue-cost-per-swap) — proportional P&L minus exit cost.
    • 

  • Outcome: :grace if exit-value > amount (profit), :violence otherwise (loss).
    • 

  • Capital: min(exit-value, amount) returns to available; full amount removed from reserved.
    • 



Both paths: trade and origin removed from their maps. Returns (Vec<TreasurySettlement>, Vec<LogEntry>).




The Runner Transition: update-trade-stops#


Replaces stop-levels on a specific trade. Handles the runner transition:


    

  • If trade is :active and the new trail-stop would recover principal (trail-stop past entry-price), the phase transitions to :runner.
    • 

  • Buy: trail-stop > entry-price; Sell: trail-stop < entry-price.
    • 



Called from outside the treasury (by the post's tick step 3c).




Bounded Loss: Structural, Not Policy#


The treasury enforces loss bounds through capital reservation — not through risk parameters or position limits.


    

  1. At funding: Capital moves from available to reserved. The reserve map is the accounting enforcement: capital in reserved cannot be redeployed. It only returns to available through settlement.
    

    2. 

  2. At settlement (safety stop): The maximum loss is amount × venue-cost-per-swap — one swap's cost on exit. The rest of the principal returns to available. This is the hard bound.
    

    3. 

  3. At settlement (trailing stop): P&L is proportional to price movement. Profits return as available capital. Losses are bounded by the same formula as the safety stop.
    

    4. 

  4. No double-spending is possible because proposals are drained after funding.
    

    5. 





"The bound is structural, not policy. The reserve map is the accounting enforcement." — T3-treasury-wat






The Broker's Gate#


The broker sits between the market and the treasury. It decides whether proposals are worth submitting.


gate_open() — Two-Phase Gate#


cold_start = grace_count < 50 || violence_count < 50
cold_start || expected_value > 0.0



    

  1. Cold start (< 50 of either outcome): gate always open — not enough data for a statistical judgment.
    2. 

  2. Warm (≥ 50 of both): gate opens only when expected_value > 0.0. A broker with negative expected value is suppressed.
    3. 



Expected Value Computation#


expected_value = grace_rate × avg_grace_net + (1 - grace_rate) × avg_violence_net



    

  • avg_grace_net and avg_violence_net are EMAs (alpha = 0.038, half-life ~50 papers).
    • 

  • P&L computed against a reference capital of $10,000, including swap fees on both entry and exit.
    • 

  • Propagated from propagate() after each resolution.
    • 





Full Lifecycle#


1. register_paper()     — broker creates a paper trade every candle
2. tick_papers()        — broker ticks papers: Grace signal, Violence stop, or runner resolution
3. propagate()          — broker accumulates outcomes, updates EV, returns PropagationFacts
4. gate_open()          — broker checks EV threshold (cold start always passes)
5. submit-proposal()    — proposal pushed to treasury
6. fund-proposals()     — treasury evaluates edge + capital gates, funds or rejects
7. update-trade-stops() — treasury updates stops; runner transition if trail recovers principal
8. settle-triggered()   — treasury settles triggered trades, returns capital to available
9. PropagationFacts     — routed back through broker to observers for learning

Known Issues#


    

  • total-equity comment says "converted to denomination" but the code sums raw f64 values without conversion. If multiple assets exist, this is a bug or incomplete spec.
    • 

  • Entry price set to 0.0 at funding time. The enterprise must update it before the first tick — otherwise settle-triggered would divide by zero (price / entry-price).
    • 

  • Broker close_all_runners accepts _current_price but does not use it — all resolution data comes from the paper's own tracked price history.
    •
