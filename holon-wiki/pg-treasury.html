Treasury — the bank. Headless. Blind to strategy. EXPLAIN

TL;DR: The treasury is a capital allocator and position lifecycle manager. It issues paper positions (fixed $10,000, no capital movement — proof of thoughts) and real positions (variable amount, actual capital — proof of execution). The gate reads paper track records to decide who earns real capital: minimum 50 papers submitted with >50% survival rate. Every position has a deadline — the clock. Positions resolve to Grace (profitable exit, residue after fees) or Violence (deadline expired, capital reclaimed). The treasury runs as its own service thread — one mailbox, two event types (Tick, Request), per-broker response queues. On shutdown, the state comes home.

One mailbox. Two event types. Per-broker response queues. The broker holds a TreasuryHandle. It sends TreasuryRequest into the shared mailbox and blocks on its own QueueReceiver<TreasuryResponse>. No polling. No spinning. No deadlock.

Position Lifecycle: PositionState#

One enum, not bool + Option. Every position (paper or real) moves through the same state machine:

stateDiagram-v2
    [*] --> Active : issue_paper() / issue_real()
    Active --> Grace : resolve_grace() — price moved favorably, residue > 0
    Active --> Violence : check_deadlines() — candle ≥ deadline
    Grace --> [*]
    Violence --> [*]
Variant Meaning
Active Open, clock ticking. Normal operating state.
Grace { residue: f64 } Exited profitably. Residue is the profit after exit fees. Terminal.
Violence Deadline hit, position reclaimed. For real positions, market value (minus exit fee) returns to balances. Terminal.

Paper Positions#

Papers don't move capital. They are proof of thoughts — a fixed $10,000 reference trade that simulates entry arithmetic without touching balances. Always issued. The cost is computation (lab) or gas (Solana).

PaperPosition struct#

Real Positions#

Real positions move capital. The treasury deducts from balances on issue and returns (market value minus exit fee) on Violence. The broker does not choose the amount — the treasury decides. Currently fixed at $50 per position (or available balance, whichever is smaller). Requires a proven record.

owner usize Broker slot index

deadline usize entry_candle + N — the clock

Distinct type, distinct issuance. Papers and real positions use separate ID counters and separate HashMap storage. The issue_paper() and issue_real() methods are fundamentally different — paper always succeeds; real requires gate + balance.

Proposer Record: the Gate#

The gate reads paper stats only to decide trust. Real stats measure whether the trust was deserved — they track performance after the gate opens.

ProposerRecord struct#

Field Type Purpose
paper_submitted usize Total papers submitted (proof of thoughts)
paper_survived usize Papers resolved to Grace
paper_failed usize Papers resolved to Violence (deadline)
paper_grace_residue f64 Cumulative profit from paper Graces
real_submitted usize Total real positions submitted
real_survived usize Real positions resolved to Grace
real_failed usize Real positions resolved to Violence
real_grace_residue f64 Cumulative profit from real Graces

Default: all zeros. Records are created lazily via entry().or_default() on first interaction.

Treasury Methods#

new(entry_fee, exit_fee) → Treasury#

Constructor. Empty maps, zero counters. Fees expressed as rates (e.g. 0.0035 = 0.35%).

issue_paper(owner, from_asset, to_asset, price, candle, deadline_candles) → PositionReceipt#

Always succeeds. Creates a paper position with fixed $10,000 amount. Computes entry fee, converts to to_asset units. Increments the owner's paper_submitted count. No balances are touched.

fee = 10_000.0 × entry_fee
units = (10_000.0 - fee) / price
deadline = candle + deadline_candles

issue_real(owner, from_asset, to_asset, price, candle, deadline_candles) → Option<PositionReceipt>#

Requires gate predicate AND positive balance. Three checks that return None:

  1. No ProposerRecord exists for this owner.
  2. gate_predicate() returns false.
  3. Balance for from_asset is zero or missing.

If approved: allocates min($50, balance) from balances, creates the real position, deducts the amount. Capital moves.

amount = min(50.0, balance[from_asset])
fee = amount × entry_fee
units = (amount - fee) / price
balances[from_asset] -= amount

The broker does NOT choose the amount. The treasury decides. For now: fixed $50 per position. Later: proportional to record quality and available balance.

get_paper_position(paper_id) → Option<&PaperPosition>#

Read a paper position. Pure lookup.

get_real_position(position_id) → Option<&RealPosition>#

Read a real position. Pure lookup.

validate_exit(paper_id, current_price) → Option<f64>#

Validates an exit proposal. Three checks:

  1. Paper must exist.
  2. State must be Active.
  3. Residue must be positive: current_value - original_amount - exit_fee > 0.
current_value = units_acquired × current_price
exit_fee = current_value × exit_fee
residue = current_value - amount - exit_fee

Returns Some(residue) if profitable, None otherwise.

resolve_grace(paper_id, current_price) → Option<TreasuryVerdict>#

Calls validate_exit(). If approved, transitions the paper to Grace { residue }, increments the owner's paper_survived and adds residue to paper_grace_residue. Returns TreasuryVerdict::Grace.

check_deadlines(current_candle, current_price) → Vec<TreasuryVerdict>#

Scans all active papers and real positions. Any position where current_candle ≥ deadline transitions to Violence.

For papers: increments the owner's paper_failed. No capital movement.

For real positions: recovers market value back to balances, computes loss.

market_value = units_acquired × current_price
exit_fee = market_value × exit_fee
recovered = market_value - exit_fee   (floored at 0)
balances[from_asset] += recovered
loss = amount - recovered
real_failed += 1
real_violence_loss += max(0, loss)

The treasury gets back what the asset is actually worth, not what was lent. If the position lost value, the treasury absorbs the loss.

gate_predicate(record) → bool#

Reads PAPER stats only. Two conditions, both required:

  1. paper_submitted ≥ 50 — minimum volume threshold.
  2. paper_survived / (paper_survived + paper_failed) > 0.5 — majority survival among resolved papers.
resolved = paper_survived + paper_failed
if resolved == 0 → false  (no resolved papers yet)
survival_rate = paper_survived / resolved
return survival_rate > 0.5

get_record(owner) → Option<&ProposerRecord>#

Read access to a proposer's track record. Pure lookup.

observe_atr(&mut self, atr: f64)#

Records current ATR. On the first non-zero reading, locks reference_atr to that value permanently. Subsequent calls only update current_atr.

scaled_deadline(&self, base: usize) → usize#

Volatility-scaled deadline: base × (reference_atr / current_atr). Returns base unchanged when current_atr is zero (no ATR data yet). Higher current volatility (larger ATR) shortens the deadline; lower volatility extends it.

TreasuryTickSender#

Handle for the main loop to send ticks. Wraps the event sender. send_tick(candle, price, atr) pushes a Tick event into the mailbox.

Program Loop#

while let Ok(event) = event_rx.recv() {
    match event {
        Tick { candle, price, .. } => {
            treasury.check_deadlines(candle, price);
            // diagnostics every 1000 candles
        }
        Request { client_id, request } => {
            let response = handle_request(&mut treasury, client_id, request, current_candle, base_deadline);
            client_txs[client_id].send(response);
        }
    }
}
// On mailbox disconnect: returns Treasury (state comes home)

On shutdown, the Treasury struct is returned — the state comes home for inspection or persistence.

Capital Conservation#

The system enforces capital conservation through three mechanisms:

  1. Papers don't move capital. issue_paper() never touches balances. The conservation_papers_dont_move_capital test verifies this with an assertion.

  2. Real positions deduct at issue, recover at Violence. issue_real() deducts amount from balances. check_deadlines() returns market_value - exit_fee (which may be less than the original amount — the treasury absorbs the loss).

  3. The gate restricts real issuance. Without a proven paper record (≥50 papers, >50% survival), issue_real() returns None. No capital is deployed to unproven brokers.

Papers are the training ground. They build the record that earns real capital. Real positions are the deployment — actual money, actual risk, actual accountability. The gate is the one-way valve between them.
