T-1-004 — Lab GUIDE.md Verification WHY

TL;DR

The GUIDE.md (2,568 lines, 126KB) is the master architectural blueprint for a self-organizing trading system built on holon-rs primitives. It declares every struct, interface, and dependency — no implementation, only specification. The system is an "organism" that observes markets via candle data, encodes observations as high-dimensional vectors (thoughts), predicts direction and exit distances via Reckoners, and learns from outcomes (Grace/Violence). The architecture is pair-agnostic: deploy capital, ride the trade, recover principal, accumulate residue (permanent gain). Five holon-rs primitives (atom, bind, bundle, cosine, reckoner), one learning mechanism (the Reckoner), one accountability surface (the curve). Everything is specified in s-expression (Scheme-like) syntax implemented by wat files; Rust implements what the wat specifies.

1. Overall Architecture and Philosophy

Core Metaphor: The Organism

The system is described as a "machine that measures thoughts against reality." A thought is a high-dimensional vector (10,000 dimensions by default). The machine encodes market candles into thoughts and measures whether those thoughts predicted value (Grace) or destruction (Violence).

The Enterprise

The enterprise is a tree of Posts. Each post watches one asset pair — (USDC, WBTC), (SPY, GOLD), any pair. No cross-talk between posts. The enterprise coordinates the four-step loop across all posts and the Treasury.

Hierarchical containment:

  • Binary → creates the world, drives the fold, writes the ledger
  • Enterprise → coordination plane, holds posts + treasury
  • Post → per-asset-pair unit, owns observers + brokers + indicator bank
  • Broker → binds observers as a team (accountability unit)
  • Observers → perceive and learn (market observers predict direction, exit observers estimate distances)
  • Reckoner → the learning primitive (one per observer concern)

ctx — The Immutable World

Born at startup, flows as a parameter. Contains three fields: ThoughtEncoder, dims, recalib-interval. Nobody owns it, everybody borrows it. One seam: the ThoughtEncoder's composition cache is mutable, but only between candles (not during). Cache misses are returned as values during parallel encoding, inserted sequentially after all steps complete.

Key Principles

  1. Values up, not queues down — side-effects flow up through return types, not down through queue parameters or shared mutation.
  2. The binary orchestrates — it does not think, predict, or learn.
  3. One encoding path — encoding is identical at prediction and resolution time.
  4. No zones, only scalars — emit continuous values, let the discriminant learn boundaries. "Overbought" is a premature measurement.
  5. Pair-agnostic — the same mechanics work for any asset pair in either direction.
  6. Residue compounds — permanent gain stays as the target asset, never withdrawn.

2. The Four-Step Loop (per candle, per post)

The enterprise's on-candle is the root. Each candle flows through:

Step 1: RESOLVE + PROPAGATE (real trades)

  • Treasury settles triggered trades against current prices
  • For each settlement: computes direction (exit vs entry price) and optimal distances (via Simulation replaying full price-history)
  • Routes outcomes through enterprise → posts → Brokers → propagation → observers learn
  • Both settlement paths: safety-stop fires → Violence (bounded loss); trailing-stop fires → Grace (residue) or Violence

Step 2: COMPUTE + DISPATCH

  • Post ticks its IndicatorBank on the raw candle → enriched Candle
  • Market observers encode simultaneously (par_iter) — the heaviest step (6× 10K-dim encodings)
  • Each market observer: calls vocabulary modules matching its Lens → produces Vec → wraps in Bundle → encodes via ThoughtEncoder → (Vector, Prediction, edge, misses)
  • Exit observers receive market-thoughts → encode exit facts → evaluate-and-compose → composed vector + distances
  • Brokers propose(composed) → Grace/Violence prediction
  • Post assembles Proposals (composed-thought, distances, edge, side, assets, prediction, indices) → submits to treasury

Step 3a: TICK (parallel)

  • All 24 brokers tick their papers simultaneously (par_iter)
  • Each broker touches only its own paper deque — disjoint, lock-free
  • Produces Vec facts (not mutations)
  • Papers don't carry price-history; approximate optimal distances from tracked extremes (MFE/MAE)

Step 3b: PROPAGATE (sequential — paper resolutions)

  • Fold over resolutions from 3a → apply to shared observers
  • Brokers learn Grace/Violence. Market observers learn Up/Down. Exit observers learn optimal distances
  • Same broker.propagate mechanism as Step 1, different source (papers vs real trades)

Step 3c: UPDATE TRIGGERS (sequential)

  • Treasury passes active trades per post
  • Posts compose fresh thoughts (with current context), query exit observers for updated distances
  • "The stops breathe" — distances adapt each candle based on current market context
  • Enterprise writes updated Levels back to treasury
  • Runner transition: when trailing stop passes break-even, trade phase changes from :active to :runner

Step 4: COLLECT + FUND

  • Treasury evaluates all proposals, sorted by edge (the Curve's accuracy measure)
  • Funds top N that fit available capital; rejects the rest
  • Reserves amount + worst-case-venue-costs (2 swaps)
  • Proposal whose edge doesn't exceed total venue cost rate → rejected
  • Drains proposals after funding

3. The Vocabulary System

Structure

Three domains, each with scoped subfiles:

Shared (shared/)

  • time.wat — minute (mod 60), hour (mod 24), day-of-week (mod 7), day-of-month (mod 31), month-of-year (mod 12). Circular scalars.

Market (market/) — direction signal

Mapped by MarketLens:

  • :momentum → oscillators, momentum, stochastic
  • :structure → keltner, fibonacci, ichimoku, price-action
  • :volume → flow
  • :narrative → timeframe, divergence
  • :regime → regime, persistence
  • :generalist → all of the above

Every MarketLens variant includes shared/time AND standard automatically.

Market vocabulary modules (each emits a defined set of atoms):

  • oscillators.wat — rsi, cci, mfi, williams-r, roc-1/3/6/12
  • flow.wat — obv-slope, vwap-distance, buying/selling-pressure, volume-ratio, body-ratio
  • persistence.wat — hurst, autocorrelation, adx
  • regime.wat — kama-er, choppiness, dfa-alpha, variance-ratio, entropy-rate, aroon-up/down, fractal-dim
  • divergence.wat — rsi-divergence-bull/bear, divergence-spread
  • ichimoku.wat — cloud-position, cloud-thickness, tk-cross-delta, tk-spread, tenkan-dist, kijun-dist
  • stochastic.wat — stoch-k, stoch-d, stoch-kd-spread, stoch-cross-delta
  • fibonacci.wat — range-pos-12/24/48, fib-dist-236/382/500/618/786
  • keltner.wat — bb-pos, bb-width, kelt-pos, squeeze, kelt-upper/lower-dist
  • momentum.wat — close-sma20/50/200, macd-hist, di-spread, atr-ratio
  • price-action.wat — range-ratio, gap, consecutive-up/down, body-ratio-pa, upper/lower-wick
  • timeframe.wat — tf-1h-trend/ret, tf-4h-trend/ret, tf-agreement, tf-5m-1h-align
  • standard.wat — since-rsi-extreme, since-vol-spike, since-large-move, dist-from-high/low/midpoint/sma200, session-depth. Takes candle WINDOW (not just current candle).

Exit (exit/) — distance signal

Mapped by ExitLens:

  • :volatility → volatility.wat — atr-ratio, atr-r, atr-roc-6/12, squeeze, bb-width
  • :structure → structure.wat — trend-consistency-6/12/24, adx, exit-kama-er
  • :timing → timing.wat — rsi, stoch-k, stoch-kd-spread, macd-hist, cci
  • :generalist → all three

How Vocabularies Compose

  1. Each vocabulary module is a pure function: candle → Vec
  2. A ThoughtAST is deferred computation — data describing a composition, not yet executed
  3. The observer calls each module matching its lens, appends all AST lists
  4. Wraps in a Bundle ThoughtAST
  5. Calls encode(encoder, bundle-ast) on the ThoughtEncoder → (Vector, misses)
  6. Multiple fact-vectors get bundled into one thought via bundle primitive

Example fact composition:


"RSI is at 0.73" → (Linear "rsi" 0.73 1.0) → encoded as bind(atom("rsi"), encode-linear(0.73, 1.0)) → Vector

The AST


(enum thought-ast
  (Atom name)                    ; dictionary lookup
  (Linear name value scale)      ; bind(atom, encode-linear)
  (Log name value)               ; bind(atom, encode-log)
  (Circular name value period)   ; bind(atom, encode-circular)
  (Bind left right)              ; composition
  (Bundle children))             ; superposition

Encoding Schemes (chosen by vocabulary, not by encoder)

  • encode-linear — naturally bounded scalars (RSI [0,1], Bollinger position [-1,1])
  • encode-log — unbounded positive scalars (ATR ratio, volume ratio, band widths)
  • encode-circular — periodic scalars (minute mod 60, hour mod 24, day-of-week mod 7)

The vocabulary owns both encode and decode — it put the value on the scalar, it can take it back.

4. The Training Approach

The Reckoner — The Only Learning Primitive

The Reckoner accumulates experience and delivers a verdict. Internally builds a Discriminant — the direction in thought-space that separates outcomes. Cosine against the discriminant → Conviction.

Two modes:

  • Discrete — classification (Up/Down for market observers, Grace/Violence for brokers). Returns (label, score) pairs + conviction.
  • Continuous — regression (trail/stop distances for exit observers). Returns scalar value + experience.

Operations:

  • observe(reckoner, thought, observation, weight) — feed an observation
  • predict(reckoner, thought) → Prediction
  • decay(reckoner, factor) — old experience fades
  • experience(reckoner) → f64 — how much learned (0.0 = ignorant)
  • recalib-count(reckoner) → usize
  • Recalibration: periodically recomputes discriminant from accumulated observations

The Curve — Self-Evaluation

The Reckoner carries its own Curve internally. resolve() feeds it, edge-at() reads it. Input: conviction. Output: accuracy. A continuous surface — not "is this proven?" but "how much edge at this conviction?"

Communicated as one scalar (f64): the producer calls edge-at(reckoner, conviction) and attaches the result to its message. The consumer encodes it as a scalar fact. No meta-journal, no curve snapshot.

Noise Subspace — OnlineSubspace

Learns what normal thoughts look like. strip-noise returns the anomalous component — what the subspace cannot explain. The Reckoner learns from the residual (unusual = signal, normal = boring).

Engram Gating

After recalibration with good accuracy, snapshot the discriminant as a "good state." A separate OnlineSubspace learns what good discriminants look like. Future recalibrations are checked against this memory.

Shared by both MarketObserver and Broker — same four fields, different reckoner, different purpose.

Scalar Accumulators

Per-magic-number f64 learning for distances (trail, stop). Lives on the broker. Separates Grace/Violence observations into separate vector prototypes. extract-scalar sweeps candidate values against the Grace prototype via cosine. "What value does Grace prefer for this pair overall?" Global per-pair, not contextual.

The Distance Cascade

When queried for a distance:

  1. If exit Reckoner is experienced → predict(reckoner, composed) (contextual — for THIS thought)
  2. Else if broker Accumulator has data → extract-scalar(accum, ...) (global per-pair)
  3. Else → default (crutch value)

Paper Trades — The Fast Learning Stream

Every candle, every broker gets a paper entry (both buy and sell sides tracked simultaneously). When both sides resolve, the paper teaches optimal distances. Papers are treated equally with real trades by the learning system. Both start ignorant (crutch values), both feed Grace/Violence back.

Ignorance → Bootstrap

Every reckoner begins with zero experience. No special bootstrap logic. Papers fill the reckoner, experience grows, the Treasury starts listening. "Start ignorant. Learn. Graduate."

Propagation — Routing Learning Signals

When a trade or paper resolves:

  • Grace/Violence → broker's reckoner + internal Curve + engram gate + track record + Accumulators
  • Actual direction (Up/Down) → market observer's reckoner via resolve
  • Optimal distances → exit observer's reckoners via observe-distances
  • Weight: magnitude of outcome (large Grace teaches harder)

5. Key Design Decisions and Principles

Residue-Based Growth (The Core Economic Model)

  • Deploy capital → ride the trade → recover principal → remainder is permanent gain
  • Residue stays as the target asset (not swapped back)
  • Both sides of any pair can grow (USDC recycled, WBTC accumulated; or vice versa)
  • Two swaps total per trade (entry + exit). Runner phase is NOT a swap — it's stop widening.
  • The goal: minimize capital lost, maximize wealth accumulated

Trade Phase State Machine

  • :active → :runner (when trailing stop covers principal) → :settled-grace (trailing stop fires with profit)
  • :active → :settled-violence (safety stop fires, bounded loss)
  • :runner → :settled-grace or :settled-violence (trailing stop fires)

Lock-Free Parallel Access

  • Brokers live in a flat vec with pre-computed slot indices
  • At construction: enumerate all sets → allocate flat vec → build frozen map
  • At runtime: frozen map (read-only) → slot-idx → &mut broker (disjoint)
  • The borrow checker proves writes are disjoint. No mutex.

Pair-Agnostic Architecture

  • Asset pair is a routing key — only the matching post receives a candle
  • Any pair, any direction — the mechanics are identical
  • N = number of market observers (one per MarketLens variant), M = number of exit observers (one per ExitLens variant)
  • N×M brokers total

Values Up, Not Queues Down

  • Cache misses, log entries, propagation facts — all flow up through return types
  • No queue parameters. No shared mutation during parallel phases.
  • The enterprise collects misses from all steps, returns them alongside log entries
  • The binary (not the enterprise) inserts misses into ctx's cache between candles

The Stops Breathe

  • Every candle, step 3c re-queries exit observers for fresh distances
  • Contextual: different thoughts get different distances based on current market
  • Not set-and-forget. Adapts continuously.
  • Update cost considered: when on-chain (gas cost), skip micro-adjustments that don't materially change levels

No Zones, Only Scalars

  • "Overbought" is a premature measurement — a magic number wearing a name
  • Emit the scalar, let the discriminant learn where boundaries are
  • The sign carries direction. The magnitude carries conviction.

The Message Protocol

Every learned message carries: (thought: Vector, prediction: Prediction, edge: f64).

  • Thought = what you know
  • Prediction = what you think will happen
  • Edge = how accurate you are when you predict this strongly ([0.0, 1.0])

6. Specific Algorithms, Formulas, and Data Structures

Struct Hierarchy (Construction Order)

  1. Asset — (struct asset [name : String])
  2. RawCandle — 8 fields: source-asset, target-asset, ts, open, high, low, close, volume
  3. Candle — raw OHLCV + 100+ computed indicators (SMA, Bollinger, RSI, MACD, DMI, ATR, Stochastic, CCI, MFI, OBV, Keltner, ROC, range position, trend consistency, multi-timeframe, Ichimoku, persistence metrics, regime metrics, divergence, cross deltas, price action, timeframe agreement, time fields)
  4. IndicatorBank — streaming state machine composed from ring buffers, EMA states, Wilder smoothers, RSI states, ATR states, OBV states, SMA states, rolling stddev, stochastic states, CCI states, MFI states, Ichimoku states, MACD states, DMI states
  5. WindowSampler — deterministic log-uniform window selection (seed, min, max)
  6. Vocabulary modules — pure functions producing Vec
  7. ThoughtEncoder — atoms dictionary (Map) + compositions LRU cache (Map)
  8. Distances — {trail: f64, stop: f64} (percentages)
  9. Levels — {trail-stop: f64, safety-stop: f64} (absolute prices)
  10. ScalarAccumulator — name, encoding (log/linear/circular), grace-acc, violence-acc, count
  11. EngramGate — recalib-wins, recalib-total, last-recalib-count
  12. MarketObserver — lens, reckoner (discrete), noise-subspace, window-sampler, resolved count, engram gate fields, last-prediction
  13. ExitObserver — lens, trail-reckoner (continuous), stop-reckoner (continuous), default-distances
  14. Broker — observer-names, slot-idx, exit-count, reckoner (discrete), noise-subspace, cumulative grace/violence, trade-count, papers, scalar-accums, engram gate
  15. Post — post-idx, assets, indicator-bank, candle-window, market-observers [N], exit-observers [M], registry (brokers [N×M]), encode-count
  16. Treasury — denomination, available, reserved, proposals, trades, trade-origins, swap-fee, slippage, next-trade-id
  17. Enterprise — posts, treasury, market-thoughts-cache

Key Enums

  • Side — :buy, :sell (trading action, on Proposal and Trade)
  • Direction — :up, :down (price movement, used in propagation)
  • Outcome — :grace, :violence (accountability)
  • ThoughtAST — Atom, Linear, Log, Circular, Bind, Bundle
  • TradePhase — :active, :runner, :settled-violence, :settled-grace
  • LogEntry — ProposalSubmitted, ProposalFunded, ProposalRejected, TradeSettled, PaperResolved, Propagated

Slot Indexing

slot-idx = market-idx × M + exit-idx — one broker per (market, exit) pair. market-idx = slot-idx / exit-count, exit-idx = slot-idx mod exit-count.

Proposed Naming Conventions (from Proposal 050/051 per CLAUDE.md)

  • "Exit observer" renamed to Position observer in the running codebase (Proposal 050)
  • Proposal 051 deleted the binary Grace/Violence reckoner on brokers — continuous reckoners only

Note: The GUIDE.md itself still uses "exit observer" throughout and describes a discrete reckoner on the broker. This is a potential divergence with CLAUDE.md's statement about Proposal 051. The GUIDE.md is declared authoritative: "When layers diverge, the guide is right."

Performance Targets

  • 75–500 candles/second at 10,000 dimensions
  • Prior Rust sustained 251/s
  • ~2M float operations per candle (10 observers × ~20 atoms × 10K dims)
  • SIMD support (AVX2/NEON) gives ~5x on hot-path primitives

Default Parameters

  • dims: 10,000
  • recalib-interval: 500
  • noise-subspace k: 8 principal components
  • good-state-subspace k: 4 principal components
  • default trail: 0.015 (1.5% of price)
  • default stop: 0.030 (3.0% of price)
  • swap-fee: 0.0010 (10 bps)
  • slippage: 0.0025 (25 bps)
  • max-window-size: 2016
